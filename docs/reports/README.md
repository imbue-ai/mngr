# Reports

Point-in-time analyses of this repository: audits, investigations, and state-of-the-world write-ups. These are snapshots, not living documentation — each file is date-prefixed, states its vantage point (branch and version) in its header, and should be read with that date in mind. When a report and the code disagree, the code won; follow the citations.

This directory exists so that reports have a legible home and do not accumulate at the repo root (see `ELECTRON_BUNDLING_AUDIT.md` at the root for the failure mode this avoids). Living documentation belongs in the relevant project's `docs/` directory, not here.
