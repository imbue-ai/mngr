# Changelog

A concise, human-friendly summary of changes. Updated nightly by the changelog consolidation agent.

For the full, unedited changelog entries, see [UNABRIDGED_CHANGELOG.md](UNABRIDGED_CHANGELOG.md).

## 2026-05-09

- New `mngr_forward` plugin extracts auth and subdomain forwarding out of the minds desktop client; a new `minds run` command spawns it as a subprocess and the legacy in-process `minds forward` paths and helpers are removed.
- New `mngr_imbue_cloud` plugin owns SuperTokens auth, pool-host leasing, LiteLLM keys, and Cloudflare tunnels; `mngr create --provider imbue_cloud_<account> --new-host` now leases a pool host and adopts its pre-baked agent in one shot, with multi-account modelled as multiple provider instances. Pool-hosts schema migrates `version` to a flexible `attributes JSONB` column; revoked sessions auto-disable the corresponding provider in minds.
- New nightly changelog system: per-PR entry files in `changelog/` (CI-enforced via meta ratchet) get consolidated into `UNABRIDGED_CHANGELOG.md` and an AI-summarized `CHANGELOG.md`, with an idempotent `setup_changelog_agent.sh` cron installer.
- minds desktop client: redesigned "Create a Project" screen separating compute provider from AI provider (`imbue_cloud` / `api_key` / `subscription`) and adding an optional `GH_TOKEN` field; "Share" button now opens an info modal instead of writing back to minds; destroyed agents disappear from the projects index automatically; account identity is no longer persisted to disk; a grandparent-death watcher self-terminates the Python backend when Electron crashes.
- minds workspace server moved out of the monorepo into forever-claude-template (`apps/system_interface/`) and is consumed at runtime; the release Dockerfile drops its node/npm build step.
- Latchkey gateway consolidated into a single shared, password-protected `latchkey gateway` subprocess for all agents (was one per agent), with per-agent `latchkey_permissions.json` symlinks, a deny-all default permissions file, DEV-mode agents now flowing through the gateway too, and the `latchkey` npm dep bumped to 2.8.0; `LATCHKEY_DISABLE_COUNTING=1` is injected to suppress per-agent goatcounter pings.
- mngr CLI ergonomics: the `host.provider` qualifier now works consistently anywhere a host identifier is accepted (`mngr limit`, `mngr snapshot`, `--host` filters, etc.); `mngr create --from` now works against remote sources and same-host source/target pairs (with `git-worktree` short-circuiting between local paths); `mngr create -vv` adds a transfer-timing log span.
- `mngr tmr` fixes: integrator and local-provider test-agent launches no longer fail with "Failed to generate a unique host name"; per-agent send failures are caught instead of crashing the orchestrator; HTML reports get a separate "Failed" section for infra failures and now include rows for tests whose agent failed to launch. New `.github/workflows/tmr.yml` runs `mngr tmr` on Modal manually and uploads the report; the CLI emits an `integrator_branch` event for consumers.
- mngr_modal refactor: `ModalMode.TESTING` removed from production paths (tests inject `TestingModalInterface` via `make_testing_provider`); `ModalMode` retains `DIRECT` (default) and `PROXIED` (reserved, raises `NotImplementedError`); shared `_construct_modal_provider` factory; `enable_output_capture` is now abstract; per-test Modal envs registered with the leak detector to catch silent cleanup failures, and best-effort modal-CLI cleanup helpers now surface non-zero exits.
- New `mngr_kanpan` staleness taint semantics: every `FieldValue` carries a `created` timestamp that propagates through cached inputs; cells older than `staleness_threshold_seconds` (default 90% of refresh interval) render dimmed, with `ShellCommandConfig` declaring its inputs explicitly.
- Wheel packaging: every workspace package now uniformly excludes `*_test.py`, `test_*.py`, `**/conftest.py`, and `**/testing.py` from built wheels (a new meta ratchet enforces this); `imbue_common.TEST_FILE_PATTERNS` extends to match. Fixes `libs/mngr` leaking `cli/testing.py`, `api/testing.py`, and `providers/docker/testing.py` and unblocks dropping the `only-include` workaround in `modal_proxy`.
- Bug fixes: local-host shell commands no longer crash with `child watchers are only available on the default loop` from worker threads; `mngr stop` no longer leaks orphan grandchildren (now matches by inherited `MNGR_AGENT_ID`); `propagate_changes` no longer wipes per-agent `.claude/settings.local.json` (which was hanging every `send_message`); WebSocket broadcaster evicts stuck clients after 50 queue-fulls and cancels wedged send tasks; remote event-tail follow-mode no longer lags by one event; `mngr create` provisions Claude credentials correctly inside nested sandboxes (lima VM on macOS); JSONL parsers raise on malformed input instead of silently dropping; `mngr connect` type-checks again; the Claude plugin update SessionStart hook no longer hangs Modal agents on SSH host-key prompts.
- Spec-only: design for exposing each host's outer machine via `OuterHost` base class and `mngr exec --outer`; design for backing up forever-claude-template's `runtime/` (including `memory/` and `tickets/`) to a private orphan branch with a periodic backup service.
- Offload bumped from 0.8.1 to 0.9.0 (history-based test scheduling balances sandbox load) and then to 0.9.2 (thin-diff fix; offload binary now in the sandbox image so `apply-diff` works without a full rebuild; `GITHUB_HEAD_REF`/`GITHUB_REF_NAME` propagate to sandboxes).
- Changelog cron: commit author email corrected from `dev@imbue.com` to `bot@imbue.com` so commits attribute to the bot's verified GitHub email.
