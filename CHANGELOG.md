# Changelog

A concise, human-friendly summary of changes. Updated nightly by the changelog consolidation agent.

For the full, unedited changelog entries, see [UNABRIDGED_CHANGELOG.md](UNABRIDGED_CHANGELOG.md).

## 2026-05-09

- New `mngr_imbue_cloud` plugin owns SuperTokens auth, pool-host leasing, LiteLLM keys, and Cloudflare tunnels; `mngr create --provider imbue_cloud_<account>` leases a pool host and adopts its pre-baked agent in one step. Pool-host schema replaces `version` with a `attributes JSONB` column.
- New `mngr_forward` plugin and `minds run` command split subdomain forwarding and auth out of the minds desktop client into a standalone process; the legacy in-process forwarder (`minds forward`, `desktop_client/runner.py`, `MngrStreamManager`, host-header middleware, etc.) is now deleted.
- Latchkey gateway redesigned: a single password-protected `latchkey gateway` per host (instead of one per agent), with per-agent JWT permission overrides and a deny-all baseline. Agent-side `latchkey` CLI must be bumped to 2.8.0; agents from older images need to be re-created. Daily goatcounter pings suppressed via `LATCHKEY_DISABLE_COUNTING=1`.
- Workspace server moved out of this monorepo into `forever-claude-template` as `apps/system_interface/` and ships as the `minds-workspace-server` CLI; the release Dockerfile no longer builds its frontend.
- Offload upgraded from 0.8.1 to 0.9.2, with history-based test scheduling, the offload binary baked into the sandbox image (so thin-diff apply works), and `GITHUB_HEAD_REF` / `GITHUB_REF_NAME` propagated to sandboxes.
- `mngr create --from` now works between any two hosts (including same remote host: short-circuits to local `git push` / rsync), accepts the `host.provider` qualifier consistently across commands, and resolves git/origin metadata via the host interface instead of raising `NotImplementedError`.
- `mngr` reliability fixes: local-host shell commands no longer crash with `child watchers are only available on the default loop` from worker threads; `mngr stop` cleans up grandchild orphans (e.g. `playwright-mcp`, `node`) by walking `/proc/<pid>/environ` for `MNGR_AGENT_ID`; `mngr connect` type-checking; remote `events.jsonl` follow-mode no longer lags one event behind.
- `mngr tmr` no longer aborts when an individual agent fails to launch, supports the `local` provider (reuses the single fixed host), and HTML reports gain a dedicated "Failed" section separate from "Blocked". A new `tmr.yml` GitHub workflow runs `mngr tmr` against Modal and opens a draft integrator PR.
- minds desktop client: redesigned the "Create a Project" screen with independent compute-provider and AI-provider choices, optional `GH_TOKEN` field, and form/server validation; revoked imbue_cloud sessions auto-disable their provider entry; an Electron grandparent-death watcher cleans up orphaned Python/mngr/latchkey processes on Electron crashes; SIGTERM during shutdown no longer surfaces as "subprocess failed"; destroyed agents disappear from the projects index without a manual click; closing the last workspace tab auto-reopens the primary chat tab.
- JSONL parsers (`MalformedJsonLineWarner.parse`, `parse_event_line`, `parse_discovery_event_line`, `parse_agents_from_mngr_output`, vps_docker batched JSON) now raise on corruption instead of silently dropping bad lines.
- WebSocket broadcaster evicts stuck clients after 50 consecutive queue-full broadcasts and cancels coroutines wedged in `websocket.send_text`, fixing a CPU/log flood from half-dead TCP connections.
- `mngr_kanpan` cell values now carry a `created` timestamp that propagates from cached inputs; values older than `staleness_threshold_seconds` (default 90% of the refresh interval) render dimmed.
- `mngr_modal` drops `TESTING` mode from production code (tests inject `TestingModalInterface` directly); production no longer imports `modal_proxy.testing`, fixing a `ModuleNotFoundError` in packaged consumers like `minds.app`. Session-scoped Modal envs now register with the leak-detection registry to prevent orphan-env accumulation.
- Wheel builds across every workspace package now uniformly exclude `*_test.py`, `test_*.py`, `**/conftest.py`, and `**/testing.py`, enforced by a new meta ratchet. Previously some packages were shipping test helpers into published wheels.
- `mngr create` provisions Claude credentials correctly in nested sandboxes (e.g. Linux lima VM on macOS) by falling back to `$CLAUDE_CONFIG_DIR` when `$ORIGINAL_CLAUDE_CONFIG_DIR` does not exist inside the sandbox.
- The `claude plugin update` SessionStart hook no longer hangs Modal-launched agents on an `ssh` first-contact prompt; the updater now sets `GIT_SSH_COMMAND='ssh -o StrictHostKeyChecking=accept-new -o BatchMode=yes'`.
- New changelog system: per-PR `changelog/*.md` entry files (CI-enforced via meta ratchet), nightly consolidation into `UNABRIDGED_CHANGELOG.md` and this file, and an idempotent setup script for the consolidation agent. Bot commit author email fixed to `bot@imbue.com` so consolidation commits attribute correctly.
- New specs: exposing each host's outer machine via `OuterHost` / `mngr exec --outer`, and backing up forever-claude-template's `runtime/` (including `memory/` and `tickets/`) to a separate orphan branch in the same private repo.
- `apps/minds/scripts/propagate_changes` now protects `.claude/settings.local.json` from `rsync --delete`, fixing a 90-second `send_message` hang after running the script.
- `mngr create -vv` emits a `Transferring agent files` log span around plugin file transfers (including the early-return path) so total file-push time is visible.
- Cleanup after the imbue_cloud / forward plugin splits: minds no longer persists `imbue_cloud` account identity to disk, the workspace "Share" button opens an informational modal instead of round-tripping through minds, and the `mngr imbue_cloud admin pool create` bake replaces the deleted `just create-pool-hosts*` recipes and `create_pool_hosts.py` script.
