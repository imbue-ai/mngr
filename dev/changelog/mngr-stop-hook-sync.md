Add a `uv-sync-pre-push` hook to `.pre-commit-config.yaml` (registered for the `pre-push` stage) that runs `uv sync --all-packages` before a push whenever that push touches dependency files (`uv.lock` or any `pyproject.toml`). This keeps the local environment in sync with just-merged dependencies, primarily for the case where the code-guardian stop hook merges `origin/main` and then pushes the merge commit: the sync now runs before the push completes and before any push-time review/CI gates. Pushes that do not change dependency files are unaffected (the hook is skipped).

To make this hook reproducibly installed (the repo deliberately avoids `pre-commit install`, which depends on the system Python), add a hand-written `scripts/githooks/pre-push` shim mirroring the existing `scripts/githooks/pre-commit`, and update `scripts/githooks/install.sh` to symlink it into `.git/hooks/pre-push` alongside the pre-commit hook.
