Add `scripts/cleanup_old_vultr_test_instances.py`, an out-of-band, age-based reaper for Vultr VPS instances left behind by the mngr_vultr release tests (e.g. when a test session is killed before its in-process cleanup runs). It destroys instances whose `mngr-vultr-test-created` tag is older than `--max-age-hours` (default 1.0) and no-ops if `VULTR_API_KEY` is unset. Wired into CI as a `cleanup-vultr-instances` job mirroring the existing `cleanup-modal-environments` job.

Correct the docstrings of `scripts/cleanup_old_vultr_test_instances.py` and `scripts/cleanup_old_modal_test_environments.py`, and the `cleanup-vultr-instances` CI job comment, which previously described the cleanup as running "periodically" / "on a schedule" / "hourly". There is no cron trigger; these CI cleanup jobs run on every push to main and every pull request. The docstrings now say so.
