Sped up the minds snapshot CI build and simplified how its CI secrets are sourced.

- Sped up the minds snapshot build: `playwright install` (chromium + apt deps, ~24s, source-independent) now runs in a cached pre-repo image layer instead of re-running on every commit; and the per-commit `uv sync` + `pnpm install` now run concurrently (disjoint paths, both PID-waited so a failure still fails the build) so their network/IO overlaps instead of summing. Only the post-`COPY` deps layer rebuilds per commit. (A producer-sandbox CPU bump was tried and reverted: the in-sandbox FCT build is network/IO-bound, not CPU-bound, so more cores did not help.)

- The minds snapshot build now always uses the local **docker** builder for the in-sandbox forever-claude-template container; `scripts/snapshot_minds_e2e_state.py` prints a per-phase timing summary (`PHASE_TIMING ...`). depot.dev was evaluated and removed: for this ephemeral build-then-run-locally flow it measured ~2.5 min slower end-to-end, because `depot build --load` must export and download the whole image back into the sandbox's docker daemon -- cost a local build avoids. The `--builder` flag, the depot CLI baking, the `MINDS_SNAPSHOT_BUILDER` knob, and the depot Vault wiring are gone.

- The snapshot image build now compiles the minds Tailwind stylesheet (`pnpm run build:css`) before driving the Electron workspace-creation flow; the gitignored `app.min.css` is otherwise absent in the image and its missing `.hidden` rule made the onboarding flow appear stuck, failing the build.

- CI secrets now use the public `imbue-ai/use-vault-secrets` action (pinned by SHA) instead of a vendored copy of the export-secrets script; the vendored `.github/actions/export-secrets` directory was removed. Only `test-minds-snapshot` needs a secret now (`ANTHROPIC_API_KEY` from Vault via OIDC, role `minds_ci_test_gh`, gated on the `minds-ci-test` Environment); the `test-offload-minds-snapshot` recipe forwards it into the offload sandbox when present.

- `build-minds-snapshot` no longer touches Vault: it is a plain docker build, with no GitHub Environment and no `id-token` permission. Both snapshot jobs still skip fork PRs and stay behind the `DISABLE_MINDS_SNAPSHOT_CI` kill switch.
