MIND-142: Cut the `test-offload` / `test-offload-acceptance` CI wall clock by removing per-invocation coverage overhead from offload's sandbox batch execution and reducing checkpoint-cache misses (speculative retry/hedging behavior is untouched).

Root `pyproject.toml` addopts no longer include `--cov-report=term-missing`, `--cov-report=xml`, or `--coverage-to-file`: those per-run reports walk every measured file and are never consumed from offload sandboxes (CI combines the raw `.coverage` data files on the runner and enforces gates there). Removing them from addopts is also what lets offload batches pass `--cov-report=` to skip report generation entirely (pytest-cov only treats the empty value as "no reports" when it is the sole `--cov-report` given). `just test-unit` and `just test-integration` now pass those report flags explicitly, preserving their existing local behavior.

Root `[tool.coverage.run]` sets `core = "sysmon"` (PEP 669 `sys.monitoring`), roughly a 3x lower tracing overhead than the default C tracer on Python 3.12 for import-heavy workloads like test collection and short batch processes. Branch coverage, dynamic contexts, and greenlet/eventlet/gevent concurrency -- the documented incompatibilities -- are unused here; `multiprocessing`/`thread` concurrency keeps working.

`offload-modal.toml` `run_args` gains `--cov-report=` so batches generate no coverage reports but still write the `.coverage` data file the gates combine.

Checkpoint `build_inputs` in `offload-modal.toml` and `offload-modal-acceptance.toml` no longer include `libs/*/pyproject.toml` / `apps/*/pyproject.toml`: per-package metadata churn previously forced a full checkpoint rebuild (+~2min when missed) to keep sandboxed `uv run` from attempting a network-less editable rebuild. That failure mode predates `post_patch_cmd`, which reruns `uv sync --all-packages` after every thin diff and rebuilds any editable whose pyproject drifted, so those commits now ride the checkpoint via thin diff like everything else. Dependency changes still invalidate the checkpoint via `uv.lock` (any per-package dependency change necessarily also changes `uv.lock`).

Two further wins prototyped in this PR -- discovery-only `--no-cov` so `pytest --collect-only` skips coverage tracing (~2x faster discovery), and invoking `.venv/bin/pytest` directly instead of `uv run pytest` -- were moved upstream to offload as feature tickets (see the OFFLOAD tickets linked from the MIND-142 follow-up) rather than living as repo-config scaffolding.

`test_meta_ratchets.py` gains `test_offload_configs_suppress_per_batch_coverage_reports`, a meta test that guards the report-suppression invariant (no coverage-report flags in root addopts, `--cov-report=` in offload-modal run_args) against regression.
