MIND-142: Cut the `test-offload` / `test-offload-acceptance` CI wall clock by removing per-invocation coverage overhead and uv indirection from offload's discovery and sandbox batch execution (speculative retry/hedging behavior is untouched).

Root `pyproject.toml` addopts no longer include `--cov-report=term-missing`, `--cov-report=xml`, or `--coverage-to-file`: those per-run reports walk every measured file and are never consumed from offload sandboxes (CI combines the raw `.coverage` data files on the runner and enforces gates there). Removing them from addopts is also what lets offload batches pass `--cov-report=` to skip report generation entirely (pytest-cov only treats the empty value as "no reports" when it is the sole `--cov-report` given). `just test-unit` and `just test-integration` now pass those report flags explicitly, preserving their existing local behavior.

Root `[tool.coverage.run]` sets `core = "sysmon"` (PEP 669 `sys.monitoring`), roughly a 3x lower tracing overhead than the default C tracer on Python 3.12 for import-heavy workloads like test collection and short batch processes. Branch coverage, dynamic contexts, and greenlet/eventlet/gevent concurrency -- the documented incompatibilities -- are unused here; `multiprocessing`/`thread` concurrency keeps working.

`offload-modal.toml` and `offload-modal-acceptance.toml` now invoke `.venv/bin/pytest` directly (via a bash wrapper that prepends `.venv/bin` to `PATH` and sets `VIRTUAL_ENV`, matching what `uv run pytest` provided) instead of `uv run pytest`: sandbox network is disabled, so uv's resolution and editable-staleness checks were pure latency and a rebuild-attempt failure mode. Group `filters` gain a discovery-only `--no-cov` so `pytest --collect-only` no longer pays coverage tracing on module imports (~4x faster collection; the two group discovery passes drop from ~126s to tens of seconds on the runner). `offload-modal.toml` `run_args` gains `--cov-report=`.

Checkpoint `build_inputs` in both configs no longer include `libs/*/pyproject.toml` / `apps/*/pyproject.toml`: per-package metadata churn previously forced a full checkpoint rebuild (+~2min when missed) to keep sandboxed `uv run` from attempting a network-less editable rebuild. That failure mode is gone -- `post_patch_cmd` reruns `uv sync --all-packages` after every thin diff, and batches no longer use `uv run` -- so those commits now ride the checkpoint via thin diff like everything else. Dependency changes still invalidate the checkpoint via `uv.lock` (any per-package dependency change necessarily also changes `uv.lock`).

`test_meta_ratchets.py` gains `test_offload_configs_keep_batches_and_discovery_cheap`, a meta test that guards these invariants (no coverage-report flags in root addopts, `--cov-report=` in offload-modal run_args, `--no-cov` in every group's filters, `.venv/bin/pytest` as the framework command) against regression.
