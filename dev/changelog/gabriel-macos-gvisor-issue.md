The minds e2e snapshot build (`scripts/snapshot_minds_e2e_state.py`) now pins the create default to runc via `MINDS_DOCKER_RUNTIME_DEFAULT=RUNC`, alongside the existing `MNGR__PROVIDERS__DOCKER__DOCKER_RUNTIME=runc`. The Modal snapshot sandbox has no gVisor, and the per-create runtime feature otherwise defaults the Linux create form to runsc and stacks the `docker_runsc` template -- which an mngr provider-config env var cannot override once explicitly stacked -- so the build's workspace creation failed with "unknown or invalid runtime name: runsc". Setting the minds default keeps `docker_runsc` from being stacked in the first place.
