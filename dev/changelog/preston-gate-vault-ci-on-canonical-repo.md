Vault-dependent CI jobs are now gated to the canonical repo (`github.repository == 'imbue-ai/mngr'`). On mirror repos such as imbue-ai/agentic-mngr (used by open-seer fixer agents), the `mngr_ci_gh` Vault role's bound claims reject the JWT login, so every Vault-gated job (test-offload, test-offload-acceptance, build-minds-snapshot, cleanup-modal-environments, vet, TMR, release tests, minds launch-to-msg) was a guaranteed failure. These jobs now skip cleanly on mirrors instead, so mirror PRs get green/skipped checks from the jobs that cannot possibly pass there, while test-docker and check-changelog continue to run everywhere. Behavior on imbue-ai/mngr itself is unchanged.
