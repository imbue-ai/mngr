## Provider specs brought into agreement with the bare-providers merge

Updated the five provider specs under `specs/` after merging `mngr/bare-providers`, so they describe the current provider state. Common to all: the package/class rename (`mngr_vps_docker` -> `mngr_vps`, `VpsDockerProvider` -> `VpsProvider`, `VpsDockerHostRecord` -> `VpsHostRecord`, `VpsDockerHostStore` -> `VpsHostStore`, `VpsDockerError` -> `VpsError`, `MinimalVpsDockerProvider` -> `MinimalVpsProvider`) and the new realizer/isolation architecture (a `HostRealizer` seam with `DockerRealizer` for `isolation=CONTAINER` and `BareRealizer` for `isolation=NONE`, selected by config).

- `specs/provider-shape.md` (prescriptive contract): incorporated the realizer/isolation contract (bare isolation requires a real machine stop/start lifecycle or must reject `isolation=NONE` via `BareIsolationNotSupportedError`; bare reports `supports_snapshots=False`). Reflected landed behavior — AWS/GCP/Azure real VM stop/start (Azure via true deallocate), idle self-stop on all three, the shared S3/Blob state bucket (`BucketHostStateStore`) and offline base classes (`OfflineCapableVpsProvider` / `TagMirrorVpsProvider`) giving uncapped offline host+agent records. Split the anti-patterns into resolved (Azure auto-shutdown billing, `--stop-host` leave-VM-running on GCP/Azure, stopped-host discovery dropping providers) and still-real (SSH `supports_shutdown_hosts` lie, `supports_volumes` True-but-`[]`, snapshot-arg no-op, AWS/GCP default help text, Docker `0.0.0.0:22`, Modal underscore tags).

- `specs/provider-uniformity-review.md` (descriptive current state): re-ranked the findings table (the `--stop-host` leak, Azure auto-shutdown billing, idle self-stop, and stopped-host visibility findings now resolved for the cloud trio, remaining open only for Vultr/OVH); added an isolation-modes/realizer section and the S3/Blob state-bucket discovery story; updated the defaults, discovery, and snapshot matrices.

- `specs/implementing-a-provider.md` (dev guide): added the realizer-seam and isolation-mode guidance; refreshed the stop/start, capability-flag, N-agents, and cost-safety sections for the landed cloud-trio behavior; updated references.

- `specs/provider-release-tests.md` (release-test proposal): flipped the `--stop-host`, idle auto-shutdown, stopped-host-visibility, and offline-mirror `xfail`s to passing for AWS/GCP/Azure (kept for Vultr/OVH); added a Trip 1 parametrization over isolation mode that folds in the already-landed bare release tests and asserts `BareIsolationNotSupportedError` on providers without bare support.

- `specs/cleanup-error-aggregation.md`: cite/naming refresh only (the core model is unchanged) — VPS-family container teardown now lives behind `DockerRealizer.teardown_placement`; left the separate local-docker `DockerHostStore` path untouched.
