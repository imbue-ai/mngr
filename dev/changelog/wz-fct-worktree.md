Added `just fct-worktree [branch] [base]` (backed by `scripts/fct_worktree.sh`): creates an independent forever-claude-template ("fct") checkout at `.external_worktrees/forever-claude-template` in the current mngr checkout, defaulting the fct branch to the current mngr branch (errors if a checkout is already there, or if the branch already exists on fct). It clones fct directly -- a full, self-contained clone that survives deletion of any other clone or cache -- and needs no configuration, so an agent developing fct alongside mngr can run it with zero setup. Set `FCT_DIR` (from a gitignored `apps/minds/.env` or your shell) to a local fct clone to accelerate the clone via `git clone --reference-if-able --dissociate`; it stays a pure speed hint with no lasting dependency.

Removed the hardcoded `~/project/forever-claude-template` guidance for creating that worktree, pointing the `minds-start` recipe, the `minds-dev-workflow` skill, and `CLAUDE.md` at `just fct-worktree` instead. `bake-slice-dev` now resolves its workspace dir from the explicit arg, else `FCT_DIR` (shell or gitignored `apps/minds/.env`), else the `.external_worktrees/forever-claude-template` checkout -- no personal path baked in -- and errors with a pointer to set `FCT_DIR` / run `just fct-worktree` when no checkout is found there. `create-new-mind-repo` no longer requires a local fct clone at all: it clones the template from the remote, so the vestigial `FCT_DIR` / `~/project/forever-claude-template` check was removed.

`.mngr/settings.toml` now COPYs the gitignored `apps/minds/.env` into each mngr agent worktree (via `work_dir_extra_paths`), so an agent inherits the operator's `FCT_DIR` speed hint. Skipped silently when the file is absent.
