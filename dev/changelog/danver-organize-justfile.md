Organized the root justfile's recipes into `[group(...)]` categories along two axes -- project (mngr, minds) by activity (install, dev, test, build, plus a minds-only ops group for pool/slice/deploy operator recipes) -- so `just` with no arguments now lists recipes under headings like `[mngr test]`, `[minds dev]`, and `[minds ops]`, making it easier to find the right command. The previous coarse `[group("minds")]` tags were refined into the specific categories; the grouping itself changed no recipe behavior.

`just minds-install` now also installs the pinned pnpm (from `engines.pnpm` in `apps/minds/package.json`) into the selected Node when it is missing or the wrong version, instead of failing with "pnpm: command not found". npm globals are per-Node-version under nvm, so a freshly installed Node never has pnpm; previously this was a manual step. The check runs from `apps/minds` so pnpm's self-version-switching (via any `packageManager` pin in a package.json above the repo) cannot skew it.
