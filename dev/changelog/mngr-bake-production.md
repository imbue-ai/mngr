Added `just bake-slice-dev` and `just bake-slice-prod` recipes for baking bare-metal slices (lima/QEMU VMs carved on a pre-registered, prepped OVH bare-metal box) into the minds pool.

These complement the existing `bake-pool-host-{dev,prod}` recipes, which only order OVH VPSes: the env-aware `minds pool create` wrapper drives only the `ovh_vps` backend, so the new recipes route to `mngr imbue_cloud admin pool create --backend slice` directly. From the activated minds tier's Vault entries they export `POOL_SSH_PRIVATE_KEY` (used to SSH the box and carve the VM) and, for shared tiers (staging/production), the host_pool DSN (`MINDS_HOST_POOL_DSN`), so an operator gets the same convenience as the OVH recipes. Both require an activated minds env and `vault login` first; `region` is the lease-region label stamped on each row, not the box's raw OVH datacenter code.
