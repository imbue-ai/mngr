`scripts/build-lima-image.sh` now bakes the pre-baked Lima image with a 20GiB disk by default (was 40GiB), matching the `--disk=20` the forever-claude-template lima create template passes. Lima can grow a smaller image to the instance size but fatals on shrink ("disk shrinking is not supported"), so a 40GiB baked image broke every default fast-path create that consumed it. Found by driving the fast-lima create end-to-end against a locally baked and published image.
