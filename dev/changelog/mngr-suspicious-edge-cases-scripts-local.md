Hardened more suspicious edge-case handling across `scripts/` (follow-up to the earlier `scripts/` review):

- `make_cli_docs.py`: `build_alias_doc` now raises instead of printing a warning and returning `None` when an alias command in `ALIAS_COMMANDS` has no registered help metadata. Alias docs are built entirely from `CommandHelpMetadata`, so a missing entry previously produced a silently-absent doc page that `--check` could not catch (it only compares files actually generated). The function now returns a non-optional `tuple[Path, str]`, the dead `output_dir is None` branch became an assertion, and the caller drops its `None` check. Added a happy-path test that the `clone` alias still generates its doc.
- `push_vault_from_file.py`: `_parse_env_dump` now raises on an entry with no `=` instead of silently skipping it. The dump comes from `env -0`, which always emits `KEY=VALUE`, so a separator-less entry signals corrupt output; silently dropping it would mean a vault secret is never pushed.
- `release.py`: tightened the best-effort failure-log `try` in `print_run_failure` to wrap only the `gh run view` call, with a comment explaining the swallow is intentional (we are already on the failure path and will exit non-zero with the run URL).
- Added clarifying comments to `poll_modal_agents.py` (the assumed `"RUNNING"` prior state for never-before-seen finished agents) and `warm_cli_example.py` (the short-FD-set early return and the `isinstance(result, int)` exit-code handling).
