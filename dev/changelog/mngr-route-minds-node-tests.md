Added a dedicated `test-node` CI job (`.github/workflows/ci.yml`) that runs all Python tests marked `@pytest.mark.node` (the new node resource guard) on a lightweight runner provisioning only Python, uv, and Node.js (no pnpm / Electron deps / xvfb / Docker). Excluded the `node` marker from every offload run (`offload-modal.toml`, both the `all` and `flaky` groups) and from the local `just test-unit` / `test-integration` / `test-quick` filter (`justfile`), since neither the shared mngr Modal image nor a typical dev machine has Node. The job runs with an explicit `--timeout 60` so the latchkey Node tests' up-to-15s child-process readiness wait fits within the per-test signal timeout (the repo-root default of 10s is too small).
