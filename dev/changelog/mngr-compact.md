Added `specs/mngr-message-delivery-signals.md`, a problem-statement spec for how `mngr message` decides that a message reached an interactive TUI agent. It catalogs the seven categories of input (prompt, local command, modal local command, session-restarting command, invalid command, queued-while-busy, empty), inventories every signal that Claude Code and mngr emit along with which component produces each one, and maps signals to cases in a matrix measured against a real Claude agent.

The spec records twelve issues found while reproducing the behavior, including two silent GNU-coreutils dependencies that break `mngr message` on stock macOS (`timeout` in the tmux `wait-for` waiter, so the `UserPromptSubmit` signal is fired into the void; and `tac` in the transcript offset reconciliation, whose stderr is painted into the agent's input box). It also documents that `mngr message` can return before Claude's input buffer is clear, so a subsequent message may be concatenated onto residue, and that modal local commands such as `/cost` and `/login` leave the agent wedged for every later send.

Added a top-level `uncertainties.md` recording two places where comments and docstrings assert `mngr message` behavior that does not hold in practice.
