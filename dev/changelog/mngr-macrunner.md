`minds-launch-to-msg` now reports **which self-hosted mac runner** ran each job. The `build` and `launch_to_msg` jobs export a `runner_name` output (from `${{ runner.name }}`), and the Slack status message gains a `build: <name> | verify: <name>` line -- so with more than one mac runner in the pool you can tell which physical machine a run (or a failure) landed on.

New `runner_label` `workflow_dispatch` input pins the self-hosted build + verify jobs to a single runner by label (e.g. `mac-runner-2`); empty keeps the shared `minds-runner` pool. Use it to isolate a run to one machine -- e.g. to test whether a failure is machine-specific. The target runner must carry the given label.
