Added `specs/claude-settings-overrides.md`: the settled design for how mngr injects Claude hooks and `settings_overrides`. mngr writes everything it owns into the per-agent config-dir `settings.json` (leak-safe) and passes no `--settings` of its own, so a user's raw `--settings` passes through and Claude layers it natively (deep-merging dicts and concatenating same-event hooks). This supersedes the interim `--settings`-merge approach. Includes a clearly-documented limitation: in `use_env_config_dir` mode, mngr can only inject hooks via `--settings`, so `settings_overrides` is unapplied and a user `--settings` collides -- an accepted, scoped limitation (that mode is not yet in production). Spec only -- not yet implemented.

Added `specs/config-deep-merge-dict-fields.md` (CONCLUDED, no change needed): considered making the `__extend` operator merge dicts recursively, but concluded mngr's existing one-level `__extend` is sufficient (place `__extend` at the level you want merged) and should stay non-recursive. Kept as a record of the reasoning.
