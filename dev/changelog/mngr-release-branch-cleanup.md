- Rewire CI's release tests off the dead "release" branch and onto `v*` release tags. There is no `release` branch -- releases are cut from `main` as `v*` tags -- so the `test-release` and `test-docker-release` jobs, gated to `refs/heads/release` push, never ran. They are now gated to `v*` tag push (`github.event_name == 'push' && github.ref_type == 'tag'`), so the full release suite (including the real-Docker job) runs when a release tag is pushed. CI now also triggers on `tags: ['v*']`, and the regular test jobs (`test-offload`, `test-offload-acceptance`, `test-docker`, `test-docker-electron`) skip tag builds via `github.ref_type != 'tag'` since release tests are a superset. Also refresh the stale "Release Tests" description in `style_guide.md` and drop the dead `release` branch from the changelog-ratchet PR-branch skip in `test_meta_ratchets.py`.
