Added `specs/host_lock_reconnect_safety.md`, a design doc for keeping a held cooperative host lock correct across SSH reconnects. It documents the silent critical-section violation that can occur when a connection drops mid-operation (the lock releases but per-operation retries silently reconnect and continue), and proposes a fix: a monotonic acquisition counter on the host plus a re-acquire-and-verify step on reconnect, so an uncontended blip transparently recovers while an intervening acquisition raises `LockLostError`. This is a follow-up to the remote host-lock acquisition resilience change (PR #2350).
