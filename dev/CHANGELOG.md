# Changelog - dev

A concise, human-friendly summary of changes for repo-level dev tooling: CI workflows, repo scripts, top-level configuration, build tooling, ratchets, and the changelog tooling itself. Entries are categorized using the [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) categories: Added, Changed, Deprecated, Removed, Fixed, Security.

For the full, unedited changelog entries, see [UNABRIDGED_CHANGELOG.md](UNABRIDGED_CHANGELOG.md).

## [Unreleased]

### Added

- Added: `aws` create-template in `.mngr/settings.toml` for dogfooding the codebase on AWS EC2 -- `mngr create -t aws <name>` builds the dev Dockerfile and runs an agent on EC2. Uses the real-source-tree build shape (context `.`, cloned + overlaid with uncommitted changes) so the inner `docker build` resolves the Dockerfile path relative to the uploaded context. The shared `[providers.aws]` config (region `us-west-2`, `t3.large`, `auto_shutdown_minutes = 120`, `builder = "DEPOT"`) is committed in `.mngr/settings.toml`; per-developer `allowed_ssh_cidrs` lives in `.mngr/settings.local.toml`. `builder = "DEPOT"` requires exporting `DEPOT_TOKEN` and `GH_TOKEN`; the template uses `pass_env__extend` so it doesn't clobber inherited `pass_env` (e.g. `ANTHROPIC_API_KEY`), and the existing `modal` template was switched to `pass_env__extend` for the same reason.
- Added: `specs/agent-plugin-parity/spec.md`, a developer reference mapping every feature the mature `mngr_claude` / `mngr_antigravity` agent plugins implement (lifecycle, subagent-aware idle gating, auth/credential sharing, HOME/config isolation, permissions, trust/onboarding, transcripts, conversation resume, session preservation, deploy contributions, and more), plus a current-state matrix, a recommended bring-up sequence, and a per-CLI investigation checklist. Refreshed with lessons from the pi-coding port (now real near-`antigravity`-parity) and updated for `mngr_opencode` / `mngr_codex` as real, fully-implemented ports rather than `BaseAgent` stubs. Adds dimension F ("Input delivery & submission confirmation"), a "Your lever: shell hooks vs an in-process extension" section, codex's lock-based subagent-aware idle gating, and notes on the "detached work makes agent report WAITING" failure mode.
- Added: Three justfile recipes -- `just release [args...]` wraps `scripts/release.py`; `just changelog-deploy` (re)deploys the nightly consolidation schedule; `just changelog-trigger` runs the consolidation on demand, opening a PR. `scripts/release.py`'s pre-release gate now points users at `just changelog-trigger` instead of printing a long `mngr schedule run ... --disable-plugin ...` one-liner.
- Added: Canonical justfile recipes for pool-host operations -- `just bake-pool-host <attributes-json> <region> [workspace_dir] [count] [extra-flags]`, `just list-pool-hosts`, `just destroy-pool-host <id>`. Thin wrappers around the env-aware `minds pool {create,list,destroy}` CLI.
- Added: `minds-justfile` skill that routes any minds task (app, pool hosts, environments, deployments, tests) through the root justfile and directs adding a recipe when one is missing.
- Added: `imbue.mngr_codex` to the root pytest coverage targets for the new codex plugin.
- Added: `blueprint/workspace-color-picker/plan-workspace-color-picker.md` describing the workspace color-picker feature (12 named palette colors + custom hex; replaces the SHA-derived per-workspace accent; implementation lands in apps/minds).

- Added: Daily TMR cron at 08:00 UTC via a new `TMR (scheduled)` workflow that gates on a prior periodic PR (`tmr-periodic` label, 4-day window) and invokes the main `TMR` workflow via `workflow_call`; manual `workflow_dispatch` runs are unaffected by the gate.
