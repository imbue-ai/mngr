Added the first behavioral unit tests for the LiteLLM proxy app (`app_test.py` + a shared `conftest.py` fixture): they parse the config the proxy actually writes at startup and assert that every model routes to `anthropic/<model_name>` via the `ANTHROPIC_API_KEY` env reference, that the exposed model set is pinned, that `database_url`/`master_key` resolve from their `os.environ/<NAME>` references, and that the config round-trips faithfully through YAML. To let each test write to an isolated temp path, `_write_config_file` now takes the output path as a required argument (the proxy passes its deployed `/tmp/litellm_config.yaml` location explicitly); no runtime behavior change.
