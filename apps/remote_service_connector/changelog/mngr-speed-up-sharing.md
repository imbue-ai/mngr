Added an O(1) `GET /tunnels/by-agent/{agent_id}` endpoint. minds always knows the exact tunnel name it wants (`<username>--<agent-prefix>`), so this resolves the tunnel via Cloudflare's server-side name filter plus a single config fetch (2 Cloudflare calls regardless of how many tunnels the account owns) and returns 404 when no tunnel exists for the agent yet. Previously the only way to find an agent's tunnel was `GET /tunnels`, which enumerated every tunnel under the user prefix and fetched each one's config (O(n) Cloudflare calls), dominating the latency of the sharing flow.
