Renamed the connector's misleading auth names. The "admin" auth was just a normal signed-in user's SuperTokens session, so `AdminAuth` is now `UserAuth`, `AgentAuth` is now `TunnelTokenAuth` (naming the credential it actually validates), `require_admin` is now `require_user_auth`, and endpoint docs/messages say "signed-in user" instead of "admin". No behavior change.

The fixed operator API key is now canonically `MINDS_ADMIN_KEY` (was `MINDS_PAID_ADMIN_KEY`, a stale name from when it only guarded the paid-list CRUD -- it also authenticates `/admin/accounts/*` and `/admin/sweep/*`). The deprecated spelling is still accepted with a warning while Vault entries and operator environments migrate, and `MINDS_ADMIN_KEY` wins when both are set. `require_paid_admin_key` is now `require_admin_key`.

Fixed the stale `test_full_lifecycle` release test: it still authenticated with HTTP Basic auth (`USER_CREDENTIALS`), which the connector stopped accepting when it moved to Bearer-only auth, and predated the quota work. It now stubs the SuperTokens auth path and entitlements store the same way the unit tests do (unique per-run user, no SuperTokens core or Neon DB needed) while every Cloudflare call stays real. Verified green against the staging Cloudflare account.

Renamed the misnamed `username` concept to `user_id_prefix` throughout: the value was never a username but the 16-hex prefix of the SuperTokens user id used to namespace tunnels, leases, and buckets (`UserAuth.user_id_prefix`, `derive_user_id_prefix`, `extract_user_id_prefix_from_tunnel_name`, etc.). Migration 016 renames the `account_entitlements.username_prefix` and `r2_cleanup_grants.username_prefix` DB columns (and the entitlements prefix index) to `user_id_prefix` to match; it applies automatically on the next `minds env deploy`.
