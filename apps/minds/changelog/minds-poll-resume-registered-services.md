Changed: Hardened the snapshot-resume service-registration test (`test_resumed_workspace_registered_expected_services`). It now polls `runtime/applications.toml` (shell-side inside `docker exec`, no Python `time.sleep`) until the expected services appear or a deadline passes, instead of reading the file exactly once -- the app-watcher re-registers services asynchronously after a resume via `scripts/forward_port.py`, so a single read raced a service that registered a moment later and failed intermittently. It also restores coverage that `browser` re-registers: `browser` is now required unless there is positive evidence it was shed under memory pressure (supervisord recorded its program dying on an OOM signal -- `terminated by SIGKILL`/`SIGTERM`, or `exit status 137`/`143` when its `forward_port.py` child was killed and the bash wrapper propagated the status). A bare "never re-registered", with no shed signal, is treated as a real regression and fails. The always-on core set is `system_interface` and `terminal` (`web` was removed from the template).
