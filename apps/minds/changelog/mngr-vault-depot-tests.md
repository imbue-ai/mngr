The e2e workspace runner now streams the Electron renderer's console output, JS errors, and failed requests into the run log, so a renderer-side fault during workspace creation (e.g. the onboarding flow getting stuck) is diagnosable from CI output instead of being invisible behind Electron's main-process-only stderr.

Added a `minds_snapshot_resume` test (`test_fct_image_rebuild_hits_depot_cache`) that rebuilds the forever-claude-template agent image via depot.dev inside a snapshot-resumed sandbox and asserts the heavy layers come back `CACHED`. This proves the depot rebuild path works in the test sandbox and reuses the layer cache populated by the snapshot build stage. The build is cache-only (no image export, no live agent), and the test skips when `DEPOT_TOKEN` is not present.
