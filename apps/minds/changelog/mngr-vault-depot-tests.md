The e2e workspace runner now streams the Electron renderer's console output, JS errors, and failed requests into the run log, so a renderer-side fault during workspace creation (e.g. the onboarding flow getting stuck) is diagnosable from CI output instead of being invisible behind Electron's main-process-only stderr.

Added a `minds_snapshot_resume` test (`test_fct_image_rebuild_hits_depot_cache`) that rebuilds the forever-claude-template agent image via depot.dev inside a snapshot-resumed sandbox and asserts the layers come back `CACHED`, proving the depot rebuild path reuses depot's shared cache. The build is cache-only (no image export, no live agent). The test only runs in depot mode (`MINDS_SNAPSHOT_BUILDER=depot`); the snapshot build now defaults to the faster local docker builder, so the test skips by default.
