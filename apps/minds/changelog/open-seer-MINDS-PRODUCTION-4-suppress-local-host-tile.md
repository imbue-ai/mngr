Fixed the landing page sometimes showing a duplicate row for a workspace that is already listed locally. A workspace recovered or re-identified under a new agent id left behind a synced record whose stored agent id no longer matched local discovery; the landing page reconciled synced records against local discovery only by agent id, so that record rendered a second "remote" tile beside the workspace's real row. Remote tiles are now also matched by the record's stable host id, so a workspace whose host is present in local discovery is never shown a second time as a remote tile.
