Documented the GNU rsync prerequisite for running the minds desktop client from source. `just minds-start` syncs the working tree into the default-workspace-template worktree with `rsync --filter=':- .gitignore'`, a GNU rsync feature that Apple's openrsync (the `/usr/bin/rsync` on recent macOS) doesn't support -- so the sync fails there. The Getting started section of `apps/minds/README.md` now calls out installing GNU rsync (`brew install rsync`, ahead of `/usr/bin` on `PATH`), and `just minds-start` fails fast with the same fix when it detects openrsync.
