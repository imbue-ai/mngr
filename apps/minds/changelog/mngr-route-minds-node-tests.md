Routed `apps/minds/scripts/build_test.py` to a Node-equipped CI job. PR #1772 renamed `apps/minds/todesktop.json` to `apps/minds/todesktop.js`, so the test that evaluates that config (`test_bundled_limactl_is_signed_with_virtualization_entitlement`) now needs a Node runtime on PATH. The test is marked `@pytest.mark.node` (the new node resource guard, defined in `libs/mngr_latchkey`) and runs in the dedicated `test-node` CI job, replacing the previous `skipif(node is None)` guard. A missing Node binary is now a hard failure surfacing a real misconfiguration rather than a silent skip. The Electron e2e test (`test_desktop_client_e2e.py`) also carries `@pytest.mark.node` now, since its Electron launcher invokes node through the resource-guard PATH wrapper (alongside its existing `docker`/`rsync` marks); it still runs only in `test-docker-electron`.
