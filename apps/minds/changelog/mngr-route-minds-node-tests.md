`apps/minds/scripts/build_test.py` no longer silently skips in CI. PR #1772 renamed `apps/minds/todesktop.json` to `apps/minds/todesktop.js`, so the test that evaluates it (`test_bundled_limactl_is_signed_with_virtualization_entitlement`) needs a Node runtime. It now carries `@pytest.mark.node` (the node resource guard) instead of `skipif(node is None)`, and runs on offload now that Node is installed in the shared mngr image -- a missing Node binary is a hard failure rather than a silent skip. The Electron e2e test (`test_desktop_client_e2e.py`) also carries `@pytest.mark.node` now, since its Electron launcher invokes node through the resource-guard PATH wrapper (alongside its existing `docker`/`rsync` marks).
