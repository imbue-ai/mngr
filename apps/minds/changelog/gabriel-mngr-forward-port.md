`minds run` no longer dictates the `mngr forward` plugin's port. The `--mngr-forward-port` flag and the `MINDS_MNGR_FORWARD_PORT` environment variable are removed: the plugin now picks its own port (its default, or an OS-assigned fallback when the default is taken) and reports it back via its `listening` envelope. `minds run` blocks briefly at startup until that envelope arrives, then uses the reported port for everything downstream; if the plugin fails to report a port within 30s, startup aborts with a clear error.
