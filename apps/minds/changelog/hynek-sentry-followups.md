Hardened the Sentry `before_send` failure path: when a `before_send` callback raises, the failure is now both recorded in the local app log and reported to Sentry via the dedicated minimal-event helper, instead of a single loguru error that risked recursing back into the Sentry event pipeline. A new loguru filter on the Sentry event handler drops records explicitly marked to skip Sentry, so the local log line cannot trigger another Sentry event.

Made the `log_error_inside_sentry` helper non-reentrant. Reporting an error goes through `capture_event`, which re-runs the whole `before_send` chain; if the failure being reported originates in `before_send` itself, that previously recursed until the stack was exhausted. The helper now drops nested calls so reporting is attempted at most once.
