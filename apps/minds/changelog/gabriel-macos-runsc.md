On macOS, the desktop app now forces the local-docker provider's container runtime back to the Docker default (`runc`) when spawning the backend. The bundled template configures `docker_runtime = "runsc"` to harden agents with gVisor, but runsc is Linux-only and is not present in Docker Desktop's VM on macOS, so creating a local agent previously failed with "unknown or invalid runtime name: runsc". The override is skipped if you have already set `MNGR__PROVIDERS__DOCKER__DOCKER_RUNTIME` yourself.
