Release runbook (`apps/minds/docs/release.md`): a pure `vendor/mngr` refresh on default-workspace-template no longer opens a PR. It lands directly on `main`, where `ci.yml`'s `test` job runs on the push as a post-merge backstop, while `minds-launch-to-msg.yml` remains the end-to-end pre-tag gate. A PR is opened only when a `system_interface` consumer fix rides along with the vendor bump (that code still needs review + pre-merge CI).
