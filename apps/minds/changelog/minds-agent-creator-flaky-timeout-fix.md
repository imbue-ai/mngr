Stabilized two load-sensitive desktop-client agent-creation tests (`test_start_creation_api_key_ai_does_not_mint_litellm_key` and `test_start_creation_subscription_ai_does_not_mint_litellm_key`). Their internal `_wait_until_finished` completion-poll deadline was widened from 10s to 20s -- matching the existing imbue-cloud sibling -- because the per-test setup (fresh `ConcurrencyGroup`s plus a recording HTTP-server fixture) can exceed 10s under CI load. The subscription variant is also marked `@pytest.mark.flaky` so offload retries a contended run instead of failing the suite. Test-only change; no runtime behavior is affected.
