The Workspace Settings "Backups" section was reorganized for clarity. A new "Recent backups" table shows the newest restic snapshots as rows -- each with its time (relative, with the exact local time on hover) and a per-snapshot Download action that saves that point-in-time snapshot as a zip. The newest row is tinted and marked with a green "Latest" badge (the same green pill as the landing page's "Backed up" badge). Rows are ordered newest to oldest. A "View all N backups" footer appears only when there are more snapshots than the table shows and opens a new full-history page. Not-configured, no-backups-yet, load-error, and backing-up states each get their own clear message. A "Restore" control is shown per snapshot but disabled for now (in-place restore is coming soon). The landing page's per-workspace "download" link was removed for local workspaces in favor of the settings-page table; remote-device rows keep it, since they have no settings page and it is the only way to retrieve their backups.

The backup controls below the table were rewritten in plain language, grouped under two subheadings with one-line descriptions: "Where your backups are stored" (formerly "Configure backups...", now "Change storage location", with friendlier provider options and a "Save" button) and "Fix backup problems" (formerly "Update backup service", now "Update backup software", explaining that it reinstalls/restarts the backup software and never touches files or existing backups). Health-check problem messages now say what is wrong and which button fixes it, and the technical installed/minimum/target version triple was replaced by a single sentence that mentions an available update only when there is one. "Backup service verification" is now a third subheading in the same style, with a one-line description of what the checks do, placed between the storage setting and "Fix backup problems" (check first, then fix).

Snapshot sizes in the backup tables are now read from the correct restic snapshot-summary field (`total_bytes_processed`; the previously read `total_size` does not exist in `restic snapshots --json` output), so sizes display for snapshots created by restic 0.17+. Snapshots created by older restic versions record no summary and still show "--".

A new backup-history page at `/workspace/<id>/backups` shows every snapshot of a workspace, newest first, 20 per page with Newer/Older pagination and a "Showing X-Y of N backups" range indicator, so a long history is never loaded all at once. Each row shows the exact local time, relative age, size, a per-snapshot Download action, and the same disabled Restore placeholder as the settings table; the repository's newest snapshot carries the green "Latest" badge. It is backed by a new paginated API route, `GET /api/v1/workspaces/<id>/backups/snapshots?offset=&limit=` (limit defaults to 20, capped at 100), which returns one page of snapshots plus the total count and -- unlike the full `/backups` route -- never execs into the workspace, so paging is a single restic listing per request and works for offline workspaces.
