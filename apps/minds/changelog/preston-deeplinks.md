The desktop app now handles `minds://` deeplinks. `minds://create?git_url=<repo>&branch=<ref>` opens or focuses the app and lands on the create-workspace page with the repository (and optionally the branch/tag/commit) pre-filled under advanced settings -- used by Inspiration links. Any other `minds://` URL, including a bare `minds://`, simply opens or focuses the app; the login-with-Imbue web page uses this to hand off back to the app.

macOS `open-url` events and Windows/Linux argv invocations (cold start and second instance) all route through a single `handleDeeplink` router. Deeplinks arriving while the app is still starting are queued and applied once startup finishes; a genuine first run keeps the onboarding flow and drops the link. Deeplinks never force a sign-in -- the create page's existing remote-vs-local flow prompts only when needed.
