The desktop app now handles `minds://` deeplinks. `minds://create?git_url=<repo>&branch=<ref>` opens or focuses the app and lands on the create-workspace page with the repository (and optionally the branch/tag/commit) pre-filled under advanced settings -- used by Inspiration links. Any other `minds://` URL, including a bare `minds://`, simply opens or focuses the app. Completing a browser OAuth sign-in now uses this: the sign-in success page redirects to `minds://` (via the plugin's new `--success-redirect-url` option), so the app regains focus instead of leaving you on a "close this tab" page.

macOS `open-url` events and Windows/Linux argv invocations (cold start and second instance) all route through a single `handleDeeplink` router. Deeplinks arriving while the app is still starting are queued and applied once startup finishes, including on a first run -- an explicit deeplink wins over the welcome screen. Deeplinks never force a sign-in -- the create page's existing remote-vs-local flow prompts only when needed.
