Make the dev-tools shortcut (Cmd+Alt+I on macOS, Ctrl+Shift+C elsewhere) and the View > Toggle Developer Tools menu item open devtools for the focused view. When the chrome (title bar) view or a modal surface has focus, devtools open in a detached window (those views are too small to host a docked panel); otherwise behavior is unchanged -- the content view's docked devtools toggle as before.

Generalize the Card primitive: ``Card.jinja`` now takes ``layout`` (``block`` / ``row`` / ``row-spread``), ``padding`` (``default`` / ``tight``), ``interactive``, ``tag`` (``div`` / ``a`` / ``button``), and ``href`` props, with arbitrary HTML attributes passed through via JinjaX's ``attrs``. The visual shell (white background, zinc-200 border, rounded corners, subtle shadow) is now a single CSS class ``.minds-card`` in ``tokens.css`` so JS-rendered cards (the providers panel on the Landing page) reference the same surface. The Landing page's project rows now use ``<Card tag="a" layout="row" interactive>`` (so middle-click opens a workspace in a new window, as users expect from anchors), and the provider rows reference ``.minds-card`` instead of inlining the same Tailwind utility string. ``CardRow.jinja`` was a thin alias for ``<Card layout="row-spread">``; its two call sites (Accounts, WorkspaceSettings) now invoke Card directly.
