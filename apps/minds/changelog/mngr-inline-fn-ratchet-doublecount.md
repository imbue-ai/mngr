Removed the one inline helper the backup-detail endpoint (`/api/v1/workspaces/<id>/backups`) still defined, hoisting it to a module-level `_append_backup_service_check` that the concurrency-group thread calls via `start_new_thread(args=...)`. Behavior is unchanged; the check still runs on its own thread and hands its result back through the results list. This keeps the inline-function ratchet honest rather than rewriting the closure as a `functools.partial`, which would only move the violation to the partial ratchet.

The inline-function ratchet count for this project drops from 9 to 6: two of the previous nine were the same doubly-nested function counted twice (a shared-checker bug fixed in this change), and one was the backup-detail helper above.
