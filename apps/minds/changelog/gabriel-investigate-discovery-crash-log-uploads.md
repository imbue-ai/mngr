Fixed a spurious "Minds has disconnected from your workspaces / Restart Minds" full-app takeover that could flash during a normal app shutdown or restart. The `mngr forward` proxy runs in the app's process group, so a shutdown signal reaches it too and it can exit before the ordered teardown formally stops it; the discovery-health watchdog was misreading that expected exit as an unexpected consumer death and latching the terminal takeover screen on the way down. The shutdown signal handler now marks the consumer's exit as expected first, so a genuine pipeline failure still surfaces the takeover but a routine restart no longer does.

Fixed diagnostic log uploads (the gzipped `minds.log` / `minds-events.jsonl` and traceback attachments sent with a bug report or automatic error report) being abandoned mid-upload when the app exits. The bounded flush that waits for those background S3 uploads to finish was the very last step of shutdown, after the concurrency-group drain -- which can block until its timeout and get the process force-killed before the flush ever runs, so in-flight uploads never landed. The flush now runs before that drain, so pending diagnostic uploads reliably complete on a normal (signal-driven) shutdown.
