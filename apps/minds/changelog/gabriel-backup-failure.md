Fixed dev-mode IMBUE_CLOUD backup setup, which was failing with "restic binary not found" and flooding the logs:

- The dev-mode Electron backend launcher (`electron/backend.js`) was not forwarding `MINDS_RESTIC_BINARY` to the Python backend -- only the packaged-mode branch set it. So even though `pnpm start`'s prestart hook downloads `resources/restic/restic`, the backend fell back to a bare `restic` PATH lookup, didn't find it, and every backup setup raised `ResticNotInstalledError`. The dev branch now sets `MINDS_RESTIC_BINARY` exactly as packaged mode does.

- A missing restic binary is a permanent, machine-level condition, but `AgentCreator._provision_backups` was retrying it for the full 300s backup-setup budget per agent -- so a workspace session with several agents produced hundreds of repeated "Configuring IMBUE_CLOUD backups ... [failed]" log lines. `ResticNotInstalledError` is now excluded from the retry: backup setup fails fast on the first attempt and surfaces a single notification instead of spinning for the whole budget.

- Backup-setup failures are now surfaced persistently in the app, not just in the logs and a transient OS notification. When setup fails, minds records a per-workspace failure marker, and the landing page shows a distinct red "Backup setup failed" badge (with the error as a tooltip) for that workspace -- previously a broken backup was indistinguishable from a workspace the user deliberately created without backups (both showed a muted "No backups"). The marker is cleared automatically once backups are configured successfully.
