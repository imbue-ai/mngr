Stabilized the minds agent-creator litellm-key tests under offload CI contention: the `_wait_until_finished` poll deadline was raised from 10s to 30s. It only caps a poll that returns the instant creation reaches a terminal state, so passing tests are unaffected; it just stops heavy test setup under CI load from tripping a spurious timeout (matching the `@pytest.mark.timeout(30)` already on those tests).

Also stabilized the `MngrCaller` forkserver tests (`mngr_caller_test.py`): each call pays a cold forkserver boot + `imbue.mngr.main` preload (the leak-checking fixture tears the forkserver down after every test), which could exceed the default 10s on a loaded CI runner. Their per-test timeout is raised to 30s and the call's own internal timeout tightened from 120s to 20s (it bounds only the fast in-child CLI run). Test-only.
