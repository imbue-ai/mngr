Fix the requests panel X button being unclickable when the panel auto-opens at startup. The modal opened before chrome.js had registered its `onModalStateChanged` listener, so the initial `modal-state-changed: { open: true }` IPC was dropped, the `modal-open` body class never got applied, and the titlebar's drag region intercepted the click. The chrome view's `did-finish-load` handler now re-sends the modal-open state if the modal is already visible.
