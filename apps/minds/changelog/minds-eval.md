`minds run` now honors a `MINDS_FORWARD_AGENT_INCLUDE` environment variable: when set, its value is used as the `--agent-include` CEL filter for the built-in `mngr forward` subprocess (replacing the default `has(agent.labels.is_primary)`). Unset, behavior is unchanged, so normal minds is unaffected. This lets a host that manages a large shared Modal env tell minds' forward to proxy nothing (via a never-match filter) and forward workspaces on demand instead, avoiding the memory blow-up from eagerly proxying every workspace at once.
