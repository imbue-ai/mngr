Foundation for the new Minds design system.

- `static/tokens.css` is now layered: primitives (Figma workspace palette, brand seeds, typography ramp, spacing/radius/shadow scales) declared on `:root` alongside light-theme semantic defaults, with `:root[data-theme="dark"]` overriding the semantic tokens for dark surfaces. Semantic tokens cover `surface-base/elevated/overlay/sunken`, `text-primary/secondary/muted/on-accent`, `border-default/strong/focus`, `accent/accent-hover/accent-soft`, and per-state `bg/fg/border` for info / warn / success / error.
- New `static/tailwind_config.js` is loaded before the Play CDN script and extends Tailwind with the semantic colors (`bg-surface`, `text-ink`, `border-line`, `bg-accent`), the typography ramp (`text-display` through `text-label`), additive spacing (`s-1`..`s-12`) and radius (`rounded-ds-sm` through `rounded-ds-xl`, `rounded-pill`), plus `darkMode: ['selector', '[data-theme="dark"]']` so any `dark:` variants co-exist with the semantic approach. Defaults (zinc/blue/red, default sizes, default radii) remain available so unmigrated production pages render identically.
- New `static/theme.js` reads `--workspace-surface` off `<body>`, computes the surface's relative luminance (OKLCH `L%` or sRGB), and writes `data-theme="dark"|"light"` on `<html>` so the semantic tokens resolve to the right palette automatically. `<html>` ships with `data-theme="light"` by default, and the bare `:root` block declares the light-theme semantic values directly, so unmigrated pages (whose bodies still use `bg-zinc-50 text-zinc-900`) get matching light-theme macro children even before `theme.js` runs. `window.mindsTheme.refresh()` lets the chrome re-pick the theme after a workspace switch.
- `_macros.html` (`btn_button`/`btn_link`/`btn_submit`, `text_input`, `notice`, `card`/`card_row`, `spinner`, `opt`) now consumes the semantic tokens. The macro signatures are unchanged, so every existing call site (`landing.html`, `create.html`, `creating.html`, `welcome.html`, ...) renders identically on its current light surface and will adapt automatically when a workspace surface flips the theme.
- The `/_dev/styleguide` page is rebuilt: a surface-color picker at the top re-tints the whole page and flips component themes, followed by full token inventories (palette, brand, typography ramp, spacing, radii, shadows, semantic and state colors), live component demos through the macros, and the existing chrome patterns (titlebar, window controls, sidebar items, accent spine).
- The drift-guard ratchet (`templates_test.py::test_dev_styleguide_token_swatches_enumerate_root_declarations`) is scoped to the bare `:root` block (which now declares every primitive plus the light-theme semantic defaults). Three new tests assert the surface picker is present, the typography ramp surfaces size/line/weight tokens for every level, and every spacing/radius primitive has a `data-token` swatch.
