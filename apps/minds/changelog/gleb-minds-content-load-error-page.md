Fixed: a workspace that fails to load at the network layer (its host unreachable or asleep, or a stale/dead forward port) now shows a recoverable error page in the content view instead of a silent blank page. The page offers a Retry -- which re-opens the workspace against the current connection rather than the failed URL, so a stale port self-heals -- and a Report a bug button. Benign cases (superseded loads, embedded-iframe failures, in-flight shutdown) are filtered out so the error page only appears on a genuine failure.
