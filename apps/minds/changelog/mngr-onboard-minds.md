Fixed the macOS dev launch (`just minds-start`) producing a window with no Dock icon that took no keyboard input.

`pnpm start` ran `electron .`, which executes the Electron binary directly. On recent macOS that does not register the process as a foreground GUI app -- the window draws but gets no Dock tile and can never become the key window (so it receives no keyboard input). The launch now goes through a small supervisor (`scripts/launch_electron.js`) that, on macOS, starts the same `.app` bundle via LaunchServices (`open`), forwarding the activated environment with `--env` (which `open` otherwise drops). The app now comes up as a normal foreground app. The supervisor forwards SIGTERM/SIGINT to the (LaunchServices-detached) app so Ctrl-C / `just minds-stop` still stop it cleanly, and stays alive for the app's lifetime so the CSS watcher running beside it is unaffected. Non-macOS platforms keep launching `electron .` directly.
