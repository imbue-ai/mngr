The `launch-to-msg` end-to-end test now scrolls the chat transcript to the live tail before checking for the agent's reply. The chat virtualizes off-screen rows, so a reply that arrived while the view was not pinned to the bottom could be present in the conversation but absent from the rendered DOM the test reads; scrolling to the tail (as a user reading the latest message would) makes the check see it. This removes an intermittent false failure of the scheduled mac-runner run.
