Fixed the ToDesktop build (and the scheduled launch-to-first-message health check) failing for every commit after the Tailwind v4 migration. ToDesktop's cloud builder installs the app with `pnpm recursive install --prod`, which omits devDependencies but still runs lifecycle scripts; the `postinstall` hook compiled CSS via the `tailwindcss` CLI (a devDependency), so the install aborted with "command not found" before any platform could be packaged. The hook now no-ops when the Tailwind CLI is absent -- the packaged app already ships the compiled `app.min.css` inside the minds wheel, so the cloud install never needs to compile CSS. A normal dev `pnpm install` still pre-builds the stylesheet.
