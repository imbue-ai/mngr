Fixed: quitting the app while a workspace window showed the recovery page no longer reopens to a blank/broken page after restart. The recovery page's URL embeds an absolute link back to the workspace that includes the run's connection port, which changes on every launch; that URL was being saved verbatim into the restored-windows state, so on the next launch it pointed at a dead port. Recovery-page windows are now saved as the workspace itself, so restore reconnects to the live workspace (and, if it is still unhealthy, the normal recovery flow re-appears with a fresh link). Windows already saved in the old broken form self-heal on their next restore.
