Hardened several over-defensive edge cases in the desktop client so that malformed input fails loudly instead of being silently papered over:

- The `mngr imbue_cloud` list wrappers (`auth list`, `hosts list`, `keys litellm list`, `tunnels list`, `tunnels services list`) now raise `ImbueCloudCliError` when the plugin exits 0 but emits a non-array body, instead of silently returning an empty list (which had turned a contract violation into a misleading "zero items", e.g. "not shared" / "nobody signed in" / "no hosts"). Removed a speculative dict-unwrap shim in `hosts list` for a wrapped shape that does not exist.
- `MindsConfig.get_auto_open_requests_panel` now raises `MindsConfigError` when `auto_open_requests_panel` is present in `config.toml` but is not a boolean (e.g. a hand-edit to `"yes"`), rather than silently coercing it to the default. An absent key still uses the default.
- The create-agent form now rejects an invalid enum value (`launch_mode`, `ai_provider`, `backup_provider`, `backup_encryption_method`) by re-rendering with an inline error, matching how the text fields and the JSON create-agent API already behave. Previously it silently substituted a default, so a tampered request or front-end bug could create a workspace with settings the user did not choose.
- The notify endpoint validates `urgency` with an explicit `isinstance` string check (like the sibling `message`/`title` fields) instead of relying on catching `AttributeError`.
- The toast urgency-color lookup now uses a direct subscript on its exhaustive enum map, so a future urgency level added without extending the map fails loudly rather than rendering with a stale fallback color.

Also added clarifying comments (no behavior change) explaining why a few correct-but-suspicious-looking handlers are intentional: the recovery-probe port-`0` placeholder is display-only, `_CreateEventCapture` intentionally skips unparseable `{`-prefixed lines on `mngr create`'s mixed human/JSON stdout stream, and the background-thread notification handlers must catch broad exception types (tkinter `RuntimeError`, `ConcurrencyGroup` `ExceptionGroup`).
