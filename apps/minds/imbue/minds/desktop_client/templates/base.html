The build scans both the Solid components and the Jinja templates for class usage, so the emitted CSS covers every page until the migration removes the remaining Jinja templates. The legacy `tokens.css` content (workspace-accent variables, spinner keyframes, .opt onboarding cards) is now part of `globals.css`, which Vite emits as `assets/globals.css` (Vite names CSS bundles after the imported `.css` source, not the JS entry). Blocks: title -- contents. Defaults to "Minds". head -- extra <head> content (page-specific <link>/<meta>). body_class -- classes on <body>. Default is the light page surface. body_style -- inline style on <body>. Used to set --workspace-accent on in-workspace pages so the top stripe picks up the deterministic per-project color. content -- the page body. scripts -- <script> tags emitted just before </body>.
