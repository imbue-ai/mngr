{# Base layout: every page extends this. Pulls in the Tailwind Play CDN JS (downloaded once into /_static/tailwind.js and served locally, so this is offline-safe after the first fetch) plus the hand-written tokens.css. The Play CDN script generates utility classes at runtime, so there is no "rebuild when you add a class" step. Load order is load-bearing: 1. tailwind_config.js -- sets window.tailwind.config (semantic colors, typography, spacing/radius, darkMode selector) BEFORE the JIT engine sees the page. 2. tailwind.js -- the Play CDN runtime. 3. tokens.css -- CSS variables (primitives + theme-scoped semantic tokens) referenced by the Tailwind config and by per-workspace rules. 4. theme.js (deferred) -- reads --workspace-surface and sets so semantic tokens resolve to the right palette. ``data-theme="light"`` is the default in markup. The light-theme values are declared directly on the bare ``:root`` block in tokens.css, so any page that has not yet been migrated -- whose body still uses raw zinc utilities like ``bg-zinc-50 text-zinc-900`` -- renders semantic-token macro children in light-theme colors that match the surrounding zinc classes. theme.js refines the attribute on workspace-scoped pages once a --workspace-surface is set; the data-theme reflow is paint-only. Blocks: title -- contents. Defaults to "Minds". head -- extra <head> content (page-specific <link>/<meta>). body_class -- classes on <body>. Default is the light page surface. body_style -- inline style on <body>. Used to set --workspace-accent on in-workspace pages so the top stripe picks up the deterministic per-project color. content -- the page body. scripts -- <script> tags emitted just before </body>. Prefer <script src="/_static/foo.js"> over inline JS. #} <!DOCTYPE html> <html lang="en" data-theme="light"> <head> <meta charset="UTF-8"> <title>{% block title %}Minds{% endblock %} {% block head %}{% endblock %} {% block content %}{% endblock %} {% block scripts %}{% endblock %}
