Fix `just minds-start`, `apps/minds/scripts/propagate_changes`, and `mngr imbue_cloud admin pool create --mngr-source` to explicitly exclude `.external_worktrees` from the rsync that populates the FCT template's `vendor/mngr/`. The project's `.gitignore` lists this directory as `**/.external_worktrees/`, but rsync's `**/` filter requires at least one preceding path component, so the top-level `.external_worktrees/` was not matched by the `--filter=:- .gitignore` filter. Without this exclude, the rsync recursed into the destination (the FCT worktree lives at `.external_worktrees/forever-claude-template/vendor/mngr`) and pulled unrelated external worktrees into `vendor/mngr/`.
