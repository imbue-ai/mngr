Two changes to flaky tests in `libs/mngr/imbue/mngr/cli/test_destroy.py`:

- `test_destroy_via_stdin`: shrink from two agents to one. The test's purpose is verifying that names piped on stdin reach the `-` argument of `destroy`; one agent is sufficient for that. Multi-agent destroy is already covered by `test_destroy_multiple_agents`. Local benchmark (N=15 interleaved pairs on the same machine): main mean 2.35s -> 1.93s, paired delta +0.42s, current faster in 15/15 pairs. `@pytest.mark.flaky` removed.
- `test_destroy_multiple_agents`: add `@pytest.mark.timeout(60)`. Multi-agent is the point of this test, so it cannot be shrunk like the sibling. A `poll_interval` bump was measured locally and showed no effect (main mean 2.33s, with-bump mean 2.33s, paired delta +0.00s, current faster in 8/15 pairs -- the polled condition is true on the first iteration so interval doesn't matter). The only reliable lever is a per-test timeout, matching the precedent on `test_destroy_transfer_none_keeps_shared_worktree`.
