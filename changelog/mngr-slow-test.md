Two changes to flaky tests in `libs/mngr/imbue/mngr/cli/test_destroy.py`:

- `test_destroy_via_stdin`: shrink from two agents to one. The test's purpose is verifying that names piped on stdin reach the `-` argument of `destroy`; one agent is sufficient. Multi-agent destroy is already covered by `test_destroy_multiple_agents`. With half the setup/teardown work and a slower `poll_interval`, the test fits under the global 10s pytest-timeout and no longer needs `@pytest.mark.flaky`.
- `test_destroy_multiple_agents`: pass `poll_interval=1.0` to the `wait_for(tmux_session_exists, ...)` calls. Each poll iteration spawns a `tmux` subprocess and that overhead dominates the test's wall-clock under contended CI; raising the interval from the default 0.1s cuts the subprocess call count by ~10x. `@pytest.mark.flaky` is kept as a safety net for outlier load.
