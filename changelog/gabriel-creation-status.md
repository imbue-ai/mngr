- The "Creating your project" page now updates its spinner caption as the setup progresses ("Starting...", "Cloning repository...", "Checking out branch...", "Provisioning AI access...", "Creating workspace...", "Waiting for workspace to be ready..."), instead of staying on "Cloning repository..." through the whole flow. Phase state is now carried on the existing ``AgentCreationStatus`` enum as the single source of truth -- the spinner caption is resolved from that enum value by the SSE stream, which polls the creation status on each loop iteration. The ``/api/create-agent/{id}/status`` JSON API now returns the new enum values (``INITIALIZING``, ``CLONING_REPO``, ``CHECKING_OUT_BRANCH``, ``PROVISIONING_AI``, ``CREATING_WORKSPACE``, ``WAITING_FOR_READY``, ``DONE``, ``FAILED``) instead of the previous ``CLONING`` / ``CREATING``.
- The create-project form now gives the desktop client's own ``ANTHROPIC_API_KEY`` / ``ANTHROPIC_BASE_URL`` env vars (sourced from your shell) an explicit, per-create UI rather than silently forwarding them to the agent. Picking "Subscription" auth scrubs both ambient values unconditionally -- choosing OAuth means you get OAuth, no surprise key from your shell silently overriding the login flow. Picking "API key" auth surfaces an opt-in checkbox per detected env var: "Use ``ANTHROPIC_API_KEY`` from your environment", same for ``ANTHROPIC_BASE_URL`` (the value is shown inline so you can sanity-check the endpoint before opting in -- the API key value is never displayed). Ticking the checkbox disables the matching text input, so it's unambiguous which value will be used. The form also gains an "Anthropic base URL" text input next to the API key input for API_KEY auth, so a custom endpoint can be set without exporting an env var. The JSON ``/api/create-agent`` endpoint accepts matching ``anthropic_base_url``, ``use_env_anthropic_api_key``, and ``use_env_anthropic_base_url`` fields.
