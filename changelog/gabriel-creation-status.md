- The "Creating your project" page now updates its spinner caption as the setup progresses ("Starting...", "Cloning repository...", "Checking out branch...", "Provisioning AI access...", "Creating workspace...", "Waiting for workspace to be ready..."), instead of staying on "Cloning repository..." through the whole flow. Phase state is now carried on the existing ``AgentCreationStatus`` enum as the single source of truth -- the spinner caption is resolved from that enum value by the SSE stream, which polls the creation status on each loop iteration. The ``/api/create-agent/{id}/status`` JSON API now returns the new enum values (``INITIALIZING``, ``CLONING_REPO``, ``CHECKING_OUT_BRANCH``, ``PROVISIONING_AI``, ``CREATING_WORKSPACE``, ``WAITING_FOR_READY``, ``DONE``, ``FAILED``) instead of the previous ``CLONING`` / ``CREATING``.
- The create-project form now detects whether the desktop client's own environment has ``ANTHROPIC_API_KEY`` or ``ANTHROPIC_BASE_URL`` set, and surfaces an inline notice (with an opt-in checkbox) under the AI-provider selector for each one that's present. The default state is "ignore" for SUBSCRIPTION auth (so an ambient shell key no longer silently overrides the OAuth login intent), and "use" for API_KEY auth when the form's API-key field is empty (so the env value can populate the agent with one click). The base-URL value is shown inline so you can sanity-check the endpoint before opting in; the API key value is never displayed. The form also gains an "Anthropic base URL" text input next to the API key input for API_KEY auth, so a custom endpoint can be set without exporting an env var. The JSON ``/api/create-agent`` endpoint accepts matching ``anthropic_base_url``, ``use_env_anthropic_api_key``, and ``use_env_anthropic_base_url`` fields.
