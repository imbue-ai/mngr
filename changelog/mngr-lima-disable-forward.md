Fix Lima provider to actually disable guest -> host port forwarding. Lima appends one internal fallback rule that auto-forwards any TCP/UDP guest socket on `guestIP 127.0.0.1` (which also matches `0.0.0.0`/`::`/`::1` bind addresses) to host `127.0.0.1`, and the previous empty `portForwards: []` did not override it. The provider now emits Lima's documented single catchall ignore rule (`guestIP 0.0.0.0`, `proto any`, full port range, `ignore: true`) so the fallback never fires. `merge_lima_yaml` also strips any user-supplied `portForwards` via a new `_LOCKED_KEYS` set so a custom `--file` YAML cannot re-enable forwarding. All Lima VM traffic now flows through SSH, eliminating cross-VM port collisions like the `localhost:8082` clash between coexisting agent hosts.
