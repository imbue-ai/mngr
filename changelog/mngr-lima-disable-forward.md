Fix Lima provider to actually disable guest -> host port forwarding. Lima auto-appends two default rules forwarding any guest port bound on 127.0.0.1 or 0.0.0.0 to host 127.0.0.1, and the previous empty `portForwards: []` did not override them. The provider now emits explicit `ignore: true` rules that catch the full port range on both interfaces. `merge_lima_yaml` also strips any user-supplied `portForwards` so a custom `--file` YAML cannot re-enable forwarding. All Lima VM traffic now flows through SSH, eliminating cross-VM port collisions like the `localhost:8082` clash between coexisting agent hosts.
