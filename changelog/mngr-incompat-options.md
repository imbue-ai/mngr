- `mngr create` now rejects `--reuse` combined with `--new-host` (and by extension `--reuse --update --new-host`) with a clear error. The two flags contradict each other by construction: `--reuse` looks up an existing agent on an existing host, while `--new-host` always provisions a fresh host (which has no agents to reuse). The combination used to be silently accepted.
- The minds desktop client no longer passes `--reuse`/`--update` when creating workspace agents. Every minds launch mode (LOCAL, LIMA, CLOUD, IMBUE_CLOUD) passes `--new-host`, so `--reuse` was a no-op or actively wrong; the `IMBUE_CLOUD` baked `system-services` agent collision is handled by mngr's duplicate-name exemption for `host.pre_baked_agent_id`.
