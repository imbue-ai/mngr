- Added: a wedged minds workspace server can now be recovered without restarting the desktop client. The landing page surfaces a per-mind health indicator, and clicking a stuck mind (or its inline restart button) restarts the workspace server in place. Inside an open workspace, an overlay appears when the server stops responding and offers the same restart action; the sidebar also dims wedged or restarting minds.
- Added: new desktop-client endpoints `POST /api/agents/{id}/restart-workspace-server` (kills the bootstrap-managed `svc-system_interface` tmux window and touches `services.toml` so bootstrap re-spawns it) and `GET /api/agents/{id}/health` (level-triggered probe of the workspace server). Health transitions are pushed to clients over the existing `/_chrome/events` SSE stream as `workspace_server_status` events.
- Fixed: main-frame HTML navigations to a wedged workspace now return a recovery page with a Restart button instead of a blank "disconnected" body.
