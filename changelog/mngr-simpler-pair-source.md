- Changed: `mngr pair` source flags simplified to a single `HostedLocation`. The positional `SOURCE` and `--source` remain (both accept `AGENT[@HOST[.PROVIDER]][:PATH]`); the redundant `--source-agent`, `--source-host`, and `--source-path` are removed. To pin an agent to a specific host, use `AGENT@HOST` syntax (e.g. `mngr pair my-agent@localhost`).
- Added: `mngr pair @HOST:PATH` is now a supported source. The host is resolved directly without going through the interactive agent picker, and PATH is used as-is (no `agent.work_dir` to relativize against, so PATH must be set when no agent is given).
- Fixed: `mngr pair AGENT:subpath`'s pre-sync now operates on the specified subpath. Previously the divergence detection ran on `subpath` but the actual git push/pull still ran against `agent.work_dir`, so subpaths under different branches than the agent's work-dir root would silently sync the wrong tree. The fix threads `agent_path` through `sync_git_state` -> `pull_git`/`push_git` -> `sync_git` (the underlying git sync API now takes `remote_path: Path` instead of deriving it from `agent.work_dir`). Behavior is unchanged for `mngr pair AGENT` (no subpath) and for `mngr push`/`mngr pull` (which always pass `agent.work_dir`).
