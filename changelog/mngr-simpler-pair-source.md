- Changed: `mngr pair`, `mngr push`, and `mngr pull` source/target flags simplified to a single `HostedLocation`. The positional `SOURCE`/`TARGET` and `--source`/`--target` remain (both accept `AGENT[@HOST[.PROVIDER]][:PATH]`); the redundant `--source-agent`, `--source-host`, `--source-path` (and the `--target-*` equivalents on `push` and `pull`) are removed. To pin an agent to a specific host, use `AGENT@HOST` syntax (e.g. `mngr pair my-agent@localhost`).
- Added: `@HOST:PATH` is now a supported source/target across `mngr pair`, `mngr push`, and `mngr pull`. The host is resolved directly without going through the interactive agent picker, and PATH is used as-is (no `agent.work_dir` to relativize against, so PATH must be set when no agent is given). `--stop` is incompatible with host-only sources/targets and reports a clear error.
- Added: `mngr push my-agent:subpath --sync-mode=git` and `mngr pull my-agent:subpath --sync-mode=git` now operate on the specified subpath. Previously these were rejected with "subpath specifications are not supported in git mode" because the underlying `sync_git` always operated on `agent.work_dir`. The git sync API has been refactored to take an explicit `remote_path: Path`, so subpaths now work uniformly across rsync and git modes.
- Fixed: `mngr pair AGENT:subpath`'s pre-sync now operates on the specified subpath. Previously the divergence detection ran on `subpath` but the actual git push/pull still ran against `agent.work_dir`, so subpaths under different branches than the agent's work-dir root would silently sync the wrong tree.
