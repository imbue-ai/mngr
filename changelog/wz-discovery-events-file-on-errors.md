- mngr: `mngr list` now writes a `DISCOVERY_FULL` snapshot to the discovery events file even when per-provider errors occurred. The new snapshot is marked `is_partial=True` so consumers (`mngr observe --discovery-only` tailers, minds desktop) can distinguish it from a complete snapshot and avoid treating partially-discovered state as authoritative. Previously, any per-provider error caused the snapshot to be skipped entirely, leaving consumers stuck on whatever stale snapshot existed before.
- Note: this adds a new `is_partial: bool` field to the `FullDiscoverySnapshotEvent` schema (defaults to False). Because `EventEnvelope` uses pydantic `extra="forbid"`, long-running `mngr observe` / minds desktop processes started before this change will reject new snapshots with `DiscoverySchemaChangedError` and must be restarted.
