- `mngr_lima`: when a Lima VM boots, `ssh-keyscan` now polls for up to 60s instead of giving up on the first attempt. Lima's macOS-side hostagent accepts TCP connections on `127.0.0.1:<port>` before the guest sshd is listening, so during that window `ssh-keyscan` connects but its first write fails with EPIPE ("Broken pipe"), yielding rc=1 + empty stdout — indistinguishable from a permanently broken host. The old single-shot code logged a warning and silently moved on, leaving `known_hosts` empty and causing downstream paramiko-based SSH to fail with a cryptic "StrictPolicy: No host key for [127.0.0.1]:<port>". Empirically the race window is ~7s for a VM restart and tens of seconds on first boot; the new code polls via `mngr.utils.polling.poll_until` (60s budget) and raises `HostConnectionError` loudly when the budget is exhausted.
