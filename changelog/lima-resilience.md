- `mngr_lima`: when a Lima VM boots, `ssh-keyscan` now polls for up to 60s instead of giving up on the first attempt. sshd often finishes loading host keys slightly after the TCP port becomes reachable, so the first `ssh-keyscan` returns empty and the old single-shot code logged a warning and silently moved on, leaving `known_hosts` empty and causing downstream paramiko-based SSH to fail with a cryptic "StrictPolicy: No host key for [127.0.0.1]:<port>". The new code polls via `mngr.utils.polling.poll_until` and raises `MngrError` loudly when the budget is exhausted.
- `mngr_lima`: bump `limactl_start_new` default timeout from 600s to 1800s; Lima first-boot can exceed 10 min on machines with slow disk or under heavy load (downloading the Ubuntu image, running cloud-init, regenerating SSH host keys, etc.).
- `mngr_lima`: switch the serial-log tailer to `tail -F`, which works on both BSD (macOS) and GNU tail. The previous `tail --follow=name --retry` was a GNU-only flag combo and exited immediately on macOS with "unrecognized option", silently losing serial-log diagnostics during VM boot.
