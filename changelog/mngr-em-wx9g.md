Suppress unknown-field warnings during `mngr plugin add`. When adding a new plugin, the existing `.mngr/settings.toml` is expected to reference fields and provider backends that the not-yet-installed plugin will provide; the resulting "Unknown fields in agent_types.X" and "Provider 'X' references unknown backend 'X'" warnings were noisy and misleading. `plugin add` now passes a `silent` flag through the config-parsing chain (`setup_command_context` -> `load_config` -> `parse_config` -> `_parse_*`) that drops unknown fields without logging. Other commands keep the previous warn-on-unknown behavior.
