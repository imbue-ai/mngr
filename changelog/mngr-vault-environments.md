Move minds to multi-environment deploys (`dev`, `staging`, `production`) backed by HCP Vault. New `minds env {create,list,destroy} <name>` CLI provisions dynamic per-developer dev environments (Modal env + Neon DB + SuperTokens app), writing a self-contained `~/.minds/envs/<name>.toml` per dev env. `minds run` gains `--config-file <path>` for env selection (defaults to a build-bundled `client.toml` for production builds, else the dev tier defaults). `scripts/push_modal_secrets.py` now reads tier secrets from Vault and pushes them to Modal -- the old `.minds/<env>/*.sh` flow is gone. Every baked-in `joshalbrecht`-owned connector URL has been removed; callers now configure the URL explicitly via the per-tier `client.toml` or env var. New docs at `apps/minds/docs/environments.md` and `apps/minds/docs/vault-setup.md` walk through the new operator workflow.
