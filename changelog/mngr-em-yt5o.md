Improve the "Unknown fields in agent_types.X" error message when X is not a registered agent type. Previously the user saw only "Unknown fields in agent_types.claude: ['is_fast']. Valid fields: [...]" which reads like a field typo. The message now also lists the most common root causes: the plugin that provides the agent type may not be installed, the type name may be misspelled, or the field names may be misspelled. The hint only appears when the agent type is truly unregistered; mistyped fields on a registered type still get the original message. Added a sibling helper `is_agent_type_registered` to the agent config registry.
