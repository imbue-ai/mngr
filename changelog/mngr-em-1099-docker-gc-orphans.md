- Changed: `mngr gc` now reaps orphaned Docker resources by default — `mngr-*` containers and `mngr-build-host-*` images that were left behind by interrupted destroy flows or by host records that were deleted without a clean teardown. Resources younger than the minimum online host age are skipped to avoid racing with in-progress host creation. Non-Docker providers are unaffected (the new `gc_orphaned_resources` provider hook defaults to a no-op). CLI summary lines now report `Orphaned containers: N` and `Orphaned images: N (freed ...)`. Closes #1099.
