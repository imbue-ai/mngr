- Desktop client now polls workspace-server readiness after `mngr create` returns, before redirecting the user from the creating-progress page
  - Two-stage poll: first waits for the workspace's `system_interface` URL to be registered via the backend resolver, then issues `GET <url>/` until it returns HTTP 200 (proving uvicorn's lifespan completed and the frontend bundle is being served)
  - On readiness-check timeout (default 60s) creation still completes and the user is redirected; the previous behavior of immediately landing on a 502/503 from the subdomain forwarder is replaced with explicit progress-page log lines reporting the wait
- Subdomain forwarder no longer serves an HTML "retry" page when the workspace URL has not yet been registered; HTML and non-HTML clients alike now receive a plain 503 in that window. The creating-page readiness probe is what gates the user-facing redirect, so this state should not be reached during the normal creation flow.
- `AgentCreator` now takes a `backend_resolver` constructor argument (previously took `server_port`); the redirect URL is no longer assembled from a port number, so the legacy `server_port` field has been removed.
