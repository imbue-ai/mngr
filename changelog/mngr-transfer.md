- Fixed: `mngr clone <agent> <new-name> --provider modal` (and `mngr migrate` for cross-host moves) now succeeds when the source agent lives on the local host. Previously the plugin-state rsync passed the destination host as both source and target, so rsync ran on the modal sandbox and failed with `change_dir "/Users/.../plugin" failed: No such file or directory` because the source plugin dir only exists on the laptop. Now `_transfer_source_plugin_data` rsyncs from the local host to the destination host, so the Claude transcript, session history, and memory carry over to the new host.
