Remove the "primary chat" concept from the minds app on the mngr side. The main workspace agent is now a non-chat `system-services` agent that runs only the bootstrap service manager; the user-chosen workspace name applies to the mngr host, not to any agent. The mngr_imbue_cloud connector gains a `host_name` column on leases plus a `PATCH /hosts/{id}/name` endpoint, and `ImbueCloudProvider.rename_host` is now implemented (previously `NotImplementedError`). The minds desktop client and pool baking switch to the `system_services` template and drop the `-host` suffix from addresses. The FCT-side companion changes (new `system_services` agent type + create template, bootstrap creating the `assistant` chat agent) ship separately in `forever-claude-template`.
