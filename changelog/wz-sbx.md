Add new `mngr_sbx` provider backend plugin: run mngr-managed agents inside Docker Sandboxes ([`sbx` CLI](https://docs.docker.com/ai/sandboxes/)). Sandboxes are bridged into mngr's SSH-based host abstraction via an in-sandbox sshd + `sbx ports --publish 22`, with a per-host "keeper" subprocess holding each sandbox in the running state across mngr CLI invocations. End-to-end `mngr create @.sbx`, `mngr list`, `mngr exec`, and `mngr destroy` work; auth follows sbx's own `sbx login` device-code flow, with a typed `SbxNotAuthorizedError` surfaced when credentials are missing. Snapshots/volumes/`start_host` are not yet supported (follow-ups).
