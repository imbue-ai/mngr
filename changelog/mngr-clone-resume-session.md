- Fixed: a cloned claude agent now actually resumes the source agent's conversation (not just inherits the session JSONL on disk). The previous fix in #1598 made the plugin/ transfer work cross-host, but the resulting JSONL was filed under the *source's* encoded work_dir (the per-project subdir name Claude uses), so Claude on the destination couldn't find it and started fresh; ``claude_session_id`` was also lost because it lives outside ``plugin/``. ``_adopt_cloned_session`` now (a) renames the project subdir to the destination agent's encoded work_dir, (b) writes ``claude_session_id`` on the destination to the source's JSONL filename stem (the ground-truth session id on disk -- the source's ``claude_session_id`` file itself often holds the agent UUID from the SessionStart hook default rather than the real id), and (c) carries forward ``claude_session_id_history`` verbatim, so the destination's startup ``claude --resume "$MAIN_CLAUDE_SESSION_ID"`` targets the source's session.
