- Fixed: a cloned claude agent now actually resumes the source agent's conversation (the model sees and acts on the source's history), not just inherits the session JSONL on disk. The previous fix in #1598 made the plugin/ transfer work cross-host, but several pieces still had to land for ``claude --resume`` to succeed on the destination. ``_adopt_cloned_session`` now does all of them: (a) renames the project subdir under ``plugin/claude/anthropic/projects/`` to the destination agent's *realpath-resolved* work_dir encoding (on Modal, ``/mngr/projects/agent-X`` is a symlink onto ``/__modal/volumes/<vol-id>/projects/agent-X``; claude uses the resolved form and looks for the JSONL there), (b) deletes the rsynced ``sessions-index.json`` so claude rebuilds it instead of using stale source-pointing entries that prevent session lookup, (c) writes ``claude_session_id`` on the destination to the source's JSONL filename stem (the ground-truth session id on disk -- the source's ``claude_session_id`` file itself often holds the agent UUID from the SessionStart hook default rather than the real id), and (d) carries forward ``claude_session_id_history`` verbatim. With all four in place, the destination's startup ``claude --resume "$MAIN_CLAUDE_SESSION_ID"`` succeeds and the cloned agent picks up where the source left off.
