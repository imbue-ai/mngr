- `apps/minds`: bundle Lima with the desktop app. The build script now downloads the official Lima 2.1.1 release tarball for the build host's platform/arch and extracts it into `resources/lima/`; the packaged backend prepends `resources/lima/bin` to `PATH` so `limactl` is found without a separate `brew install lima` step. On macOS Apple Silicon this is fully self-contained because Lima's default `vz` backend uses Apple's Virtualization framework directly; on macOS Intel and Linux, Lima still requires QEMU on the host machine (out of scope for this change). `mngr_lima` is unchanged — the bundled binary easily satisfies its existing minimum version, and standalone `mngr` users keep the same Lima floor they had before.
