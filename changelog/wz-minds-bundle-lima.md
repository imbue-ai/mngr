Bundle Lima 2.1.1 into the Minds desktop app and make `lima` the default compute provider. `scripts/build.js` now downloads the official Lima release tarball for the build platform/arch and extracts the full `bin/`, `libexec/`, `share/` tree into `resources/lima/`; the Electron shell adds `resources/lima/bin` to the `PATH` of the `mngr` subprocess so `mngr_lima`'s `shutil.which("limactl")` resolves the bundled binary. The agent-creation form, API fallback, and `AgentCreator.start_creation` API default all switch from `LOCAL` to `LIMA`, so a fresh user on a pristine macOS install can accept the defaults and create an agent without installing anything. On Apple Silicon Lima uses the built-in `vz` (Virtualization.framework) backend, so no QEMU is bundled.
