`mngr list` no longer drops imbue_cloud hosts when SSH to a leased container fails. Discovery is now connector-API driven (no SSH); the per-host SSH happens during detail collection and falls back to a richer offline path that consults the leased VPS via outer SSH (port 22) and `docker inspect`s the container so the listing shows the actual state (STOPPED / PAUSED / DESTROYED / CRASHED with exit code, or UNAUTHENTICATED if the container is up but inner SSH was unreachable). The offline `HostDetails` for imbue_cloud now also carries the SSH target and the underlying connection error in `failure_reason`, so JSONL output is self-describing. Same defense added to vps_docker/vultr so a single unreachable VPS no longer drops the others, and a pre-existing crash in the framework offline path (`CommandString("")` violating `NonEmptyStr`) is fixed.
