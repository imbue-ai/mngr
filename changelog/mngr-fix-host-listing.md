`mngr list` no longer drops imbue_cloud hosts when SSH to a leased container fails. Discovery is now connector-API driven (no SSH); the per-host SSH happens during detail collection and falls back to an offline `HostDetails` (with state `UNAUTHENTICATED` for auth failures, or a derived offline state otherwise) instead of silently filtering the host out. Same defense added to vps_docker/vultr so a single unreachable VPS no longer crashes discovery for the others.
