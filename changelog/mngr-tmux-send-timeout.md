Bound `tmux send-keys` / `load-buffer` / `paste-buffer` / `delete-buffer` calls in the agent send-message pipeline with a 30s timeout. Previously these were unbounded; if a tmux client got stuck talking to its server, the worker thread would hold the per-agent `message.lock` flock forever, queueing every subsequent send to that agent behind it. Observed in the wild when a single chat-window send to a Claude agent wedged for hours and turned every later send into a 504 at the proxy.
