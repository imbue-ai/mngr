- `mngr create` now rejects `--reuse` combined with `--new-host` (and therefore `--reuse --update --new-host`) with a clear error. The two flags contradict each other: `--reuse` adopts an existing agent on an existing host, while `--new-host` always provisions a fresh host that has no agent to adopt. The combination used to be accepted silently.
- The minds desktop client no longer passes `--reuse` / `--update` when creating workspace agents. Every minds launch mode (LOCAL, LIMA, CLOUD, IMBUE_CLOUD) passes `--new-host`; the IMBUE_CLOUD pre-baked `system-services` agent is adopted via mngr's `host.pre_baked_agent_id` duplicate-name exemption, so `--reuse` was unnecessary.
