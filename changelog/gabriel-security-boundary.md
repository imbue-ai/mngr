Document a cross-mind boundary leak: services emitting absolute `http://localhost:<port>/...` URLs (e.g. a FastAPI app started without `ROOT_PATH=/service/<name>`) bypass the per-mind subdomain proxy and, combined with Lima's default auto-port-forwarding of `0.0.0.0`-bound guest ports to the host, can deliver form submissions from one mind to another. See `specs/minds-security-boundary/investigation.md` for the failure path and proposed defense-in-depth fixes (CSP, blocking host-bound guest ports, absolute-URL rewriting, `ROOT_PATH` enforcement, migration hygiene). No code changes in this PR.
