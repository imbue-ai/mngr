- `mngr_modal`: fix sharding-dependent flake in `routes/test_snapshot_and_shutdown.py`. The `deployed_snapshot_function` fixture is now function-scoped instead of module-scoped, so every test in the file invokes the Modal SDK during its own per-test phase. This satisfies the resource-guard's per-test `@pytest.mark.modal` enforcement check, which previously failed for any test that consumed the cached module-scoped fixture without making its own direct Modal call. Whether a given test tripped the check depended on offload's shard partitioning of acceptance tests, so the flake only surfaced on PRs whose test-count changes perturbed the partition map.
