`mngr create -t modal` (and `mngr clone ... -t modal`) now build multistage Dockerfiles instead of crashing with `AssertionError: Multistage Dockerfiles are not supported yet`. The mngr default Dockerfile is multistage (it compiles the `offload` binary in a `rust:1-bookworm` builder stage), and the Modal provider used to assert against any Dockerfile with more than one `FROM`. The provider now routes multistage Dockerfiles through `modal.Image.from_dockerfile` (single-shot, no per-layer caching) while keeping the per-instruction `dockerfile_commands` path for single-stage Dockerfiles.
