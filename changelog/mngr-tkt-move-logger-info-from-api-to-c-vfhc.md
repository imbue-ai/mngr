Refactored where user-visible info-level log lines originate so that `libs/mngr/imbue/mngr/api/` no longer emits `logger.info`; user-facing messages now come from `libs/mngr/imbue/mngr/cli/` and the api uses `logger.debug` / `log_span` instead. The visible CLI output is unchanged for the high-traffic commands (`mngr create`, `mngr connect`, `mngr provision`): "Starting agent {name} ...", "Sending initial message...", "Provisioned agent: {name}", and the existing "Connecting to agent: {name}" still surface at default verbosity. Two minor lines that previously appeared from inside `ensure_host_started` / `ensure_agent_started` ("Host is offline, starting it..." and "Agent {name} is stopped, starting it") no longer surface at default verbosity when an auto-start happens transitively through `materialize_agent` / `find_one_agent`; `--verbose` restores them.
