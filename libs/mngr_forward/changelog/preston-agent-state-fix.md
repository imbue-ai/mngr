The `mngr forward` reverse proxy now fails visibly instead of getting silently stuck when an agent's backend dies (e.g. after the laptop sleeps and the container restarts). Backend-failure HTTP responses (502/503/504) now return a JSON `{"detail": ...}` body to callers that send `Accept: application/json`, so a frontend whose request library parses the body as JSON surfaces a real error message instead of the literal string "null" (browser navigations still get the styled loading page unchanged).

Websocket forwarding now closes the browser-facing socket as soon as either direction ends, rather than waiting on a client-to-backend pump that can block forever once the backend dies -- so the browser sees the disconnect and can reconnect instead of freezing. A mid-stream backend websocket death now also emits a `system_interface_backend_failure` signal, so a consumer's health-tracking and recovery still fire when the websocket was the only open channel.
