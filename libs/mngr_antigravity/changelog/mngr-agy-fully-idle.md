`antigravity` agents now stay RUNNING while a subagent or backgrounded task they launched is still working, instead of flipping to WAITING the moment the root agent's turn ends.

- The `Stop` hook no longer clears the `active` lifecycle marker unconditionally. It now runs `clear_active_marker_when_idle.sh`, which removes the marker only when agy's Stop-hook payload reports `"fullyIdle":true` (the root agent and every subagent / background task it launched have all finished). agy runs Stop hooks once per execution and emits `fullyIdle:true` on the final, fully-idle stop, so the agent reports WAITING only when it is genuinely idle.
- The marker drives `BaseAgent`'s RUNNING/WAITING detection (present => RUNNING, absent => WAITING), and `PreInvocation` still touches it before every model call. Verified live against agy 1.0.5 that the Stop payload carries `fullyIdle` (and that it is `true` for a simple terminal stop); the not-fully-idle background-task transition rests on agy's documented per-execution stop-hook + full-idle-waiter behavior.
