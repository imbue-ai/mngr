`antigravity` agents now stay RUNNING while a subagent or backgrounded task they launched is still working, instead of flipping to WAITING the moment the root agent's turn ends.

- The `Stop` hook no longer clears the `active` lifecycle marker unconditionally. It now runs `clear_active_marker_when_idle.sh`, which removes the marker only when agy's Stop-hook payload reports `"fullyIdle":true` (the root agent and every subagent / background task it launched have all finished). agy fires the Stop hooks each time the root agent goes idle: an interim Stop carries `"fullyIdle":false` (marker kept, agent stays RUNNING) and a final Stop carries `"fullyIdle":true` once the async work finishes (marker cleared, agent flips to WAITING), so the agent reports WAITING only when it is genuinely idle.
- The marker drives `BaseAgent`'s RUNNING/WAITING detection (present => RUNNING, absent => WAITING), and `PreInvocation` still touches it before every model call. Verified live against agy 1.0.5: a backgrounded shell task (started, then left to run while the agent went idle) produced a `fullyIdle:false` Stop followed by a `fullyIdle:true` Stop -- exactly the sequence this gate relies on.
