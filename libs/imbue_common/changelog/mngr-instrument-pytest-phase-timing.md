Added opt-in pytest phase timing for diagnosing per-batch test overhead. When the `PHASE_TIMING` environment variable is set, each pytest session writes a `.test_output/phase_timing_<uuid>.json` recording the wall-clock split between startup (process start to session start), collection, and the call phase. It is a complete no-op unless `PHASE_TIMING` is set, so normal and CI runs are unaffected. offload already downloads `.test_output/**`, so the files surface automatically as CI artifacts.
