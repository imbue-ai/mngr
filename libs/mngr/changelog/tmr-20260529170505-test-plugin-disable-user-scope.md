- Fixed the release e2e test `test_plugin_disable_user_scope` to match `mngr plugin disable`'s actual (intended) behavior. Disabling an unregistered plugin is a soft pre-configuration: the command succeeds, warns that the plugin is not registered, and persists `plugins.<name>.enabled = false` to the chosen config scope. The test now asserts success, the warning, persistence at the user scope (via `mngr config get ... --scope user`), and scope isolation (the setting is absent from the project scope).
