Fixed the `test_advanced_create_reuse_modal` e2e release test so its `mngr list` observation commands scope discovery to the modal provider (`--provider modal`). The isolated e2e environment leaves every registered backend enabled by default, so an unscoped `mngr list` would try to reach the credential-less AWS provider and exit with the provider-inaccessible code (6) even though the modal agent was found, failing the test for a reason unrelated to the `--reuse` idempotency it verifies.
