Audited and strengthened the unit tests under `imbue/mngr/utils/`, replacing weak, tautological, or fragile assertions with ones that actually verify behavior. None of these are user-visible behavior changes; they harden the test suite.

Process-death watchers (`parent_process_test.py`): added a real-process end-to-end test that spawns an intermediate parent and a watched child, kills the parent, and verifies the watcher actually sends SIGTERM to the reparented child (the previously-untested positive branch), plus a Linux cross-check of `_read_grandparent_pid` against psutil's process tree with the macOS `/proc` gap documented. Tests poll with timeouts (no `time.sleep`) and use real processes (no mocks).

Logging (`logging_test.py`): setup-logging file tests now assert the logged text lands in the target file (not just that the file exists); the flat-JSONL test pins the deterministic loguru fields to exact values; the paramiko routing test records and asserts the emitted log *level* instead of only substring presence on a level-blind sink (the mislabeled WARNING case was split out and renamed). Added a `@log_call` decorator entry/exit-record test, a suppressor test proving console output is withheld until replay, and a paramiko-traceback helper that asserts the traceback passes through a paramiko frame. Removed an unused fixture parameter and a `monkeypatch.setattr` (using the production `setup_logging` path instead), lowering the `PREVENT_MONKEYPATCH_SETATTR` ratchet for `utils/` from 33 to 32.

Editor (`editor_test.py`): the `wait_for_result` success/whitespace/empty tests now use a real fake-editor script that writes to its `$1` argument (instead of `EDITOR=true` with a pre-seeded file), so they verify the editor's output is read. Added coverage for the `on_exit` monitor-thread callback / cached-result path and for cleanup-after-finished plus idempotent double-cleanup.

CEL utils (`cel_utils_test.py`): added a positive test that a genuine (non-tolerant) eval error logs the warning; the `compile_cel_sort_keys` tests now evaluate the compiled program rather than checking only length/direction; `evaluate_cel_sort_key` asserts the real `StringType` result instead of a `str()` coercion; added a non-matching `endsWith` case; and the `TolerantPathError` dual inheritance (`TypeError` + `MngrError`) is now asserted.

Git utils (`git_utils_test.py`): `get_head_commit` and the local-clone test now assert against the repo's real HEAD instead of just a sha-shaped string / the presence of `.git`; seven `git init` tests gained `setup_git_config` for consistent isolation.

Detail renderer (`detail_renderer_test.py`): converted eight single-input rendering tests from loose substring checks to full inline-snapshot assertions; added multi-stem and repeated-stem linkification boundary tests; renamed/strengthened the empty-directory test.

Deps (`deps_test.py`): removed a tautological `ALL_DEPS` union test, scoped a misleading docstring, and guarded the bash-version test with `skipif(no bash)`.

Smaller utils: `env_utils_test.py` gained coverage for `build_source_env_shell_commands` quoting and `parse_env_file`'s None-value drop; `file_utils_test.py` gained an empty-file `read_json_dict` case; `rsync_utils_test.py` collapsed four near-duplicates into one parametrized test and added a prefix-collision case; `click_utils_test.py` gained a multi-alias accumulation case; `duration_test.py` added trailing-garbage/whitespace negative cases; `polling_test.py` replaced wall-clock timing assertions with deterministic call-counting; `name_generator_test.py` replaced near-tautological non-empty checks with wordlist-membership and style-keyed cache assertions (using a non-flaky token-count lower bound for coolname slugs).
