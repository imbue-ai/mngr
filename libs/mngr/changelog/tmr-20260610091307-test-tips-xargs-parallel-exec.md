Made the `test_tips_xargs_parallel_exec` e2e tutorial test disable the docker provider in its isolated user config before running. The test demonstrates fanning `mngr exec` out across modal hosts via xargs, but `mngr list` discovers across every enabled provider before applying the `host.provider == "modal"` filter, so an unreachable docker provider made `mngr list` exit 1 even though the modal ids printed correctly. Scoping discovery to the modal-only tutorial removes a hidden dependency on docker being available.
