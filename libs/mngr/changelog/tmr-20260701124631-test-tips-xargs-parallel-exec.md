Fixed the `test_tips_xargs_parallel_exec` tutorial e2e test so it no longer requires a standalone `mngr list --include 'host.provider == "modal"'` to exit 0.

`mngr list` enumerates every enabled provider backend, so a credential-less cloud provider (e.g. aws) makes it exit `EXIT_CODE_PROVIDER_INACCESSIBLE` while still printing the modal ids to stdout. The tutorial command pipes those ids into `xargs -P 5 ... mngr exec`, so the pipeline's exit code is xargs' (not `mngr list`'s) -- which is exactly the behavior the test's documented scope covers. The test now captures the agent id from `mngr list` stdout without asserting that standalone command succeeds, and continues to verify the pipeline exits 0 and its output carries the echoed `$MNGR_AGENT_ID` and an absolute `pwd` path.
