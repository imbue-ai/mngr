Hardened the `test_plugin_disable_affects_create` e2e release test. It previously disabled the claude plugin at the default project scope, which created a fresh `settings.toml` lacking `is_allowed_in_pytest = true`; the subsequent `mngr create` then failed the pytest config guard rather than because the plugin was disabled, so the test passed for the wrong reason. The test now disables/re-enables at the local scope (the opted-in `settings.local.toml`), verifies the plugin shows as disabled in `mngr plugin list`, and asserts that `mngr create` fails specifically because the claude agent type's plugin is disabled (and not on the config guard).
