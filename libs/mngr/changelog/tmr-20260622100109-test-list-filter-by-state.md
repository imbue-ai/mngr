Scoped provider discovery per test in the e2e fixture: each test now enables only the `local` backend plus `docker`/`modal` when it carries the matching `@pytest.mark`. The dev/test checkout installs every provider plugin (`uv sync --all-packages`), so the backend registry previously also exposed `aws`/`gcp`/`azure`/`vultr`/`imbue_cloud` (and `docker`/`modal` even for tests that did not use them). Because `mngr list` defaults to `--on-error abort`, an unfiltered `mngr list` aborted with exit 1 the moment the first reachable-state-unknown backend failed discovery -- e.g. AWS raising `ProviderUnavailableError` for missing credentials, or a missing Docker daemon. This broke e2e tutorial tests that run a bare `mngr list`. Pinning `enabled_backends` to the providers each test actually set up matches what a real user running those tutorial commands would have installed.
