Make the tutorial e2e test environment deterministic about which provider backends it discovers.

The e2e fixture now pins `enabled_backends` to the providers each test actually declares it can reach: `local` is always enabled, and `modal`/`docker` are added only when the test carries the corresponding resource-guard mark. Previously every installed provider plugin (aws, vultr, gcp, azure, ...) got a default instance that `mngr list` would query during discovery; in a full monorepo checkout the AWS backend raises a provider-unavailable error when its credentials are not configured, and docker fails when no docker daemon is running, both of which made `mngr list` exit non-zero and broke otherwise-passing tutorial tests (e.g. `test_list_current_project_only`). This is a test-infrastructure change only; it does not affect the shipped `mngr` CLI.
