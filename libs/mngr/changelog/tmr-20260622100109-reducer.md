`mngr list` no longer aborts when an installed-but-unconfigured provider backend is unreachable (e.g. a cloud plugin with no credentials, or a local daemon that is down). Auto-enumerated default providers that raise `ProviderUnavailableError` are now skipped so the reachable providers still list, while a provider you explicitly configured under `[providers.*]` still honors `--on-error` and fails loudly.

Also integrates a batch of e2e/tutorial test fixes: provider discovery is scoped to reachable backends so list-based tests stay hermetic against unconfigured cloud providers, stale rsync resource-guard marks were dropped, several timeouts were widened, and additional happy/unhappy-path coverage was added.
