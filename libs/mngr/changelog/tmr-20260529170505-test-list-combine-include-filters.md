- Fixed the e2e release test `test_list_combine_include_filters` (under `libs/mngr/imbue/mngr/e2e/tutorial/test_labels.py`). It previously ran `mngr list --include ...` against an empty environment and only asserted the command succeeded, which both failed the `@pytest.mark.modal` resource guard (the command never contacted Modal) and verified no real filtering behavior. The test now creates three labeled command agents that each isolate one half of the AND (backend+alive, frontend+alive, backend+stopped), runs the exact tutorial command, and asserts that combining two `--include` filters applies AND logic: the tutorial's `state == "RUNNING"` predicate correctly excludes the alive-but-not-RUNNING backend agent, while a parallel `state == "WAITING"` filter returns only the backend agent that is both correctly-labeled and alive. The spurious `@pytest.mark.modal` mark was removed (the test exercises only local command agents) and `@pytest.mark.tmux`/`@pytest.mark.timeout(300)` were added to support agent creation.
