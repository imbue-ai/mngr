Strengthened the `test_start_dry_run` e2e tutorial test so its "nothing was actually started" check is genuinely discriminating. The test now stops `my-task` before running `mngr list --ids | mngr start - --dry-run`, so the before/after lifecycle-state comparison starts from `STOPPED`. Previously the agent was left running, and because `mngr start` is idempotent on a running agent, the state comparison could not distinguish a true dry-run no-op from an erroneous real start. This is a test-only change; no user-facing behavior changed.
