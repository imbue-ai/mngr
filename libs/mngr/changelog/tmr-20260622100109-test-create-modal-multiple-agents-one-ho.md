Scoped the e2e test profile's `enabled_backends` to the provider backends each test is actually marked for (always `local`/`ssh`, plus `docker`/`modal` when the test carries the matching marker). The full workspace installs every provider plugin (aws, azure, gcp, vultr, imbue_cloud) with discovery enabled by default, so unfiltered commands like `mngr list` would try to reach providers the test never configured -- AWS raising `ProviderUnavailableError` on missing credentials, or docker when no daemon is running -- which `mngr list` reports as an error and exits non-zero on, flipping the exit code even though the agents under test listed correctly. This fixes `test_create_modal_multiple_agents_one_host` (and any other test running an unfiltered `mngr list`) so the listing's exit code reflects only the providers the test set up.
