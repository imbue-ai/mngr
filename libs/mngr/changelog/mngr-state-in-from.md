Added a design spec (`future_specs/agent_cloning.md`) for generalizing how `mngr create --from <agent>` copies per-agent state. It documents the current Claude-only session-adoption mechanism and proposes an opt-in, plugin-registered facility: a declarative `register_clone_state` hook (core copies a plugin's own `plugin/<name>/` sandbox for any agent type) plus an imperative `on_clone_from_agent` hook for plugins that must transform copied data. No behavior change yet -- spec only.
