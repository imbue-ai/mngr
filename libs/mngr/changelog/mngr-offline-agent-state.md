## Derive offline agent lifecycle state from host state

- Added `get_offline_agent_state(host_state)` in `imbue.mngr.hosts.common`, the canonical (pure) mapping from a host's offline/last-known `HostState` to an agent's `AgentLifecycleState` when the agent's process cannot be observed directly. No agent lifecycle state is persisted anywhere (it is computed live from tmux/process info), so the host state is the only available signal once the host is offline. The function sits beside the existing `determine_lifecycle_state` (online derivation) and mirrors `derive_offline_host_state` for hosts.
- A host that is provably down (`BUILDING`, `STARTING`, `STOPPING`, `STOPPED`, `PAUSED`, `CRASHED`, `FAILED`, `DESTROYED`) yields a `STOPPED` agent. When the host's own state is indeterminate (`RUNNING` yet unreachable, `UNAUTHENTICATED`, `UNKNOWN`) the agent state is now reported as `UNKNOWN` rather than a misleading `STOPPED`. There is deliberately no agent equivalent of `DESTROYED`: agent destruction is modeled at the host/discovery layer, so a destroyed host still maps to a `STOPPED` agent.
- `build_agent_details_from_offline_ref` (used when discovery falls back to the offline view after an SSH failure) now derives the agent state via this function instead of hardcoding `STOPPED`; a `None` host state maps to `UNKNOWN`.
