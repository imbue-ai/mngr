Scoped the `mngr list` verification in the in-place-alias e2e test (`test_create_in_place_alias_target`) to `--provider local`, so the assertion no longer depends on (or contacts) remote cloud providers. A bare `mngr list` exits non-zero when an enabled-but-unreachable provider (e.g. AWS without credentials in CI) cannot be queried, which made the test flaky/failing in environments where those providers are not configured.
