- Fixed and strengthened the `test_list_exclude_filter` e2e release test for the LABELS AND FILTERING tutorial section. The test now creates labeled `frontend`/`backend` command agents before running `mngr list --exclude 'labels.team == "frontend"'`, so the filter operates on real data instead of an empty agent list. Added a `@pytest.mark.timeout(120)` override (the slow cross-provider `mngr list` exceeded the default 10s func timeout), added the `rsync`/`tmux` marks that local command-agent creation requires, and removed the `modal` mark (the command never actually invokes Modal). The test now asserts the excluded agent is dropped while the non-matching agent is kept, and a companion `test_list_exclude_filter_no_match` covers the edge case where the filter matches nothing.
