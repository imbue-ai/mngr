- Fixed `mngr config get` and `mngr config list` (merged, all-scopes view) to address command defaults and create-template options at the same flat path that `mngr config set` writes and the docs document (e.g. `commands.create.provider`, `commands.create.connect`, `create_templates.<name>.<option>`). Previously these merged-mode reads leaked the internal `defaults` / `options` wrapper, so a value written with `mngr config set commands.create.provider modal` could not be read back via `mngr config get commands.create.provider` and never appeared in `mngr config list`.
