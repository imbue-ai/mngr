Hardened offline host state derivation against untrusted `stop_reason` values. `derive_offline_host_state` now only honors the legitimate offline stop reasons (`PAUSED`/`STOPPED`/`DESTROYED`); any unknown, malformed, or otherwise-untrusted value (e.g. a stop reason of `RUNNING` arriving via a provider's HTTP request body) falls back to `CRASHED` instead of raising a `ValueError` out of discovery or being interpreted as a live state. `None` continues to mean `CRASHED` as before.

Corrected the `supports_shutdown_hosts` docstring on `ProviderInstanceInterface`: it gates stopping and starting a host's compute directly, not resuming from a snapshot.

Fixed `mngr destroy`/`start` (and other agent-resolving commands) silently failing to find an agent on a *running* Modal host. `mngr list` reads a running host's agents live over SSH, but resolution read them only from Modal's state volume, whose per-agent records can lag a still-running host -- so a live, running agent was unresolvable, and `mngr destroy --force` reported a misleading "No agents found to destroy" with a success exit (the UI spun on "Destroying..." forever). When a requested identifier is not found, the resolver now re-reads the agents of any still-missing *running* host live (the same source `mngr list` uses) and resolves against those. This is scoped to Modal via a new `discovers_agents_from_live_host` provider flag (default True; only Modal sets it False) -- every other provider already discovers agents live and is left completely untouched. It runs only on a miss (zero extra work on the common path) and degrades gracefully, keeping existing records for any running host it cannot reach.
