Made the `test_list_pipe_stdin` e2e tutorial test deterministic across environments. The test exercises the `mngr list --format "{id}" | head -n 2 | mngr list --stdin` pipe, which runs full all-provider discovery. In an isolated environment several default-enabled providers are unreachable -- the cloud providers (AWS/GCP/Azure/Vultr/imbue_cloud) have no credentials and Docker has no running daemon -- so each surfaces a `ProviderUnavailableError` and `mngr list` exits non-zero, even though the local agent lists correctly. The test now disables those unreachable providers (the remediation each error message itself recommends) before running the verbatim pipe, leaving the local and Modal providers enabled, so the pipe's exit status reflects only the local discovery it actually depends on.
