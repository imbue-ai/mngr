- Fixed the `test_list_stopped_filter` e2e release test (`libs/mngr/imbue/mngr/e2e/tutorial/test_list.py`): added a `@pytest.mark.timeout(120)` override (the bare `mngr list --stopped` triggers cross-provider discovery that can exceed the default 10s pytest timeout) and removed the `@pytest.mark.modal` mark. `mngr list --stopped` against the empty test environment only performs Modal discovery via the Python SDK inside the `mngr` subprocess (invisible to the in-process resource guard) and never shells out to the `modal` CLI, so the guard's NEVER_INVOKED check could not be satisfied.
- Added `test_list_stopped_filter_shows_stopped_agent`, a behavior test sharing the same tutorial block, which creates a local command agent, verifies it is absent from `mngr list --stopped` while running, stops it, and verifies it then appears as `STOPPED`. This actually exercises the `--stopped` filter rather than only smoke-testing the command against an empty environment.
