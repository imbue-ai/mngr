Internal test fix (no user-visible change): made the `test_full_lifecycle` e2e release test reliable in environments that register credential-based cloud provider plugins (e.g. AWS) without credentials.

The agent under test is local, so the test now scopes every command to the local provider (`--provider local` for `mngr list`, the `@localhost.local` address for targeted commands, and `--no-gc` on `mngr destroy`). Previously the unscoped commands fanned discovery out to unreachable remote providers, which made `mngr list` abort with a non-zero exit and made `mngr start` / the post-destroy garbage-collection sweep hang on a provider credential lookup. Also removed the test's spurious `@pytest.mark.rsync` mark: the agent is created in a git repo on the same host, so the transfer uses git-worktree and never invokes rsync.
