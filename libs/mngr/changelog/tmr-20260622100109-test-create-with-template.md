Fix the `test_create_with_template` and `test_create_with_nonexistent_template` e2e release tests, which were inheriting the default 10s func-only pytest timeout and failing before `mngr create` could finish provisioning an agent. Both tests now carry explicit `@pytest.mark.timeout(...)` overrides (300s for the happy path that creates and execs into an agent, 120s for the fast-failing unknown-template path), matching the convention used by the other e2e tests in this suite.
