Hardened the `test_tips_exec_env_inspect` tutorial e2e test so its agent-id cross-check scopes `mngr list` to the local provider (`--provider local`) instead of doing a full provider scan. The full scan made the test exit non-zero whenever an unrelated provider backend was unavailable in CI (e.g. a stopped Docker daemon or an unconfigured AWS account), even though the local agent under test was healthy. Also removed the superfluous `@pytest.mark.rsync` marker: the test creates a local agent inside a git repo, which uses git-worktree transfer and never invokes rsync.
