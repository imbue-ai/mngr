Scoped the `mngr list` verification in the `test_create_quiet_suppresses_output` e2e test to the local provider (`--provider local`). The test creates a local `command` agent and is not marked as requiring Docker, but the unscoped `mngr list --format json` queried every backend and aborted when Docker was not installed. Scoping discovery to the agent's own provider keeps the verification precise while removing the incidental dependency on other backends being present.
