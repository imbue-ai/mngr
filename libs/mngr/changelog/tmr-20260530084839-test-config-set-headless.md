Fixed the `test_config_set_headless` e2e release test: it now pre-seeds the project-scope `settings.toml` with the pytest opt-in flag so the merged `mngr config get headless` read succeeds. Previously the test failed because `mngr config set headless true` writes a fresh project `settings.toml` without `is_allowed_in_pytest = true`, which the pytest-gated config loader refused to load during the merged read. This is a test-harness-only gate and does not affect real users.
