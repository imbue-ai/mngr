Test-only: the e2e tutorial fixture now restricts `enabled_backends` to the provider backends that are actually reachable in the test runner (always `local`; `modal` when Modal credentials are present; `docker` only for `@pytest.mark.docker` tests) instead of leaving every installed provider plugin enabled. The monorepo dev environment installs uncredentialed cloud plugins (aws, azure, gcp, ovh, vultr), and an unavailable backend makes `mngr list` -- which defaults to `--on-error abort` -- fail with a fatal exit-1, which was breaking the bare `mngr list` tutorial tests (e.g. `test_list_format_jsonl_recap`).

Test-only: added `test_list_format_jsonl_with_agent`, a data-bearing companion to `test_list_format_jsonl_recap` for the same tutorial block. It creates a real agent and asserts the `--format jsonl` stream emits that agent as a self-contained JSON object on its own line (carrying per-agent fields like `name`/`id`, not the `{"agents": [...]}` envelope that `--format json` uses), exercising the non-empty JSONL path the original empty-listing test cannot reach.
