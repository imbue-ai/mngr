Strengthened the `mngr observe --discovery-only` end-to-end tutorial test so it verifies both halves of its scope. The test previously wrapped the command in a 1-second `timeout` and only checked that it was killed by the timeout (exit 124) -- observe was still starting up at that point, so no discovery events were ever streamed or observed. It now runs long enough for the discovery pollers to emit their per-provider snapshots and asserts that the streamed stdout parses as genuine `DiscoveryEvent` JSONL, confirming the command actually streams discovery events rather than merely starting without crashing.
