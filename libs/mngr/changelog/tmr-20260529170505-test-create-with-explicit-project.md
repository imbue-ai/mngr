- Fixed the release e2e test `test_create_with_explicit_project` in `libs/mngr/imbue/mngr/e2e/tutorial/test_projects.py`. The test was marked `@pytest.mark.modal`, but `mngr create` with the default local provider never invokes modal, so the resource guard failed the test (modal mark but modal never invoked). Removed the spurious `modal` mark, added a `@pytest.mark.timeout(120)` margin, and added a verification step (`mngr list --provider local --project other-project --fields "name,labels.project"`) that asserts the created agent is actually tagged with the explicit `other-project`. The list is scoped to the local provider so the assertion is deterministic across local and CI runs.
- Also added a negative assertion that filtering by an unrelated project excludes the agent, guarding against the `--project` filter being a no-op.
