Fixed the e2e plugin tests (`test_plugin_disable_enable_roundtrip` and `test_plugin_disable_affects_create`) so they seed the project-scope `settings.toml` with the pytest opt-in before disabling a plugin. `mngr plugin disable`/`enable` default to project scope, which previously created a `settings.toml` lacking `is_allowed_in_pytest = true`, causing every subsequent `mngr` invocation in the same test to abort on the pytest config guard. This was a test-harness-only issue and never affected real users.
