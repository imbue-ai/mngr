Fixed the `test_create_named_agent` e2e test. It creates a local (`--type command`) agent and so never invokes the Modal CLI, but it carried `@pytest.mark.modal`, which the resource guard fails with "marked with @pytest.mark.modal but never invoked modal". Removed the incorrect marker (matching the documented policy in `test_create_modal.py`: the mark is only for e2e tests that create a Modal agent and thus shell out to `modal environment create`). Also added `@pytest.mark.timeout(120)` because this test runs three subprocess commands (create, list, exec) plus first-run host provisioning, which can exceed the default 10s per-test timeout.
