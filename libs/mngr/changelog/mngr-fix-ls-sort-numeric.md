Fixed `mngr ls --sort` (aka `mngr list --sort`) ordering numeric fields lexicographically instead of numerically. Sorting by a numeric field such as `idle`, `age`, or `runtime` (e.g. `mngr ls --sort 'idle asc'`) now orders by value, so 9 precedes 100 rather than "100" preceding "9". String and timestamp fields (e.g. `name`, `create_time`) are unchanged, and `None`/missing values still sort last in both directions.
