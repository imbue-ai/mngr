Tightened the `test_message_filtered_via_stdin` e2e test to match its documented scope: it no longer asserts that the standalone `mngr list --include ... --ids` precondition command exits 0. That exit code reflects provider reachability (`mngr list` walks every enabled backend, and an unreachable provider exits with code 6), which is outside the no-op path under test. The test still verifies the only precondition the scope requires -- that the filter emits an empty id list -- and that piping it into `mngr msg -` exits 0 without claiming any delivery.
