Removed the spurious `@pytest.mark.rsync` marker from the `test_exec_cwd` e2e tutorial test. The test creates a local command agent in a git repo and only runs `mngr exec ... pwd`, which never invokes rsync (rsync is used by `mngr rsync` and by preservation of non-git projects). The marker caused the resource guard to fail the otherwise-passing test with "marked with @pytest.mark.rsync but never invoked rsync".

Strengthened the test's default-directory assertion to positively check that `mngr exec` without `--cwd` runs in the agent's `.mngr/worktrees/my-task-<id>` work_dir, directly validating the tutorial's documented default rather than merely checking the output is not `/tmp`.
