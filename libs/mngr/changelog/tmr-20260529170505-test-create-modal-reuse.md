- Fixed the `test_create_modal_reuse` e2e tutorial test, which failed in environments with multiple agent-type plugins installed because `mngr create sisyphus --reuse --provider modal` could not resolve a default agent type. The test now passes an explicit `--type` to resolve the type unambiguously, mirroring how `test_create_modal_idle_mode_run` already passes `--type command`.
- Strengthened `test_create_modal_reuse` to actually exercise the `--reuse` idempotency contract: it now creates the agent, re-runs the same command, and asserts the second invocation reuses the same agent (identical `agent_id`, with `mngr list` confirming exactly one `sisyphus` agent) rather than creating a duplicate. The test uses the lightweight `command` agent type (a `sleep`) to avoid the SSH-heavy claude provisioning path and is marked `@pytest.mark.flaky` because remote Modal/SSH provisioning can hit transient "SSH protocol banner" failures.
