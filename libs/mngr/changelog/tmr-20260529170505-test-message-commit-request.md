- Fixed the release-marked e2e test `test_message_commit_request` (under `libs/mngr/imbue/mngr/e2e/tutorial/test_git.py`): it carried a spurious `@pytest.mark.modal` mark even though it creates a purely local command agent and never exercises Modal, which tripped the resource-guard check that a modal-marked test must actually invoke Modal. It was also missing the `@pytest.mark.timeout(...)` override that agent-creating tutorial tests need, so pytest's 10s default timeout fired before the command agent finished creating. Removed the modal mark, added `@pytest.mark.timeout(120)`, and raised the per-command run timeout to 120s for the create (shared helper) and the message send.
- Strengthened the test to verify the message was actually delivered rather than only asserting that `mngr msg` exits successfully: it now asserts the CLI confirms delivery to the targeted agent and that the message text appears in the agent's tmux pane (captured via `mngr capture my-task --full`).
