Make the tutorial e2e fixture pin provider discovery to the backends each test actually exercises.

The e2e fixture previously left every registered provider backend enabled. In the dev monorepo that includes credential-requiring cloud backends (e.g. `aws`) shipped as separate packages, which raise `ProviderUnavailableError` during discovery when unconfigured -- so an unscoped `mngr list` recorded that as an error and exited non-zero, by design. The fixture now derives `enabled_backends` from the test's resource markers (always `local`, plus `modal`/`docker` only when the matching `@pytest.mark` is present), mirroring the existing convention that a test touching those providers must carry the corresponding mark. This keeps tutorial commands deterministic regardless of ambient cloud credentials or a running Docker daemon.

Also fixed the `test_jsonl_works_across_commands` output-formats test: removed its incorrect `@pytest.mark.modal` (its `mngr list` only reaches Modal via in-process subprocess gRPC, which the resource guard cannot observe, so the mark tripped the NEVER_INVOKED check), and gave it a slightly larger timeout than the 10s default to account for each `mngr` subprocess loading every installed plugin.
