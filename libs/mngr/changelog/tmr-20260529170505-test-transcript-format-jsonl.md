- Fixed the `tutorial/test_transcript.py` e2e tests. `mngr transcript` only works for agent types that emit a common transcript (claude, via `HasCommonTranscriptMixin`), but the tests were creating a `command`-type sleep agent, so every variant failed with "Agent 'my-task' has type 'command', which does not produce a common transcript." The shared `_create_my_task` helper now creates a claude-typed agent backed by a `sleep` stand-in (real claude cannot run in the e2e harness), bypasses the install check and trust/onboarding dialogs so creation completes headlessly, disables remote providers to keep the flow local and deterministic, and seeds a small `claude/common_transcript/events.jsonl` directly into the agent's event directory. The tests now assert on the actual transcript content (message text, `--role` filtering, `--tail` truncation, and JSONL structure) rather than just the exit code, and carry `@pytest.mark.tmux`/`@pytest.mark.rsync` (used by claude provisioning) plus a longer per-test timeout, without `@pytest.mark.modal`.
