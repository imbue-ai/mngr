- Removed the superfluous `@pytest.mark.modal` mark from the `test_create_from_another_agent` e2e tutorial test. The test creates two local-provider command agents and lists them; it never invokes the Modal CLI (the only path that the Modal resource guard tracks across the `mngr` subprocess boundary is a Modal *host* creation, which this test does not perform). With the mark present, the resource guard's "superfluous mark" check failed the test with "Test marked with @pytest.mark.modal but never invoked modal". The `rsync` and `tmux` marks are retained because they are genuinely exercised (untracked-file transfer during the clone, and the tmux connect command, respectively).
- Strengthened `test_create_from_another_agent` to verify the actual clone effect: it now drops a distinctive untracked file into the source agent's work directory before cloning and asserts (via `mngr exec`) that the file is present in the cloned agent's work directory. Previously the test only checked that the cloned agent existed with a distinct work dir and its own branch, without confirming that the source's working-directory content was actually copied.
