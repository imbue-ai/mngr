The e2e test suite now enables remote provider backends per-test rather than globally. Previously the e2e fixture left every installed provider backend enabled, so any unscoped `mngr list` (or other discovery) probed all of them: the credential-requiring cloud backends (aws, azure, gcp, ovh, vultr, imbue_cloud, imbue_cloud_slice, lima) reported themselves inaccessible and failed the command with a provider-inaccessible exit code, Docker did the same without a running daemon, and Modal blocked on SDK auth. Now the cloud backends no e2e test uses are always disabled, and Modal/Docker are enabled only for tests marked `@pytest.mark.modal` / `@pytest.mark.docker` (which provision them). This keeps discovery in the credential-free e2e environment fast and deterministic.

Removed the superfluous `@pytest.mark.rsync` mark from the `test_list_filter_by_state` release test: it only creates local command agents from a clean working tree, which never invokes rsync (rsync transfers uncommitted "extra files" and only over SSH to remote hosts), so the resource guard flagged the unused mark.
