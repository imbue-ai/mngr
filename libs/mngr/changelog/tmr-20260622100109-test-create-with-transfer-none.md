`mngr list` no longer aborts when an auto-enumerated (unconfigured) provider backend is unreachable. If you have a provider plugin installed but never configured it -- for example the Docker daemon is not running, or a cloud backend has no credentials -- that default provider instance is now skipped during listing instead of failing the whole command. Providers you explicitly configured under `[providers.*]` still surface their errors as before (and still fail loudly under `--on-error abort`).
