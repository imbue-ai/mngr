Hardened the `mngr plugin list --fields` end-to-end tests. The happy-path test now cross-checks the structured JSON output to confirm `--fields` filters the underlying data (not just the rendered table columns), and a new test covers the unhappy path where an unrecognized field name renders the `-` placeholder for every plugin instead of erroring. Also raised the test timeout to match the other subprocess-driven plugin tests so the ~10s mngr cold start no longer trips the default per-test timeout.
