- Removed the `@pytest.mark.modal` mark from the `test_create_with_agent_args` e2e tutorial test (`libs/mngr/imbue/mngr/e2e/tutorial/test_create_basic.py`). The test only creates a local `command` agent and lists it; the Modal interaction it triggers (`mngr list` discovery) happens via the Modal Python SDK inside the `mngr` subprocess, which the resource guard cannot observe (the SDK guard is an in-process monkeypatch, and `mngr list` never invokes the `modal` CLI binary that the PATH-based binary guard would catch). The mark was therefore unsatisfiable and the resource guard failed the test with NEVER_INVOKED. The `tmux` and `rsync` marks remain because those resources are exercised via CLI binaries in the subprocess and are correctly tracked.
- Tightened the same test's verification: it now asserts the agent's stored command equals exactly `sleep 100073 --model opus` instead of checking for two substrings, confirming the args after `--` are passed through verbatim, in order, and with nothing injected.
