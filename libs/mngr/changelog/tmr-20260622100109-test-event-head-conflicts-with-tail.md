Fixed the `test_event_head_conflicts_with_tail` release test, which was failing the resource guard with "marked rsync but never invoked rsync". The `mngr event --head --tail` conflict is rejected during argument validation before any event read happens, and reading events is the only thing that rsyncs a local agent's work dir into place, so the test never invokes rsync. Removed the inaccurate `@pytest.mark.rsync` mark (keeping `@pytest.mark.tmux`, which agent creation still exercises).
