Integrated the qualifying e2e tutorial-test fixes from the 2026-07-23 TMR run and normalized shared setup across the suite.

- The codex agent-type tutorial tests (`test_create_codex_positional`, `test_create_codex_explicit_type`) no longer each disable the codex install check by hand. Since no e2e host has a codex binary or npm, `[agent_types.codex] check_installation = false` is now seeded once in the e2e fixture's `settings.local.toml`, and the per-test `mngr config set` step is removed from both tests. This matches the design intent the positional test's comment already assumed.

- Escalated (not fixed here) a real, verified failure in `test_observe_discovery_pipe_python`: after the `mngr list` cache-warm, `mngr observe --discovery-only` emits no stdout within its 5s window, so its `DISCOVERY_FULL` assertion fails. The legacy global `DISCOVERY_FULL` event is no longer produced (superseded by per-provider `DISCOVERY_PROVIDER`), and the warm-cache fast-path premise appears broken by that migration. Several sibling tutorial tests also still carry stale `DISCOVERY_FULL` docstring prose. A coordinated fix is needed and could not be verified in the integration environment.

- Escalated the suite-wide duplication of per-test `@pytest.mark.timeout` markers (papering over slow cold `mngr` subprocess startup) and of `mngr list --provider local` scoping, both of which are applied by hand test-by-test and would be better expressed as a shared default.
