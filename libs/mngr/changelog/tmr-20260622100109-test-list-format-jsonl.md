Test-only: pinned `enabled_backends` in the e2e tutorial fixture so a fresh, uncredentialed test environment lists cleanly. The cloud VPS backends (aws, azure, gcp, vultr) are registered via plugin entry points but, by design, raise `ProviderUnavailableError` when their credentials are absent; since `mngr list` defaults to `--on-error=abort`, any one of them would abort every fresh-environment listing (exit 1). The fixture now enables only the backends the suite actually exercises (local, ssh, modal, and docker when its daemon is reachable), matching the fixture's documented intent.

Test-only: added a `@pytest.mark.timeout(60)` override to `test_list_format_jsonl` (the single full-discovery `mngr list` it runs makes an authenticated Modal SDK lookup that routinely exceeds the default 10s per-test timeout), and added `test_list_format_jsonl_with_agents`, which populates the listing with two local agents and verifies the JSONL contract (one standalone JSON object per line, distinct from the single array `--format json` emits).
