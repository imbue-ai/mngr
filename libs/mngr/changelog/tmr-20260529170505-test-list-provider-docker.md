Fixed the `test_list_provider_docker` e2e tutorial test so it actually exercises Docker. `mngr list --provider docker` only talks to the Docker daemon through the in-process SDK (never the `docker` CLI binary), so on an empty environment the test never invoked Docker and the resource guard failed the `@pytest.mark.docker` test. The test now first creates a Docker agent (which shells out to the `docker` CLI for build/run) and then verifies the agent appears in `mngr list --provider docker`.
