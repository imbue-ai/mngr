Fix the flaky `test_list_watch_mode` e2e release test. The test wraps the tutorial's `watch -n5 mngr list` in a `timeout` so it doesn't block forever, then asserts the wrapped `mngr list` output ("No agents found") appears in watch's rendered frame. The old 8-second window was too short: `watch` only paints a frame after the wrapped command exits, and a full `mngr list` provider discovery routinely takes ~10s, so watch entered and left the alternate screen without ever rendering the list. The window is now 30s (with the e2e per-command and per-test timeouts raised to match), giving discovery generous headroom to complete and render at least one frame. The test also now asserts that watch's own header (`Every 5.0s: mngr list`) appears, confirming the listing was produced by watch at the `-n5` interval rather than by a bare `mngr list`.
