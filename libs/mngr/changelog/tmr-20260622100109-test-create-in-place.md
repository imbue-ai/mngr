Scoped the `mngr list` verification in the `test_create_in_place` e2e tutorial test to `--provider local`. The in-place agent is a local agent, and `mngr list` defaults to `--on-error abort`, so an unconfigured cloud backend (aws/azure/gcp) present in the environment would otherwise abort the listing and fail the test. This matches the existing `--provider local` pattern used by the agent-types tutorial tests.
