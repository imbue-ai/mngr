Made the e2e test fixture pin provider discovery to the backends each test declares via its resource marks (`local` always, plus `docker`/`modal` when marked). Previously every provider plugin installed in the test venv was discoverable, so a plain `mngr list` could exit non-zero when an unused backend was unreachable (cloud backends without credentials, or docker with no daemon running). This makes the e2e release suite deterministic across sandboxes with differing daemons and credentials.
