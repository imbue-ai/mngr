Fixed the `test_advanced_create_reuse_modal` e2e release test so its agent-ID lookups scope discovery to the modal provider (`mngr list --provider modal ...`). The agent under test lives on modal, but the unscoped `mngr list` queried every configured provider and exited non-zero when an enabled-but-unconfigured provider (e.g. AWS) could not be reached, failing the test even though the modal agent was found correctly.
