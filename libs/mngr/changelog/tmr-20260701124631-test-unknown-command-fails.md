Raised the per-test timeout on the `test_unknown_command_fails` e2e release test. Running an unknown `mngr` subcommand is rejected at argument-parsing time with usage exit code 2, but the CLI's module import (which loads every provider/agent plugin, including the google-cloud, azure, and anthropic SDKs) routinely approaches the global 10s pytest timeout on its own. The test now allows 120s, matching the other parse-time rejection tests, so it no longer flakes on CLI startup cost while still verifying the documented behavior (exit code 2, empty stdout, and a stderr error that names the offending command and points the user back to `mngr --help`).
