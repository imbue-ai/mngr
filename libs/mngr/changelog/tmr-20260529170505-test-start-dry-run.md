- `mngr start` now supports `--dry-run`, which reports the stopped agent(s) that would be started (one per line, with their host) without actually starting anything. This matches the dry-run behavior already offered by `mngr archive`, `mngr cleanup`, `mngr gc`, and `mngr rename`, and makes the documented `mngr list --ids | mngr start - --dry-run` tutorial command work as described.
- Strengthened the `test_start_dry_run` e2e test to stop the target agent first (so the dry-run has a deterministic stopped candidate to report), assert the dry-run output actually names the agent it would start, and verify the dry-run does not start the agent (it remains stopped afterward).
