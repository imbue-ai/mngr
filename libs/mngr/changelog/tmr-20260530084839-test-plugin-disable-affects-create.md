Fixed the `test_plugin.py` e2e release tests. Added `@pytest.mark.timeout(120)` so slow `mngr` cold-start invocations don't trip the global 10s function timeout, and seeded the project-scope `settings.toml` in the e2e fixture with `is_allowed_in_pytest = true`. The latter fixes a false positive: `mngr plugin disable --scope project` writes that file, and without the opt-in every subsequent `mngr` command failed the pytest config guard (making `test_plugin_disable_affects_create` pass for the wrong reason and breaking `test_plugin_disable_enable_roundtrip`). Strengthened `test_plugin_disable_affects_create` to assert the create failure is specifically due to the disabled plugin and that the cleanup re-enable succeeds.
