Fixed the `test_plugin_disable_affects_create` e2e release test so it disables and re-enables the `claude` plugin at user scope (`--scope user`, as the tutorial demonstrates). Disabling at the default project scope created a fresh project `settings.toml` lacking the `is_allowed_in_pytest` opt-in, which made the subsequent `mngr create` fail on the pytest config guard instead of because the plugin was disabled. Writing at user scope merges into the already-opted-in profile config, so the test now exercises the real plugin-disabled create failure.
