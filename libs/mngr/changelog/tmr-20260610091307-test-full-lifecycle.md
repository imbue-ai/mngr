Scoped the `mngr list` verification calls in the `test_full_lifecycle` e2e release test to `--provider local`, matching the existing pattern used elsewhere in the e2e suite. The agent under test runs on the local provider, so an unscoped `mngr list` (which fans out to every enabled provider) needlessly made the test abort with exit code 1 whenever a remote provider such as Docker was unreachable.
