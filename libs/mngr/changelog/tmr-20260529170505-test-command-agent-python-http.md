- Fixed the release-marked e2e test `test_command_agent_python_http` (under `libs/mngr/imbue/mngr/e2e/tutorial/test_command_agents.py`): it was missing the `@pytest.mark.timeout(...)` override that every other agent-creating tutorial test carries, so pytest's 10s default timeout fired before the local command agent finished creating. It also carried a spurious `@pytest.mark.modal` mark even though it creates a purely local command agent, which tripped the resource-guard check that a modal-marked test must actually exercise modal. Added a 120s timeout (matching sibling create tests) and removed the modal mark.
- Strengthened the test to verify the managed process actually runs: it now asserts the configured `sleep` process is alive inside the agent (via `mngr exec`) and that the agent appears in `mngr list` JSON output with the expected command, rather than only asserting that `mngr create` exits successfully.
