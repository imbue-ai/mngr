Removed a superfluous `@pytest.mark.rsync` mark from the `test_create_with_label` e2e tutorial test. The test creates a local command agent, which is a local-to-local operation that never invokes rsync, so the resource guard was failing the test with a "marked with @pytest.mark.rsync but never invoked rsync" error. The mark is also outside the test's documented scope (agent labels, host tags, and label-based filtering).
