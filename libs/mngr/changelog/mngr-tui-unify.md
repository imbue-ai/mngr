Sending a message to an agent whose tmux server is gone now fails immediately instead of after the full submission timeout. For a Claude agent -- which confirms a submission from either its submit hook or a freshly recorded acceptance marker -- an unreachable tmux used to be reported only once the marker had been polled to the end of the 90-second deadline, even though the failed hook waiter already knew at once. Measured against a `tmux` that exits non-zero: 91.7s before, 0.0s after.

The submission command also no longer leaves a `tmux wait-for` client running after it returns. Whenever the acceptance marker confirmed a submission -- the normal case for Claude, since the marker is recorded the instant the message is queued while the hook may only fire once it reaches the model -- the hook waiter was left behind, because it ran inside a subshell and the cleanup killed the subshell rather than the `tmux` client inside it. It then sat on the channel until its own deadline expired, up to 91 seconds later. Measured: one leftover client per marker-confirmed submission before, zero after.

Both come out of the same restructuring. The waiter now runs as a direct background job, so the cleanup can reach the `tmux` client itself, and the poll loop reads the waiter's exit status rather than a file it writes on success -- which is what lets an outright tmux failure end the wait at once. That in turn removes the reason the command needed an external deadline: it used to bound the waiter with `timeout(1)`, or with a perl `Time::HiRes` alarm on hosts that ship no `timeout` (stock macOS), purely because nothing else could stop it. The deadline is now a loop in the script, so a submission needs only bash builtins plus `tmux`, `date` and `sleep` -- no `timeout`, no perl, no temp file.

The two submission-command builders are now one. Waiting on the hook alone is the same command with an acceptance-marker probe that prints an empty token and so can never confirm, leaving the hook as the only thing that can. Agents that watch an acceptance marker (Claude) and agents that do not (codex, antigravity) are unchanged in behavior, except that a hook-confirmed submission for the latter now returns up to one poll interval (0.25s) later than before, against a submission budget of 15-90 seconds.
