Changed: `mngr gc` no longer aborts its work-directory sweep for a host that is classified online but is briefly unreachable (for example a Lima VM that reports `Running` while its guest sshd is still coming up, or a transient network blip). The work-dir and source-dir phases now skip such a host and continue, matching the deliberate skip the idle-machine phase already does -- previously only the offline/auth error subclasses were caught, so a bare connection error propagated and could abort the whole sweep.

Added: `OfflineHost` now accepts an optional provider-supplied `state_override`, letting a provider surface a transient live state that persisted data cannot express (used by the Lima provider to report a still-booting VM as `STARTING` rather than the `CRASHED` a stop-reason-less record would otherwise derive). Also corrected a stale comment on the `mngr start` path describing the host-start routing.
