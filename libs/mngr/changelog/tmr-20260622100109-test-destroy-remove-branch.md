Fixed the `test_destroy_remove_branch` e2e tutorial test (`mngr destroy --force --remove-created-branch`). Branch deletion runs after the default post-destroy garbage-collection pass, which scans every provider (including Modal) over the network and routinely exceeds the default 30s command timeout, so the destroy command now gets a longer timeout and the test-level timeout was raised to match. Also removed the test's superfluous `@pytest.mark.rsync` mark, since a localhost git-worktree agent is created and destroyed without ever invoking rsync.
