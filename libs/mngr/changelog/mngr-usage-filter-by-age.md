CLI docs (`libs/mngr/docs/commands/secondary/usage.md`, regenerated via `scripts/make_cli_docs.py`): reflect the `mngr usage` help-output regrouping in `imbue-mngr-usage`. `--max-age` and `--detail` now appear under a `## Display` section; `--since` and `--preserved/--no-preserved` now appear under `## Filtering` alongside the agent-filter flags. The old `## Other Options` section is gone. The rendered synopsis is updated to `mngr usage [--max-age DURATION] [--detail] [--since DURATION] [--no-preserved] [COMMAND]`.

`test_synopsis_lists_all_non_optout_flags` (in `libs/mngr/imbue/mngr/cli/help_formatter_test.py`): no longer silently skips commands whose synopsis is a placeholder like `[OPTIONS]`. Such commands' custom non-Common flags are now reported as missing from the synopsis (and the author must either enumerate them or add them to `_SYNOPSIS_OPTOUT_FLAGS`). Pre-existing placeholder-synopsis commands (`connect`, `gc`, `kanpan`, `list`, `snapshot.create`, `snapshot.list`, `snapshot.destroy`, plus `mngr usage` itself) are captured as baseline opt-outs; promoting any of them to an enumerated synopsis is a fine follow-up. Factored out a shared `_AGENT_FILTER_FLAGS` constant covering the 11 flags injected by `add_agent_filter_options`, used by both new and existing opt-out entries.
