Fixed the `test_config_set_unknown_key_fails` e2e test. It incorrectly assumed the e2e fixture pre-seeds the project-scope `settings.toml`, so it asserted that the file could be read after a rejected `config set`. The fixture only seeds `settings.local.toml`, and `config set` validates the key before writing, so a rejected set leaves the project file uncreated. The test now asserts the project `settings.toml` was never created, which is the genuine evidence that the invalid value was not persisted.
