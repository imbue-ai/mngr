Fixed a set of over-defensive edge-case handlers surfaced by the `identify-suspicious-edge-cases` skill, in three areas:

- config: empty/whitespace `backend`/`plugin` config values are no longer silently replaced by the section name; an untraversable `__extend` base path now raises instead of silently degrading to a plain assign; the `completion_writer` cache write no longer swallows unrelated `OSError`s from the build phase; and `model_construct` tuple-field normalization now rejects unrecognized types (the prior "Pydantic will validate" comment was wrong).
- utils: `count_commits_between` now raises instead of returning a `0` that was indistinguishable from the valid answer; `get_editor_command` raises instead of returning a known-absent `vim`; CEL filtering no longer catches broad `TypeError` (which masked real bugs as the wrong filter result); `LoggingConfig.merge_with` now merges on the set fields (its prior `is not None` guards never fired and clobbered unset fields with defaults); ANSI truecolor sequences render correctly; and `install_hint`/`read_json_dict` no longer silently mishandle unknown platforms / non-object JSON.
- hosts: `get_uptime_seconds` and `get_ssh_connection_info` now return `None` (typed `float | None` / `Path | None`) instead of misleading `0.0` / `Path("")` sentinels; submodule detection no longer swallows connection failures; env-file values are quoted with `shlex.quote`; and several missing/corrupt agent-data paths now raise instead of fabricating defaults. The SSH-key return-type change threads `Path | None` through `SSHInfo`/`RemoteSSHInfo` and their callers, and the `host.ssh.key_path` CEL field is documented in `list.md` as nullable (None when the host has no mngr-owned key).
