- Fixed the e2e test fixture so `mngr config set` (which defaults to project scope) works under the per-file pytest config guard: the fixture now also writes a project-scope `settings.toml` with `is_allowed_in_pytest = true`, alongside the existing `settings.local.toml`. Without this, any e2e test that runs `mngr config set` followed by another `mngr` command (e.g. `test_create_codex_agent`) failed because the freshly written `settings.toml` did not opt into pytest. Test-only change; no user-visible behavior change.
