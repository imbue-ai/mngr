- Fixed the release e2e test `test_start_multiple_agents` (in `libs/mngr/imbue/mngr/e2e/tutorial/test_lifecycle.py`): it created and started three command-type agents but only had the default 10s pytest timeout, so it timed out while still creating the agents. - Strengthened the test to verify the actual restart behavior: it now stops all three agents before running `mngr start agent-1 agent-2 agent-3`, then parses `mngr list --format json` to assert each agent is back in a running state rather than only checking the start command's exit code.
