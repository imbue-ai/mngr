Added the agent capability registry (`imbue.mngr.agents.agent_capabilities`): a code-derived description of which agent types have which capabilities. Each capability declares how its presence is detected (a class mixin via `issubclass`, a `waiting_reason` field generator, a plugin hookimpl, or a sibling usage plugin), and a matrix of capability x agent type is rendered directly from the registered agent classes -- the basis for replacing the hand-maintained parity matrix and guarding against doc/code drift.
