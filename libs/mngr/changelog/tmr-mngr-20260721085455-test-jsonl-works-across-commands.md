Removed the spurious `@pytest.mark.modal` mark from the `test_jsonl_works_across_commands` e2e tutorial test (the `mngr list --format json && mngr plugin list --format jsonl` block). `mngr list` is a read-only path that reaches Modal only via in-process gRPC inside the `mngr` subprocess (which the resource guard's in-process Modal SDK monkeypatch cannot observe) and never shells out to the `modal` CLI binary, and `mngr plugin list` does not touch Modal at all, so the mark tripped the resource guard's NEVER_INVOKED check and failed the otherwise-passing test. Added an explanatory NOTE and an extended `@pytest.mark.timeout(180)` (provider discovery is slow when Modal credentials are present), matching the sibling `test_list_format_jsonl_recap`/`test_jsonl_with_jq_stream` tests.
