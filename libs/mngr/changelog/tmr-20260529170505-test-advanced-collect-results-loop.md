- Fixed the "collect results from all agents" advanced-workflow block in `mega_tutorial.sh`: `mngr exec` takes the agent name(s) followed by a single quoted `COMMAND`, so the command must be quoted (`mngr exec "$agent" "git log --oneline -3"`). The previous unquoted `-- git log --oneline -3` form failed with `Not a valid agent name or ID: '--oneline'` because the extra tokens were parsed as additional agent names.
- Hardened the corresponding e2e test (`test_advanced_collect_results_loop`): added a `@pytest.mark.timeout(180)` so the four-agent create/exec sequence does not hit the default 10s timeout, dropped the superfluous `@pytest.mark.modal` mark (the test exercises only local command agents and never invokes modal), and added assertions that each agent's `git log` was actually collected (per-agent header, "Command succeeded", and the repo's initial commit) instead of relying on a `|| true`-masked loop exit code.
