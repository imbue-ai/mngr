Made the `test_create_aborts_on_dirty_tree_by_default` e2e tutorial test robust against unconfigured remote providers. Its post-abort verification now scopes the listing to the local provider (`mngr list --provider local --format json`) -- the only place a default-provider create could have placed the agent -- so the check no longer fails when an unrelated, unconfigured provider (e.g. AWS) is registered but unreachable in the test environment and would otherwise abort the default `--on-error abort` listing.
