Fixed the `test_list_compound_cel` e2e tutorial test (LABELS AND FILTERING section). The test exercises `mngr list` with a compound CEL filter (`labels.team == "backend" && state == "RUNNING"`). It previously timed out under the default 10s pytest signal timeout because `mngr list` performs provider discovery (including a read-only Modal probe) that takes longer, and it carried a superfluous `@pytest.mark.modal` mark that the resource guard could never satisfy (a list-only command performs only a read-only Modal SDK discovery probe in the `mngr` subprocess and never invokes the `modal` CLI binary, which is the only Modal usage the guard can detect across the subprocess boundary). Added `@pytest.mark.timeout(120)` and removed the unsatisfiable `@pytest.mark.modal` mark.

Also added a companion test (`test_list_compound_cel_excludes_non_matching`) that shares the same tutorial block but verifies the actual filtering behavior: it creates a real (non-matching) agent and asserts the compound CEL expression excludes it from the filtered listing while it is still present in an unfiltered listing.
