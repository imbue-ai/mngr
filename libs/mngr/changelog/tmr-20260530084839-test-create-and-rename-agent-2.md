Fixed the `test_create_and_rename_agent` e2e release test in `test_rename.py`: removed an incorrect `@pytest.mark.modal` marker. The test creates a local `--type command` agent and only runs `mngr rename`, `mngr list`, and `mngr exec`, none of which invoke the Modal CLI binary (e2e tests drive `mngr` as a subprocess that uses the Modal Python SDK, which the PATH-wrapper resource guard cannot observe). The mark therefore tripped the resource guard's "marked with @pytest.mark.modal but never invoked modal" check.
