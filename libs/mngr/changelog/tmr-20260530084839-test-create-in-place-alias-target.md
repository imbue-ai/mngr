Removed the incorrect `@pytest.mark.modal` from the `test_create_in_place_alias_target` e2e release test. The test creates a local in-place agent (`--transfer=none --type command`) and only runs `mngr list` for discovery; it never creates a Modal host, so it never invokes the Modal CLI (`modal environment create` / `modal deploy`). The mark therefore tripped the resource guard's "marked but never invoked" check. This is a test-only change with no user-visible behavior impact.
