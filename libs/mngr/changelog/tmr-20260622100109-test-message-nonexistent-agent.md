Fixed a flaky-by-design timeout in the `test_message_nonexistent_agent` e2e tutorial test. Messaging a bare nonexistent agent name forces a full discovery scan across every enabled provider (local, Docker, Modal, ...) to prove the name matches nothing, which reliably exceeds the default 10s pytest and 30s subprocess timeouts. The test now allows the same generous headroom as the other multi-provider message tests.
