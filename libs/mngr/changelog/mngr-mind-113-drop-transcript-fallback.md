`mngr transcript` (human format) now renders an assistant turn from the ordered `parts[]` only, with no fallback to the flat `text`/`tool_calls`. The fallback was a back-compat shim for agents running a common-transcript emitter that predated `parts[]` (before 2026-06-15); those agents have since aged out, so the reader is back to a single code path. Every current emitter fills `parts[]`, so this does not change what any live agent's transcript renders.
