- Fixed the release e2e test `test_connect_explicit_host` (the `mngr conn my-task@my-host` tutorial block). It was failing two ways: the `agent@host` error path never attaches a tmux session, syncs, or reaches the Modal SDK in a fresh env, so the `@pytest.mark.tmux`/`rsync`/`modal` resource guards reported "never invoked"; and the full provider scan plus mngr's cold start could edge past the default 10s function timeout. Removed the inapplicable resource marks (matching the sibling error-path test `test_connect_by_agent_id_fictional`), added `@pytest.mark.timeout(120)`, and strengthened the assertion to verify the clean `No hosts found matching my-host` error instead of merely a non-zero exit code.
