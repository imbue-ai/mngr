Test-only: raised the per-test timeout on the connect e2e test `test_connect_explicit_host_and_provider` from the default 10s to 120s. The provider-qualified `mngr conn my-task@my-host.modal` syntax narrows discovery to the Modal provider, which loads the Modal SDK and queries it for matching hosts; that work pushes the command just past the default 10s timeout. The test stays unmarked by `@pytest.mark.modal` (the SDK usage happens in the e2e subprocess and never invokes the `modal` CLI, so the resource guard cannot observe it).
