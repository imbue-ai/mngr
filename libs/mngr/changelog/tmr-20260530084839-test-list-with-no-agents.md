Removed the spurious `@pytest.mark.modal` from the `test_list_with_no_agents` e2e release test. In a fresh environment with no agents, `mngr list` never creates a Modal environment and discovers agents via the Modal Python SDK, so it never invokes the guarded `modal` CLI binary. The mark therefore tripped the resource guard's "marked with @pytest.mark.modal but never invoked modal" check and caused the test to fail. Added an explanatory comment so the mark is not reintroduced.
