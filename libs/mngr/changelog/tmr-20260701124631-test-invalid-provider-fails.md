Gave the `test_invalid_provider_fails` e2e release test more timeout headroom. It runs two full `mngr` invocations back-to-back (the failing create plus a follow-up `mngr list`), and mngr's cold-start alone approaches the default 10s per-test timeout, so the pair reliably exceeded it even though each command returns promptly. The test now carries an explicit `@pytest.mark.timeout(120)` so it verifies the invalid-provider behavior without spuriously timing out.
