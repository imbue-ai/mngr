Widened the timeout windows in the `mngr observe --discovery-only` tutorial e2e test (`test_advanced_observe_stream`). The initial `DISCOVERY_FULL` snapshot is only emitted after the first full discovery sync finishes, which must wait for every configured provider -- including unconfigured cloud backends (azure/aws/gcp) that take ~10-15s to fail their availability checks. Combined with mngr's cold start, the snapshot lands ~20-25s after launch, so the previous 20s window killed the stream before the documented snapshot was ever written.
