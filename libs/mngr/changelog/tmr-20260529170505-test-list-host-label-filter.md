- Fixed the release-marked e2e test `test_list_host_label_filter` (in `libs/mngr/imbue/mngr/e2e/tutorial/test_list.py`). It previously ran `mngr list --host-label ENV=staging` against an empty environment, which (a) exceeded the global 10s per-test timeout while contacting Modal and (b) tripped the `@pytest.mark.modal` resource guard because a no-agent `mngr list` reaches Modal only via in-process gRPC in the spawned subprocess, which the guard cannot observe (it only tracks the `modal` CLI binary across processes). The test now creates a Modal command agent whose host is labeled `ENV=staging`, then asserts that `mngr list --host-label ENV=staging` includes it and that `mngr list --host-label ENV=production` excludes it -- verifying the actual filtering behavior and exercising the Modal discovery path the mark expects. Added `@pytest.mark.rsync` and `@pytest.mark.timeout(300)` to match the cost of creating a real Modal host.
