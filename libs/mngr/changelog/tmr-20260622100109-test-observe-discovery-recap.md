Fixed the `test_observe_discovery_recap` release test for `mngr observe --discovery-only`. The test previously assumed the first JSONL line emitted on a fresh run was the full discovery snapshot, but a fresh run emits provider-discovery failures (e.g. for unconfigured cloud providers) as `DISCOVERY_ERROR` lines first. The test now greps the stream for the `DISCOVERY_FULL` snapshot line and asserts on its documented JSON shape (a `DISCOVERY_FULL` object carrying `agents`/`hosts` arrays) rather than matching only the first line.
