Fixed `mngr transcript` failing with "Unknown agent type" for config-defined agent subtypes (a custom `[agent_types.X]` with a `parent_type`, e.g. a `write-plus` template whose `parent_type` is `claude`). The command now resolves the type through its parent chain (like every other command) instead of a flat class-registry lookup, so it correctly recognizes such subtypes and reads their transcript.
