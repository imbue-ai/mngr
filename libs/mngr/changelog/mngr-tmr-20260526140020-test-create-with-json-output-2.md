## mngr e2e

- `test_create_with_json_output` (libs/mngr/imbue/mngr/e2e/test_create_basic.py): replace `@pytest.mark.modal` with `@pytest.mark.timeout(120)`. The default 10s pytest timeout caused the test to fail mid-run when invoked directly (e.g. by the TMR test agent); release tests use offload's `--timeout=900` override in CI, but direct invocation needs an explicit per-test timeout. The `modal` marker was also a false positive: the test only creates a local-provider agent and `mngr list` skips Modal entirely when no Modal environment exists, so the modal binary is never invoked during the call phase, which made the resource-guard's "marked but never invoked" check fail.
- Cross-check the `agent_id`/`host_id` returned by `mngr create --format json` against the corresponding entries in `mngr list --format json`, and assert the listed agent's `type` and `state` -- the previous assertions only verified the IDs were present in the create output and that the name appeared in the list, which would not have caught e.g. a create command that returned IDs unrelated to the agent it actually started.
