The config loader now lets an agent-type config class resolve its own deprecated/renamed keys to their canonical fields before the block is parsed, via a new `AgentTypeConfig.normalize_deprecated_raw_config` hook (a no-op on the base class; agent-type plugins override it). Running this on the raw config -- before `model_construct` -- means a renamed key becomes its canonical field in `model_fields_set` from the start, so it survives the config-layer and `parent_type` overlay merges (which propagate only set fields and otherwise let a defaulted field resurface). This is used by the claude plugin to keep its deprecated `use_env_config_dir` alias working on the in-host `mngr create` path.
