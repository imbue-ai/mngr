Hardened the `test_advanced_fan_out_create` e2e tutorial test so its process-liveness check can no longer pass spuriously. The check previously ran `pgrep -f 'sleep 101010'`, which self-matched the invoking `bash -c` shell (whose own command line contained the pattern) and so could report a match even when no agent process was actually running. The pattern is now anchored (`^sleep 101010`), matching only the real agent `sleep` processes and never the shell that launched pgrep, so the test truly verifies the fan-out loop launched each agent's command rather than merely registering agent state.
