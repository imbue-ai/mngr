Fixed the `test_snapshot_list` e2e release test for `mngr snapshot list -`. The test previously had no `@pytest.mark.timeout` override (so it inherited the global 10s default and was killed before the piped `mngr` commands could finish) and never created a modal agent (so it tripped the resource guard for being marked `@pytest.mark.modal` without exercising modal). It now creates a modal agent, runs `mngr list --ids | mngr snapshot list -` with an adequate command timeout, and verifies the agent's automatic "initial" snapshot appears in the listing.
