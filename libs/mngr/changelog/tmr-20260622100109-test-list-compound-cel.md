Disable the unconfigured cloud provider plugins (aws, azure, gcp, ovh, vultr, lima) in the e2e test fixture.

These provider backends register entry points whenever their packages are installed in the workspace, so every read path (e.g. `mngr list`) queried them even though the e2e environment has no credentials or CLIs for them. Several of these backends (aws, azure, gcp) intentionally surface a missing session as a `ProviderUnavailableError`, which made `mngr list` exit non-zero and broke the LABELS-section tutorial e2e tests (e.g. `test_list_compound_cel`). The e2e fixture now disables these providers via `is_enabled = false`, matching a realistic user setup where only configured providers are enabled and keeping the discovery path to the providers the suite can actually exercise (local, Docker, Modal).
