Regenerated the `mngr aws` / `mngr azure` CLI doc pages to cover the state-bucket setup these commands now perform (the providers' state-bucket feature is described in the `mngr_aws` / `mngr_azure` changelogs).

`TagLimitExceededError` now accepts an optional `actual` count (defaulting to omitted) and an optional `remediation` string appended to the message, so providers can surface actionable guidance (e.g. "run `mngr aws prepare`") when a host exhausts the provider's tag mirror.

Test-only: raised the per-test timeout on the tmux lifecycle tests `test_start_restart_running_agent` / `test_start_restart_stopped_agent` from the default 10s to 30s (they run several sequential tmux create/stop/restart operations that can exceed 10s on a loaded CI runner).

Reworked the shared operator-command output in `mngr.cli.output_helpers`. `emit_operator_result(event_name, parts, output_format)` now takes a sequence of `OperatorResultPart`s -- each pairing a structured-data fragment with its matching human line -- and is the single place the JSON / JSONL / human format is matched, so no provider call site enumerates the formats and the machine and human renderings of a fact are authored together (no separate `human_lines`). Build parts with `OperatorResultPart.shown(human, **data)` (line always prints) or `OperatorResultPart.shown_if(present, human, **data)` (line prints only when an optional resource is present; its fields are emitted either way). Also extracted a `write_event_line` primitive for the `{"event": <type>, ...payload}` JSONL shape shared by every JSONL emitter in that module (events, info, errors, operator results).
