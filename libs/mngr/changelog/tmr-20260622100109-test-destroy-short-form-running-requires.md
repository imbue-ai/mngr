Restricted the e2e tutorial test profile to the provider backends the suite can actually reach (local/ssh/modal, plus docker only when a Docker daemon is reachable). Previously, credential-requiring cloud plugins (aws, azure, gcp, vultr) installed without credentials -- or an enabled-but-unreachable Docker backend -- made their default provider instances raise errors during discovery, causing `mngr list` (and the destroy/gc assertions that follow it) to exit non-zero across the e2e tests even though no test uses those providers.

Dropped the incorrect `@pytest.mark.rsync` from the destroy test that exercises the running-agent guard (`mngr rm` without `--force`): that unhappy path never completes a destroy and creates only a local command agent, so it never invokes rsync, and the resource guard fails a test that declares a mark it does not exercise.
