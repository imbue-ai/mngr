Made the `test_create_command_python_http` e2e tutorial test robust to unavailable providers by scoping its verification `mngr list` to the local provider (`--provider local`), matching the sibling codex/yolo tests. The command agent lives on the local provider, so a full provider fan-out would exit non-zero whenever an unrelated enabled provider is inaccessible (e.g. `aws` without credentials, or an unreachable Docker daemon), which is orthogonal to what the test verifies.
