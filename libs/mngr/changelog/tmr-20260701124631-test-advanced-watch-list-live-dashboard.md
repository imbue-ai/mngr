Fixed the e2e test suite's provider setup so an unfiltered `mngr list` no longer aborts on unconfigured cloud providers. The dev checkout installs every provider plugin (aws, azure, gcp, ...), so each was enabled by default in the e2e environment; without credentials, `mngr list` (which defaults to abort-on-error) exited with the provider-inaccessible code even though it printed the correct agent listing. The e2e profile now disables the credential-requiring VPS provider backends (aws, azure, gcp, imbue_cloud, ovh, vultr), leaving local, docker, and modal -- the providers the suite actually exercises -- enabled.

Also removed a spurious `@pytest.mark.rsync` mark from `test_advanced_watch_list_live_dashboard`: the test only creates a local-provider agent and lists it, so it never invokes rsync.
