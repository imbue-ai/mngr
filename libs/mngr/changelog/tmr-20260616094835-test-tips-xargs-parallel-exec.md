Made the `test_tips_xargs_parallel_exec` release test robust to dev environments that have extra cloud-provider plugins (e.g. aws/gcp/vultr) installed and enabled-by-default but without credentials. In that case `mngr list --ids` correctly exits non-zero (a provider is unreachable) while still printing the modal host id. The test now asserts that the modal host id is discovered and that the `xargs` pipeline actually exec'd on it, rather than requiring `mngr list` to exit 0 -- matching how the documented `mngr list ... | xargs` pipeline behaves for a real user (the pipe ignores `mngr list`'s exit code).
