Fixed the `test_connect_by_agent_id_fictional` e2e tutorial test, which was timing out. Connecting to a non-existent agent id forces a full multi-provider discovery scan (the id cannot be resolved via the event-stream optimization), which takes longer than the default 10s per-test timeout. The test now carries an explicit `@pytest.mark.timeout(120)` so the slow-but-correct "agent not found" path has room to complete.
