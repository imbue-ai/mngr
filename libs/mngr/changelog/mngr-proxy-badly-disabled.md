Added a reusable `check_path_gitignore_status` helper (with a `GitignoreStatus` result) to `mngr.api.git`. Given a host, a repo path, and any repo-relative path, it reports whether that path is gitignored -- resolving a symlinked leading path component (e.g. `.claude -> .agents`) first so `git check-ignore` doesn't choke. Plugins use it to guard files they write into an agent worktree against showing up as untracked changes.
