Added a reusable `check_path_gitignore_status` helper (with a `GitignoreStatus` result) to `mngr.api.git`. Given a host, a repo path, and any repo-relative path (which need not exist yet), it reports whether that path is gitignored -- resolving symlinks anywhere along the path (e.g. `.claude -> .agents`) first so `git check-ignore` doesn't choke with "beyond a symbolic link". Plugins use it to guard files they write into an agent worktree against showing up as untracked changes.
