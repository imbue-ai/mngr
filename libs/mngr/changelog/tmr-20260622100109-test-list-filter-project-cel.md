Fixed the `test_list_filter_project_cel` PROJECTS tutorial e2e test. It previously inherited the default 10s pytest timeout (too short for the mngr/plugin cold-start) and ran an unscoped `mngr list`, which aborts with a non-zero exit when any of the fixture's enabled-but-unconfigured cloud provider backends is unreachable. The test now scopes discovery to the always-available local provider (still exercising the `--include` CEL filter path) and allows a longer timeout, making it deterministic across environments.

Added `test_list_filter_project_cel_discriminates`, which seeds a local agent and verifies that the `--include` CEL filter genuinely discriminates by project (a matching expression includes the agent, a non-matching one excludes it) rather than merely being accepted.
