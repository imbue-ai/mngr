Plugins can now contribute standalone help topic pages via the new `register_help_topics` hook. Topics from an installed plugin show up in `mngr help` and are viewable via `mngr help <topic>`, just like mngr's built-in topics. Implement the hook to return `TopicHelpPage` objects (for aliases, "See Also" references, and inline content), or use the new `build_topics_from_directory` helper to expose a directory of markdown files as topics (one per file). Plugin topics that collide with a built-in topic key are skipped so built-in topics always win.

Internally, the topic data model, registry, and directory-scanning helper moved to a new lightweight `imbue.mngr.cli.help_topics` module so plugins can import them without pulling in the rest of the CLI (they remain importable from `imbue.mngr.cli.help` for backwards compatibility). mngr's own built-in topics are now registered through this same hook (as a built-in plugin), the same way built-in provider backends and agent types register through their hooks.
