Strengthened the `test_create_modal_custom_image_base` release test so it actually proves the `-b image=python:3.12` build arg took effect. The test now asserts that create reports installing required host packages at runtime ("not pre-installed in the base image"), which only happens for a user-supplied registry base image. Previously the test relied solely on `python --version` reporting 3.12, but mngr's default base image resolves its Python version to the local interpreter (also 3.12 in the test environment), so that check would have passed even if the custom-image flag were silently ignored.
