Removed the unused `@pytest.mark.rsync` mark from the `test_connect_no_start` e2e tutorial test. A local agent is created via a git worktree (the default transfer mode for a same-host git repo) and `mngr connect` simply execs `tmux attach`, so rsync is never invoked. The resource guard fails any passing test that declares a resource it never uses, so the stale mark caused the test to fail despite the connect behavior working correctly.
