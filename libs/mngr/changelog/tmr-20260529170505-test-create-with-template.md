- Removed the superfluous `@pytest.mark.modal` from the e2e release test `test_create_with_template` (`libs/mngr/imbue/mngr/e2e/test_config.py`). The test creates a local in-place agent (`transfer=none`) and verifies via `mngr list`; neither step invokes the `modal` CLI binary, and the Modal SDK guard's monkeypatch is only active in the pytest process, never in the `mngr` subprocess. The mark therefore always tripped the resource guard's "marked with @pytest.mark.modal but never invoked modal" check.
- Strengthened `test_create_with_template` to also confirm in-place execution at runtime via `mngr exec my-task pwd` (matching the agent's actual cwd against the reported `work_dir`), and added an unhappy-path test `test_create_with_nonexistent_template` that asserts `mngr create --template no_such_template` fails with a "not found" error and leaves no agent behind.
