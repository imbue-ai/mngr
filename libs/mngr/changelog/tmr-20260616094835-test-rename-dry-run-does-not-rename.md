Fixed the `test_rename_dry_run_does_not_rename` e2e release test so it no longer depends on the runner's provider configuration. The post-rename verification now scopes `mngr list` to `--provider local` (the agent is created in-place locally, so other providers have nothing to contribute and an unconfigured provider such as AWS would otherwise abort the default `--on-error abort` listing). Also removed the superfluous `@pytest.mark.rsync` mark: the local git-worktree create path uses git, not rsync, so the test never invokes rsync.
