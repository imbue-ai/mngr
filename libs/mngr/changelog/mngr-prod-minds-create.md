Fixed a bug where a long-running `mngr observe` discovery stream could never see a host that appeared after it started. The per-provider poller reuses one provider instance for every poll but never reset its per-cycle caches, so the first poll's result (including an empty leased-hosts list) was cached for the lifetime of the process. This most visibly affected the minds app: because its discovery observer starts a moment before a workspace's cloud host is leased, the observer kept emitting empty imbue_cloud snapshots that clobbered the correct ones in the shared discovery log, so a freshly created workspace flickered in and out and its web UI got stuck on "Waiting for system interface to be ready...". The poller now resets each provider's per-cycle caches at the start of every poll (matching the `reset_caches=True` already used by `mngr list` and on-demand discovery), so newly leased/destroyed hosts are picked up on the next poll.
