Added a `--system-source <slug>` flag to `mngr message` (threaded through `send_message_to_agents`) for marking an automated, non-human message as such. When set, the message content is wrapped in a `<system-injected source="...">...</system-injected>` sentinel (see the new `api/system_injected.wrap_system_injected`).

Automated sends -- e.g. the agentic browser fleet nudging a queued agent that its browser is free -- otherwise arrive in the agent's transcript as ordinary user turns, indistinguishable from something the human typed, and the transcript UI renders them as bare user bubbles. The sentinel lets the `system_interface` transcript parser recognise such a message, strip the wrapper, and stamp `system_source` on the emitted event so the UI can render it collapsed (like a stop-hook feedback chip).

This is UI-only: it changes neither how the message is delivered (the wrapper only prefixes/suffixes the content, adding no new newlines, so it types into the pane identically to the same content sent unwrapped) nor how turns are grouped -- the receiving agent still resumes its next turn exactly as before. `--system-source` accepts a lowercase slug (e.g. `browser-fleet`); a malformed slug is rejected with a `UserInputError`. Genuine human messages omit the flag and are untouched.
