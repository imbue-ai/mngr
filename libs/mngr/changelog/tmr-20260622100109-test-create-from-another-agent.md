Test-only: hardened the `test_create_from_another_agent` e2e tutorial test (cloning an agent via `mngr create --from <agent>`). Resolving `--from <agent>` runs full multi-provider discovery, so the clone-create command and the cloned-agent verification now use generous per-command timeouts instead of the 30s default, and the test-level timeout was raised to match. The verification listing is now scoped to `--provider local` (the provider both agents actually run on) so it is not aborted by unrelated providers that are merely enabled-but-unavailable in the test environment (e.g. a stopped Docker daemon or an unconfigured cloud backend whose plugin is installed in the monorepo venv), which would otherwise make the default `mngr list --on-error abort` exit non-zero.
