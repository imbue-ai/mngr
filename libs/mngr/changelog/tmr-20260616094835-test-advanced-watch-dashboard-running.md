Fixed the `test_advanced_watch_dashboard_running` release test (covering the `watch -n 5 mngr list --running` tutorial block). It was timing out: the test had no `@pytest.mark.timeout` override (so the default 10s applied to the whole function), and its verification query (`mngr list --running --format json`) ran full discovery across every enabled provider (modal/aws/docker/local), taking ~21s. The verification query is now scoped to `--provider local` (matching `test_advanced_fan_out_create`) so it stays fast and never contacts remote providers, and the test carries an explicit `@pytest.mark.timeout(60)`.
