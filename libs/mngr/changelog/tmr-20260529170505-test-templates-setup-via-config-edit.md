Fixed the create-template tutorial e2e tests (`test_templates.py`): the `config edit --scope project` step created a fresh project `settings.toml` that did not opt into the pytest guard, so the template tests could not run under pytest. The setup test now writes the template (with `is_allowed_in_pytest = true`) into the project config the editor would have created. The three template tests now also carry `@pytest.mark.tmux` (they start a local agent) and dropped the inaccurate `@pytest.mark.modal` (the substituted `transfer = "none"` keeps the agent local). Added an unhappy-path test verifying that creating with an undefined template fails with a clear "not found" error.
