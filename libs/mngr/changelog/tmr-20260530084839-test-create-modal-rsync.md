- Release test `test_create_modal_rsync` now names an explicit agent type (`--type command`) on the `mngr create --provider modal --rsync` invocation. The test previously relied on `--type` defaulting to `claude`, which was removed when the installer became responsible for setting the default; the e2e fixture runs no installer, so the command was erroring with "No agent type provided".
- The same test now verifies the actual `--rsync` behavior instead of only asserting success: it stages a gitignored payload file (which the default git transfer skips) plus a gitignored `node_modules/` tree, then confirms via `mngr exec` that rsync carried the payload onto the host while `--rsync-args "--exclude=node_modules"` kept `node_modules/` off it.
