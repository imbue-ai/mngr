Fixed a flaky unit test (`test_create_plugin_manager_blocks_disabled_plugins`) and hardened several sibling config pre-reader tests. These tests previously resolved their project config directory by walking up from the current directory to the git worktree root, which spawns a `git` subprocess; under heavy parallel test load that subprocess could transiently fail, causing the project config layer to be silently skipped and the test to fail. The tests now provision config deterministically via `MNGR_PROJECT_CONFIG_DIR`, removing the git dependency. The git-root resolution path remains covered by the dedicated `resolve_project_config_dir` tests. No user-facing behavior change.
