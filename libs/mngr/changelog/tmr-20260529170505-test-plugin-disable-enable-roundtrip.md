Fixed the `test_plugin_disable_enable_roundtrip` and `test_plugin_disable_affects_create` e2e release tests. The e2e fixture now seeds the project-scope `settings.toml` with `is_allowed_in_pytest = true` so that commands which write project-scope config (the default scope for `mngr plugin enable/disable`) do not leave behind a config file that subsequent commands refuse to load under pytest. Both plugin tests also gained a `@pytest.mark.timeout(120)` marker, since each runs several `mngr` invocations and would otherwise exceed the global 10s per-test timeout.
