Fixed the e2e tutorial tests' shared fixture to scope provider discovery (`enabled_backends`) to the backends actually reachable in the test environment (`local`, plus `modal` when credentials are loaded and `docker` when a daemon socket is present). Previously `mngr list` fanned discovery out across every registered provider plugin, so an unconfigured cloud backend (e.g. AWS, with no credentials) or a missing Docker daemon raised `ProviderUnavailableError` and made commands like `mngr list --include 'host.provider == "modal"'` exit non-zero, breaking the parallel-exec tutorial release test (and any other e2e test that runs `mngr list`).
