Fixed the `mngr list` e2e tutorial tests (e.g. `mngr list --local`) failing with a non-zero exit code when credential-requiring cloud provider plugins (aws, gcp, azure, vultr, ovh, imbue_cloud) are installed but unconfigured. The e2e test profile now disables those providers, matching the pattern the per-provider release tests already use, so the full `mngr list` discovery path no longer trips on a provider it has no credentials for. Modal and Docker stay enabled because the e2e tests exercise them directly.
