- Updated the mega tutorial's "create a copy instead of a worktree" example to use the current `--transfer=git-mirror` flag. The old `--copy` and `--clone` flags were removed when transfer modes were unified under `--transfer` (`none`/`rsync`/`git-mirror`/`git-worktree`), so the tutorial example no longer worked. `git-mirror` is the git-repo way to get an independent copy (its own `.git`) instead of a worktree that shares the source's `.git`; rsync remains the default copy mechanism for non-git projects.
- Brought the corresponding `test_create_copy` e2e release test in line with the corrected tutorial and added verification that the agent runs in an independent copy (separate work_dir, standalone `.git`, mirrored repo content).
