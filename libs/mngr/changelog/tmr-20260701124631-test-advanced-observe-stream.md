Increased the timeout budget of the `test_advanced_observe_stream` release test so it reliably captures the initial `DISCOVERY_FULL` snapshot from `mngr observe --discovery-only`. The stream writes its first full snapshot only after batch discovery finishes polling every configured provider, and a slow-failing cloud provider (e.g. gcp, whose credential resolution probes the absent GCE metadata server for ~12s before raising) plus mngr's cold start pushed that snapshot past the previous 20s bound. No user-facing behavior changed.
