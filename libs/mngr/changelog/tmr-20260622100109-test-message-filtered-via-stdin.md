Hardened the e2e tutorial test fixture so provider discovery is deterministic regardless of the host's ambient cloud credentials or Docker availability. The fixture now restricts `enabled_backends` to the backends the e2e suite actually exercises (`local`, `modal`, and `docker` only when a daemon is reachable).

Previously, the dev monorepo's full set of installed provider plugins (AWS, Azure, GCP, VPS, etc.) all participated in `mngr list` discovery, so a credential-requiring backend with no ambient credentials (e.g. AWS) would raise `ProviderUnavailableError` and make `mngr list` exit non-zero, breaking standalone `mngr list` assertions in the tutorial tests. Docker is now enabled only when a daemon is actually present (probed at the socket level, not via the Docker SDK, to avoid tripping the in-process `docker_sdk` resource guard), since an enabled-but-unreachable backend triggers the same non-zero exit.
