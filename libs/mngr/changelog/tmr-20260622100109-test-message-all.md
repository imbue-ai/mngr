Made the `test_message_all` tutorial e2e test robust on slow filesystems by giving the agent-creation step and the `mngr list | mngr msg -` pipeline (two CLI invocations, each paying the full startup cost) generous per-command timeouts, and raised the overall test timeout to match. Also removed the superfluous `@pytest.mark.rsync` mark: the test targets a single local command agent, so message delivery never crosses a host boundary and rsync is never invoked.
