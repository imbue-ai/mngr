Fixed the `test_plugin_disable_enable_roundtrip` e2e release test so that it disables and re-enables the `claude` plugin at `--scope user` instead of the default project scope. Writing to the default project `settings.toml` produced a fresh config file without the `is_allowed_in_pytest` opt-in, which caused every subsequent `mngr` invocation in the test to be rejected by the pytest config guard. This matches the convention already used by the other e2e plugin tests.
