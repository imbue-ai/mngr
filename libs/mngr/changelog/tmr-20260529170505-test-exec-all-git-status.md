- Fixed the release-marked e2e test `test_exec_all_git_status` (tutorial GIT section). It was marked `@pytest.mark.modal`, but the command it runs (`mngr list --ids | mngr exec - "git status --short"`) only triggers Modal *discovery* via the in-process Modal SDK inside the `mngr` subprocess; it never invokes the `modal` CLI binary that the resource guard tracks, so the test always failed with "marked with @pytest.mark.modal but never invoked modal". Removed the spurious `modal` mark (the `rsync` and `tmux` marks are correct and retained, since `mngr create` genuinely invokes those binaries).
- Strengthened the test's assertion: in addition to checking the pipeline exits 0, it now verifies the fan-out actually reached the created agent by asserting the per-agent `Command succeeded on agent my-task` line appears in stdout.
