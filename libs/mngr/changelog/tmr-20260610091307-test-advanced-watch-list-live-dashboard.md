Made the `test_advanced_watch_list_live_dashboard` e2e tutorial test robust to an enabled-but-unreachable remote provider. The dashboard-content verification now scopes its `mngr list` to the local provider (`--provider local`), so it depends only on the local agent it creates rather than aborting (with the default `--on-error abort`) when, for example, the Docker provider is enabled but not running. The actual tutorial command (`watch -n 5 mngr list`) is unchanged.
