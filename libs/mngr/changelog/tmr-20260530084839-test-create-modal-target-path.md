- e2e tests: the shared e2e session fixture now sets a default agent type (`[commands.create] type = "claude"`) in the test config. The source-coded `claude` default was removed in favor of an installer-written user setting, which left every tutorial `mngr create` invocation that omits `--type` failing with "No agent type provided" in the e2e environment. Configuring the default mirrors what `scripts/install.sh` does for real users.
- `test_create_modal_target_path` now verifies the `:PATH` address suffix actually sets the agent's remote `work_dir` (via `mngr list --format json`), rather than only checking the command's exit code.
