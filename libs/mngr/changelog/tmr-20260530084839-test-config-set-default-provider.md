Fixed the e2e test fixture so that `mngr config set` (which defaults to the project scope) works under pytest. The fixture now seeds the project-scope `settings.toml` with the pytest opt-in and the e2e `connect_command`, instead of putting the connect_command in the local-scope file. This keeps a project-scope `config set` (e.g. `mngr config set commands.create.provider modal`) in the same config layer, so it is not silently narrowed away by a higher-precedence layer and the written file still opts into the pytest run.
