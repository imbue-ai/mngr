Fixed the `test_start_all_via_stdin` e2e tutorial test (STARTING AND STOPPING AGENTS section). Removed a superfluous `@pytest.mark.modal` mark: the test exercises a local-only `mngr list --ids | mngr start -` flow, and mngr reaches Modal only through the in-process Python SDK inside the mngr subprocess, which the resource guard's process-local monkeypatch cannot observe. The mark therefore tripped the "marked with modal but never invoked modal" resource-guard check. Also strengthened the test to stop the agent first and then assert it is running again after the stdin-driven start, verifying the actual effect rather than just the exit code.
