Added an explicit `@pytest.mark.timeout(120)` to the `mngr list --format jsonl` tutorial e2e test (`test_list_format_jsonl`). The test creates two command agents and then runs `mngr list`, each of which runs the full provider-discovery path (routinely ~10s in a fresh environment) -- well past the default 10s per-test timeout. The release CI lane already overrides this globally to 90s, but the explicit per-test timeout makes the test robust locally too, matching the sibling create-then-list tests (`test_list_limit`/`test_list_remote_filter`).
