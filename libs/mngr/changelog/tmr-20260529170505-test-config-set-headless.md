- Fixed the release e2e test `test_config_set_headless` (tutorial `mngr config set headless true` block): added a `@pytest.mark.timeout(120)` marker so the test is not killed by the 10s default timeout while running multiple ~9s mngr invocations, and switched its persistence check to read the project `settings.toml` directly. The previous merged-config readback (`mngr config get headless`) could not work under pytest because `set` writes a fresh project config file that lacks `is_allowed_in_pytest = true`, which the config loader rejects.
- Added an unhappy-path companion test `test_config_set_unknown_key_rejected` that verifies `mngr config set` rejects an unknown configuration key with a clear error and persists nothing.
