The e2e test harness now confines provider discovery to the backends each test actually opts into. Previously every registered backend was loaded during discovery, so a bare `mngr list` in an e2e test would contact remote backends the test never asked for (an unconfigured cloud plugin such as aws, a missing Modal environment, or a Docker daemon that is not running). Any of those reports itself unavailable, which made an otherwise-successful listing exit with the provider-inaccessible code and spuriously fail the test. The e2e fixture now derives an `enabled_backends` allowlist from the test's resource markers -- `local` always, plus `modal`/`docker` only for tests carrying the matching marker -- so unmarked tests stay local-only while provider tests still exercise the full discovery path.

Also dropped a stale `@pytest.mark.rsync` from the launch/check/cleanup common-tasks recipe test: it was inherited from an earlier `--provider modal` version of the test, but the test now uses a local `command` agent (a git worktree, which never invokes rsync), so the resource guard correctly flagged the mark as unused.
