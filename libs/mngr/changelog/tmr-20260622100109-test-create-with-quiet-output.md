Fixed the `test_create_with_quiet_output` release e2e test (tutorial BASIC CREATION output-format block).

The test's verification step ran a full `mngr list --format json`, which under the default `--on-error abort` aborts whenever any registered-but-unconfigured remote backend (e.g. AWS without credentials, or a missing Docker daemon) reports itself unavailable during discovery -- unrelated to whether `--quiet` actually created the agent. The verification is now scoped to the local provider (`mngr list --provider local`), which is where this local command agent lives. The stale `@pytest.mark.rsync` mark was also removed: the default transfer is a git worktree (`git worktree add`), which never invokes rsync, so the mark tripped the resource guard's NEVER_INVOKED check once the test reached completion.
