`mngr wait` documents the new `UNREACHABLE` host state in its valid-states help text and treats it as terminal: a wait completes when a host reaches `UNREACHABLE` (observed up but inner sshd not answering -- the settled condition previously carried by `UNAUTHENTICATED`). `UNAUTHENTICATED` (access credential rejected) now follows `UNKNOWN` semantics -- non-terminal, so a wait keeps polling rather than completing.
