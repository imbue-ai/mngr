The pi lifecycle extension now also writes per-message usage events (cost + tokens) for `mngr usage`, gated on a `pi_emit_usage` marker that the new `imbue-mngr-pi-coding-usage` package provisions. pi reports cost client-side, so each assistant `message_end` appends a `cost_snapshot` (reported cost, tokens, provider-qualified model) to `events/pi-coding/usage/events.jsonl`. Its `cost_mode` is derived from the message provider's `auth.json` login type (`oauth` -> SUBSCRIPTION; `api_key` or anything missing/unrecognized -> API_KEY). Inert unless the gate marker is present, so behavior is unchanged for agents without the usage plugin installed.
