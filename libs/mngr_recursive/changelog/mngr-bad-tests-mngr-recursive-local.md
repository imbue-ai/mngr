Fixed a config-merge bug in the recursive plugin: `RecursivePluginConfig` reimplemented `merge_with` with "override wins if not None" guards, but its fields are never None, so an override layer (e.g. project settings) that touched only one field silently reset the others to their defaults -- dropping values an earlier layer (e.g. user settings) had explicitly set. The plugin now inherits the base `PluginConfig.merge_with`, which merges by `model_fields_set` so untouched fields are preserved.

Also a test-quality pass over the plugin's unit tests (no user-facing behavior change): the uv install-command construction is now built by pure `_build_package_install_command` / `_build_editable_install_command` helpers (snapshot-tested), the `direct_url.json` parsing is testable via `_resolve_editable_source_dir`, and the file-watcher parent-dir dedup is testable via `_unique_parent_dirs`.
