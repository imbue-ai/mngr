`ModalProviderInstance` now surfaces per-sandbox discovery failures to `mngr list` via the new `on_error` callback. When a running sandbox's tags can't be parsed during `mngr list`, `_list_running_host_ids` emits a `ProviderErrorInfo` on `result.errors` instead of letting the sandbox silently vanish from the listing. `mngr list` exits non-zero so the missing host is not mistaken for an absent one.
