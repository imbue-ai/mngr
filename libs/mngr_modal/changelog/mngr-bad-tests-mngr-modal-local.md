The "Done."-only create tests now run a synchronous `mngr exec` after create and assert the actual result: the echo agent's marker file, the custom Dockerfile's marker file, the build-args memory limit (via the enforced cgroup `memory.max`), the transferred untracked file, the checked-out git branch, and the default image's bundled tools (`ttyd`/`uv`). The invalid-snapshot test now requires both the bad snapshot id and a resolution-failure phrase instead of a too-broad generic banner.
