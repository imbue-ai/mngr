- `mngr usage wait --timeout` is now a true upper bound on the wait. Previously, when the poll interval was longer than the timeout (e.g. `--timeout 1` with the default `--interval 30s`), the command slept a full interval before re-checking the deadline, so a 1-second timeout could block for ~30 seconds. The inter-poll sleep is now capped at the remaining time before the timeout, so the wait gives up promptly when the deadline passes.
