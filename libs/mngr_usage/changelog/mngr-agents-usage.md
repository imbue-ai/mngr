Began generalizing `mngr usage` beyond Claude (groundwork; no user-visible change yet).

Added a `TokenSnapshot` data type and a `pricing` module (`compute_cost`) so usage sources that report token counts rather than a pre-computed dollar cost can be priced centrally by the reader. The Anthropic prices mirror `apps/modal_litellm` (and were independently confirmed against a live pi session); an unknown model resolves to no estimate rather than a misleading `$0`.

Moved per-source aggregation behind a reader hook (`aggregate_usage_source`, contributed via `register_hookspecs`) so a usage plugin can ship its own reader. `mngr_usage` now provides two reusable aggregation utilities -- `aggregate_process_cumulative` (the existing Claude strategy, where one cost counter spans `session_id`s) and `aggregate_session_cumulative` (for harnesses where each `session_id` is its own counter, with cost derived from tokens and flagged `REPORTED` vs `ESTIMATED`). The reader dispatches to whichever plugin claims a source, falling back to process-cumulative. `SessionCostRecord` gained `tokens`, `model`, and `cost_provenance` fields.
