Claude agents now confirm the session is ready to receive input before a message is typed, fixing a race where restarting an agent and immediately sending it a message (e.g. the web UI's "interrupt and send") could fail with "Failed to send message to agent (0 successful agents)".

`ClaudeAgent` implements the new `wait_until_ready_for_input` send-gate hook by polling the hook-emitted `session_started` marker (the same readiness signal the startup path already uses), rather than scraping the tmux pane. Because the marker persists for the whole session, an already-running agent -- including one mid-turn whose welcome banner has scrolled off-screen -- passes the check instantly; after a restart the marker is absent until the new session's SessionStart hook re-creates it, so a send that lands during a restart blocks only until the new session is ready, then proceeds.
