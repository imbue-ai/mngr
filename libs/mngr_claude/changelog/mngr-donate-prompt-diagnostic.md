Fixed: the headless-claude "exited without producing output" diagnostic no longer reports a false `.mngr-prompt: does not exist`. The prompt-inputs block previously listed both `.mngr-prompt` and `.mngr-system-prompt` under the agent *work* dir, but `.mngr-prompt` is staged in the agent *state* dir (`$MNGR_AGENT_STATE_DIR/.mngr-prompt`, by `stage_initial_message`) and never lives in the work dir -- so that line read "does not exist" on every run, healthy or not, and dominated the error headline (reading exactly like "build could not find a file / .mngr system prompt") while the real cause (auth, a silent claude exit, etc.) sat further down. The block is now `[prompt-inputs]` and probes each file at the path the assembled command actually reads it from: `.mngr-prompt` in the state dir, `.mngr-system-prompt` in the work dir (the latter is written only by `mngr ask`'s `pre_create_setup`).
