Fixed a regression where the deprecated `use_env_config_dir` claude agent-type option stopped working on the in-host `mngr create` path, which made `minds` mind creation hang forever on "waiting for initial chat agent". An agent-type config that set only `use_env_config_dir = true` (to share the user's `$CLAUDE_CONFIG_DIR`) was rejected with a spurious "Contradictory Claude config" error: constructing the agent in-host serializes and reparses the config, which materialized the default `isolate_local_config_dir=True` into the set fields and made the deprecated-alias reconciliation misread it as a hand-written contradiction.

The deprecated alias is now resolved to a concrete `isolate_local_config_dir` value at the first (faithful) validation, so a later serialize/reparse round-trip stays consistent and keeps honoring the alias. A genuine contradiction -- both `isolate_local_config_dir` and `use_env_config_dir` set to the same (non-inverse) value in a hand-written config -- is still rejected. The deprecation warning is unchanged.
