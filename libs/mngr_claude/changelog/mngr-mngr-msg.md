The claude common-transcript converter now reclassifies slash-command scaffolding as a tool result rather than leaving it as a user message. Claude records a slash command's expansion and its local output as user turns whose content is only an XML-ish envelope (e.g. `<command-name>/login</command-name>`, `<local-command-stdout>...`); no human typed them. These are now handled the same way as `isMeta` framework messages (stop-hook output, etc.), so `mngr transcript --role user --role assistant` and other consumers see the readable conversation without the scaffolding. The reclassified event keeps its original `user` event id, so re-converting a session written before this change does not double-emit the message. Only newly converted turns are affected; scaffolding already materialized as user messages is left as is.
