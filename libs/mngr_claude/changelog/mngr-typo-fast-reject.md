Sending an unrecognized slash command (a typo like `/zzz`) to a Claude agent via `mngr message` now returns in about 2 seconds with an accurate warning ("rejected (unknown command); nothing was executed") instead of polling the full 15-second relaxed window and reporting plain success. Claude records a structured warning the instant it rejects an unknown command; two new rejection probes watch for that record (in both the raw and native transcripts) past the pre-Enter baseline, keyed on Claude's own runtime verdict rather than any list of known commands -- so built-in, plugin, and user-defined commands are never affected, and a typo of any of them is still caught.
