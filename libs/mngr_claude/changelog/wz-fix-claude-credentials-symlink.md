- plugin.py: on remote hosts, after staging this agent's `.credentials.json`, symlink the host's default `~/.claude/.credentials.json` to the per-agent copy (`_bridge_credentials_to_default_claude_home`). An mngr running inside the agent's VM reads credentials from the standard `~/.claude/.credentials.json`, not from the per-agent config dir, so any agent it spawns there would otherwise start unauthenticated -- that path is empty on a Linux VM, since Claude CLI 2.x on macOS moved host-side credential storage to the macOS Keychain (nothing left to rsync in). The symlink (`mkdir -p -m 0700 ~/.claude && ln -sfn`) is idempotent, replaces stale links in place, and lets host-side credential refreshes propagate without re-staging. No-op on local hosts and when no credentials were staged.
