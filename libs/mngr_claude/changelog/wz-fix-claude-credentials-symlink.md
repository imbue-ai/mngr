- plugin.py: bridge `.credentials.json` to the remote's default `~/.claude/.credentials.json` via a symlink to the per-agent dir after `_write_generated_files`. Without this, the host's mngr_claude correctly delivers credentials to the outer agent's per-agent dir, but any nested mngr_claude running inside the VM (e.g. minds workspace_server's mngr creating its own inner agents) looks at `~/.claude/.credentials.json` and finds nothing -- because Claude CLI 2.x on macOS moved host-side storage from `~/.claude/.credentials.json` to the macOS Keychain, the host has no source file to rsync into the VM. The symlink (idempotent, `-sfn` overwrites stale targets) makes the standard-path lookup find the per-agent credentials transparently. Only added on remote hosts and only when credentials were actually staged.
