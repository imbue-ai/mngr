The `--adopt-session` create option and its agent-agnostic validation moved out of the claude plugin into core (a built-in `imbue.mngr.agents.builtin_adopt_session` plugin), since session adoption is now a capability shared by every interactive agent. Claude's `register_cli_options` no longer declares the option, and its `on_before_create` retains only the claude-specific fail-fast pre-resolution of named session ids. Claude's session-store scanner now routes through the shared `iter_agent_session_paths` helper. No change to claude's user-facing adoption behavior.
