Generalized the `--adopt-session` create option (declared by the claude plugin) from claude-only to any agent type that supports session adoption (`HasSessionAdoptionMixin`). Its `on_before_create` validation now rejects only agent types that lack session-adoption support, and claude's fail-fast session pre-resolution runs only for claude agents; other adoption-capable agents resolve against their own native stores. The option help text is now agent-agnostic.
