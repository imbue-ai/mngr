Fixed: `imbue-mngr-donate` now declares a runtime dependency on `imbue-mngr-claude`. `mngr donate` launches a `headless_claude` agent (provided by `imbue-mngr-claude`), but the plugin did not depend on that package, so a standalone install of donate via `uv` (e.g. `uvx --from ...#subdirectory=libs/mngr_donate` or `uv tool install` without a separate `--with imbue-mngr-claude`) failed at agent-type resolution with `Error: Unknown agent type 'headless_claude'`. The `uv run --project <fork>` install path worked only because `uv sync --all-packages` happens to install the whole workspace; the standalone path had no such luck. `imbue-mngr-claude` is published on PyPI (not in `UNPUBLISHED_PACKAGES`), so the `==0.2.17` pin resolves both from the workspace and from PyPI.
