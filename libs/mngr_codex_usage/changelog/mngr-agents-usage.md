New package `imbue-mngr-codex-usage`: cost/usage tracking for Codex agents in `mngr usage`. Codex reports cumulative token usage (no dollar cost), so cost is estimated from the pricing table and aggregated session-cumulatively. The writer (`codex_usage.sh`) reads codex's rollout stream and emits one `cost_snapshot` per `token_count` item; mngr_codex's background-tasks supervisor launches it when present. It also maps codex's 5h/7d rate-limit windows (subscription mode), so Codex subscription agents get Claude-style windows.

The writer now tracks a byte-offset cursor (persisted under `plugin/codex/.usage_cursor`) so each poll processes only the new tail of the rollout rather than re-reading the whole transcript -- O(new bytes) per tick, important for long Codex sessions. Re-emission after a crash is harmless because the session-cumulative reader keeps the freshest reading per session.

The writer's event-emitting logic moved out of an inline `python3` heredoc in `codex_usage.sh` into a standalone `codex_usage_emit.py` (installed alongside the writer), so it is type-checked, linted, and unit-tested directly. Malformed rollout lines and unreadable cursor state are now logged at warning level instead of being swallowed silently.
