Serialize all `gh` CLI invocations in the GitHub data source behind a process-wide lock. Kanpan used to fire two `gh pr list` calls in parallel (and the conflicts/unresolved metadata fetches alongside them); on a cold cache, the concurrent writes corrupted gh's HTTP cache file (the SearchType introspection entry), after which every `gh pr list` returned `invalid character '{' after array element` until the 24h TTL expired. Running gh calls one at a time eliminates the race; the worst case is a few hundred ms of added latency on a background refresh.
