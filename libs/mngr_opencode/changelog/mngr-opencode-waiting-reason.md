Implemented the `waiting_reason` listing field for the `opencode` agent type, matching claude and codex. `mngr list` now reports *why* a WAITING opencode agent is blocked: `PERMISSIONS` when a tool is waiting on an approval prompt (an `ask` permission policy), or `END_OF_TURN` when the agent is idle with its turn complete.

The in-process lifecycle plugin now tracks opencode's `permission.updated` / `permission.replied` events, keeping a `permissions_waiting` marker present while any prompt is open (it tracks pending permission ids, so concurrent prompts from task-tool subagents are handled). While that marker is present the agent is promoted from RUNNING to WAITING, so a blocking approval prompt no longer reads as RUNNING. A stranded prompt is cleared when the root turn ends, as a safety net.
