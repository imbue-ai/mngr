Fixed imbue_cloud discovery dropping agent labels in the streaming (per-provider) path. `discover_hosts_and_agents` read each agent's raw data but only kept its id and name, so the `DiscoveredAgent` it emitted had no `certified_data` and therefore no labels -- even though the same raw data (and its labels) is what the richer `get_host_and_agent_details` path reads. Any consumer that filters agents by label got nothing back: most visibly, the minds app's system_interface forward runs with `--agent-include has(agent.labels.is_primary)`, so every streaming discovery snapshot silently filtered out the workspace's primary agent, leaving the workspace stuck loading. The streaming refs now carry the raw per-agent data as `certified_data`, matching the rich path, so labels flow through and label-based filters work.
