The outer-SSH authentication-rejection fallback in discovery now mints the new `HostState.UNREACHABLE` instead of `UNAUTHENTICATED`. The two conditions previously shared a state but need opposite treatment: `UNAUTHENTICATED` (container observed running, inner SSH dead) is fixed by a host restart, while an outer-SSH auth rejection means observation is impossible and a restart routes through the same rejected key. Consumers (e.g. the minds recovery page) can now tell them apart. The non-auth outer-SSH failure fallback stays `UNKNOWN`, and cached agent identity is still re-attached through the unreachable window.

Degraded observations no longer read as crashes: an outer listing that succeeds but produces no container state, and an unrecognized docker container status, now surface `UNKNOWN` instead of `CRASHED` (with a "could not determine state" note). Unreadable state is non-evidence about the container, and `CRASHED` triggers unattended auto-restarts in consumers and makes the host GC-eligible.
