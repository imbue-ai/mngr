Git commits on imbue cloud pool hosts are no longer attributed to the operator who baked the host (e.g. "Josh Albrecht").

When mngr bakes a pool host it does a cross-host `mngr create`, which copies the baker's `git config user.name/email` into the baked services checkout at `/mngr/code`. Because every adopting user's agent mirrors from that checkout, users' commits inherited the baker's identity. The pool-host finalize step now overwrites that repo-local identity with the neutral `minds-bootstrap <bootstrap@minds.local>` before the host ships. Per-agent commits are separately attributed to the agent by the workspace template's Bash-command rewrite hook; this only governs the leftover non-agent commits on the shared checkout. Local `mngr` worktree agents are unaffected -- they never take this copy path, so they still inherit your identity as before.
