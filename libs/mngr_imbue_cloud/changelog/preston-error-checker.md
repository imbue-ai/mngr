Added `.external_worktrees` to `_VENDOR_RSYNC_MANUAL_EXCLUDES` (the single source of truth for the monorepo -> FCT `vendor/mngr/` rsync excludes used by `sync_mngr_into_template`, and by the `pool create --mngr-source ...` bake flows). It is gitignored only via a `**/`-prefixed rule that macOS's stock rsync (openrsync) does not honor, so without an explicit exclude a macOS bake would copy a stale `.external_worktrees/` tree into the baked container image. This keeps the constant in step with the matching `just minds-start` and `propagate_changes` fixes. No product behavior change.
