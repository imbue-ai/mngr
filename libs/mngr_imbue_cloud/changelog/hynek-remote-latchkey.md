When rebuilding a leased VPS's container (the slow-path remote workspace creation), imbue_cloud now publishes the per-VPS latchkey gateway port into the container by setting `extra_container_port_mappings` on the delegated `vps_docker` provider config. The mapping (`0.0.0.0:OUTER_PORT -> INNER_PORT`) is sourced from `mngr_latchkey`'s `OUTER_PORT`/`INNER_PORT` constants, so an agent in a remote workspace can reach the latchkey gateway running on the leased VPS. Adds a dependency on `imbue-mngr-latchkey`.
