Discovery now distinguishes two conditions that previously both collapsed onto `HostState.UNAUTHENTICATED`. An outer-SSH authentication rejection (the host answered but rejected this machine's key) keeps `UNAUTHENTICATED`: observation of the container is impossible and a restart routes through the same rejected key, so consumers (e.g. the minds recovery page) treat it as terminal. A running container whose inner SSH is unreachable now surfaces as the new `HostState.UNREACHABLE` (the host is up; we just can't get inside it), which a host restart can revive. The non-auth outer-SSH failure fallback stays `UNKNOWN`, and cached agent identity is still re-attached through the unreachable window.

Degraded observations no longer read as crashes: an outer listing that succeeds but produces no container state, and an unrecognized docker container status, now surface `UNKNOWN` instead of `CRASHED` (with a "could not determine state" note). Unreadable state is non-evidence about the container, and `CRASHED` triggers unattended auto-restarts in consumers and makes the host GC-eligible.
