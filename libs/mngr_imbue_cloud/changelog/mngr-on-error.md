`ImbueCloudProvider` now surfaces per-host discovery failures to `mngr list` via the new `on_error` callback. When a leased host's outer SSH fails during `mngr list` -- key rejected, host unreachable, or the outer listing script exits non-zero -- `_collect_listing_raw_via_outer` emits a `HostErrorInfo` on `result.errors`. The host still appears as a fallback stub (so it is not dropped from the listing), and `mngr list` exits non-zero so the degraded result is not mistaken for a clean one.
