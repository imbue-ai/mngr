Fixed a bug where applying a latchkey permission grant could fail with a 500 (`ENOENT ... latchkey_permissions.json.tmp.<hex>`) when the per-host directory did not exist yet. The gateway's `permissions` extension (`POST /permissions/rules`) now creates the target file's parent directories (e.g. `hosts/<host_id>/`) before writing, matching its documented "creates the target file if it does not yet exist" behavior.

Added `recover_host_permissions_for_agent` to `agent_setup`: a best-effort repair that, given an agent's opaque permissions handle, host id, and agent id, materializes the canonical per-host permissions file (and swings the opaque handle's symlink onto it) when that file is missing, and idempotently re-registers the agent in the host's `minds-api-proxy` allowlist (closing the gap where discovery-time auto-register skipped the agent because the host file did not exist yet). Used by minds to self-heal hosts whose agent-creation finalize/link step was skipped or failed.
