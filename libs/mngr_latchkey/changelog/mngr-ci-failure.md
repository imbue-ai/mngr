Added `find_host_for_agent`, a best-effort reverse lookup over the per-host `latchkey_permissions.json` allowed-agent lists that `register_agent_for_host` maintains. The minds desktop client uses it to map a permission request filed by an agent that has dropped out of the live discovery snapshot (e.g. the in-VM chat agent) back to its host, so the request can be displayed against the host's workspace instead of being silently hidden.
