Registered a `node` resource guard (`register_guards.py` + a `resource_guards` entry point), so node joins tmux/docker/lima/modal as a guarded binary: invoking `node` in a test without `@pytest.mark.node` now fails loudly, and the marker is provided automatically (no hand-maintained marker list). Migrated the two Node-driven extension tests (`minds_api_proxy_test.py`, `permission_requests_test.py`) off their `skipif(shutil.which("node") is None)` guards to `@pytest.mark.node`; their node-spawning fixtures declare `@fixture_uses_resources("node")`, and the spawns now inherit the test environment (bare `node`, dropping the `shutil.which` absolute-path lookup). These tests previously skipped silently on the Node-less offload image, so they exercised nothing in CI; they now run in the dedicated `test-node` job, and a missing Node binary is a hard failure.
