Marked the two Node-driven extension tests (`minds_api_proxy_test.py`, `permission_requests_test.py`) with the repo-wide `node_required` pytest marker, replacing their `skipif(shutil.which("node") is None)` guards. These tests spawn a Node child process to drive the `.mjs` gateway extensions; previously they silently skipped on the Node-less offload image, so they exercised nothing in CI. They now run in the dedicated `test-node` job, and a missing Node binary is a hard failure rather than a silent skip.
