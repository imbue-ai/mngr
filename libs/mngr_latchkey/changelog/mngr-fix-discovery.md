The latchkey discovery handler no longer emits an error-level log (and Sentry event) every 30s discovery cycle for an agent host whose sshd is unreachable. The tunnel manager's new per-host connect-failure backoff turns repeat attempts into a cheap `SSHConnectionBackoffError`, which the handler logs at debug; genuine (post-backoff-window) connection attempts still log failures at error level, at most once per five minutes per host in the steady failing state.
