`VpsDockerProvider._read_records_from_vps` (inherited by the Vultr and OVH providers) now surfaces a per-VPS outer-SSH failure to `mngr list` via the new `on_error` callback: a VPS that fails outer SSH produces a `ProviderErrorInfo` on `result.errors` instead of being silently swallowed by `logger.warning`. The other VPSes' hosts still list normally, and `mngr list` exits non-zero so the partial result is not mistaken for a complete one.
