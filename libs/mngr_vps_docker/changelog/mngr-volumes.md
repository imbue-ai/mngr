Added two no-op extension hooks to `VpsDockerProvider` -- `_persist_host_record_externally` and `_delete_host_record_externally` -- so a provider can mirror the authoritative on-volume host record to an external store (e.g. an object-storage bucket) for offline reads. The base provider calls them right after every on-volume host-record write (create, stop, snapshot, rename, certified-data sync) and on host destroy/delete. Default behavior is unchanged for providers that do not override them (Vultr, OVH, imbue_cloud).

Added a `HostStateStore` abstract base (`host_state_store.py`): the uniform interface for a provider's external host/agent-record mirror (persist/delete host record, persist/remove/list agent records, read host record). The AWS and Azure providers select one implementation -- bucket-backed or instance/VM-tag-backed -- and call it instead of branching on bucket-vs-tags at every site. As part of this, offline discovery (`discover_hosts_and_agents`) now tolerates a transient per-host store error -- it skips just that host's agents instead of aborting the whole offline sweep.
