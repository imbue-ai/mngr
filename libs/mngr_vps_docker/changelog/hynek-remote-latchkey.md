Added an `extra_container_port_mappings` field to `VpsDockerProviderConfig`: a generic `host-bind -> container-port` map (e.g. `{"0.0.0.0:1989": "1989"}`) that is merged with the always-present sshd mapping when creating a container. This lets higher-level providers (e.g. `imbue_cloud`) publish additional ports into every container -- such as the per-VPS latchkey gateway port -- without this generic provider needing to know what they are for. The container port map is now built by a small `_build_container_port_mappings` helper.
