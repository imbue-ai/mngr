Added `scripts/repro_ttyd_tmux_copy_paste.sh`, a self-contained local reproduction of the minds workspace copy-paste / mouse-wheel issue. It runs ttyd against a tmux session on a dedicated socket (mirroring how the ttyd plugin attaches to an agent's separate session) and exposes the relevant tmux options (`mouse`, `alternate-screen`) and ttyd client options as environment-variable knobs, so the combination that makes both copy-paste and scroll work can be found quickly.
