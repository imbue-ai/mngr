Closed several gaps found by an audit of the modal_proxy test suite, and fixed one latent bug in the local testing fake:

- Added behavioral tests for `DirectVolume`'s retry wiring. Previously only the retry *predicate* (`_should_retry_volume_op`) was tested; nothing exercised the `@retry` decorator itself. The new tests confirm a transient error is retried and recovers, a non-retryable error surfaces immediately after one attempt, and `stop_after_attempt(5)` bounds the retries while `reraise=True` surfaces the original error -- so deleting the decorator, swapping the predicate, flipping `reraise`, or changing the attempt count is now caught.
- Added tests for the `_translate_exceptions` decorator (the actual error boundary applied to most `Direct*` methods), covering modal-error translation with cause chaining, pass-through of non-modal exceptions, and the no-error return path. Only the pure `_translate_modal_error` helper was tested before.
- Pinned the `_should_retry_volume_op` behavior for `PermissionDeniedError` (not retried for volume data ops), an untested branch, with a comment noting the deliberate contrast with the app-lookup/teardown retry flows in mngr_modal.
- Fixed a path-escape check in `TestingVolume._resolve`: it used a string-prefix (`startswith`) comparison that would let a sibling directory whose name merely extends the volume root's (e.g. `<root>_evil`) slip through. It now uses `Path.is_relative_to`.
